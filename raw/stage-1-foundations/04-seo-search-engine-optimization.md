## 01. SEO의 개념

### 검색엔진 최적화란

**핵심 질문**: 어떻게 우리 쇼핑몰이 잘 보이게 할까?

### 우리의 쇼핑몰을 방문하는 경로

```
검색(네이버, 구글) → SNS 광고 → 쇼핑몰 접속
```

**사람들이 어디로부터 유입이 가장 많이 되는가?**
- 사람들은 검색을 통해 접속하거나, SNS 광고 등을 통해 접속

### 검색 결과

검색 결과 페이지에는 여러 사이트가 나열됨 (예: 여성 쇼핑몰 검색 시 스타일닷컴, 소니유나라, 쿠팡, 나무위키, 아뜨랑스, Wix 등).

핵심 포인트:
- SNS 홍보 외 일반 검색
- **어떻게 상단에 노출이 가능한가?**
- 알고리즘에 더 많이 노출되는 방법
- 키워드 설정

### 검색엔진 최적화 = Algorithm?

검색엔진 최적화의 핵심은 결국 **알고리즘**을 이해하는 것.

## 02. 현대 검색엔진의 시작

### 시작 시점

**1996년**

### 연구 프로세스

```
웹 사이트 다운로드 → DB 저장 → 웹 구성요소 분류·분석
```

웹의 구성 내용들을 분석, 분류하여 **최적의 검색 알고리즘을 만들어감**.

### 서버 증설

프로젝트 기간이 길어짐에 따라 서버를 증설.
(작은 서버 → 거대한 서버 팜으로 확장)

### Google

구글의 공동 창업자: **래리 페이지(Larry Page), 세르게이 브린(Sergey Brin)**

1996년 스탠포드 대학원에서 시작된 검색엔진 연구가 1998년 Google 창립으로 이어짐.

## 03. 검색엔진 최적화

### 검색엔진 최적화의 본질

**"웹 사이트를 다운받아 모은다"**

- 검색엔진이 새로운 사이트를 탐색하는 방법
- **114억 개**에 달하는 웹 사이트
- 핵심 키워드를 통한 노출 전략

### Hashtag

해시태그 예: #크리스마스, #맛집, #새해, #오운완, #럽스타그램, #휴가

**SNS에서 흔히 사용되는 해시태그도 검색 최적화의 시도 중 하나**이다.

### 사이트 직접 등록

- 검색엔진이 내 웹페이지를 찾기를 가만히 기다리고 있지 않음
- **네이버 웹마스터 도구, 구글 콘솔에 들어가 직접 사이트를 등록하는 방법도 존재**
- 네이버에서 제공하는 도구: **Search Advisor**

### 구글의 검색엔진 3단계

```
크롤링 → 색인 생성 → 최종 검색결과 제공
```

**색인(Index)이란?**
많은 데이터들이 쌓였을 때 원하는 정보를 검색하는 속도는 당연히 느려진다.
이 검색 성능의 속도를 높여주기 위해서 데이터들을 저장할 때 별도의 내용들을 저장해 주는 방법.

### 구글의 검색엔진 최적화 — 용어 설명

가이드에 사용된 중요한 용어:

- **색인 (Index)**: Google에서는 알고 있는 모든 웹페이지를 색인에 저장합니다. 각 페이지의 색인 항목은 해당 페이지의 콘텐츠 및 위치(URL)를 명시합니다. Google에서 페이지를 가져와서 읽고 색인에 추가하면 색인이 생성됩니다.
  - 예: Google에서 오늘 내 사이트의 페이지 색인을 생성했다.

- **크롤링 (Crawling)**: 신규 또는 업데이트된 웹페이지를 찾는 프로세스입니다. Google은 링크를 따라가거나, 사이트맵을 읽거나, 다른 여러 방법으로 URL을 찾아냅니다.
  - 예: Google은 웹을 크롤링하여 새 페이지를 찾은 다음 필요한 경우 색인을 생성한다.

- **크롤러 (Crawler)**: 웹에서 페이지를 크롤링(가져오기)한 다음 색인을 생성하는 자동 소프트웨어입니다.

- **Googlebot**: Google 크롤러의 일반적인 이름입니다.
  - 예: Googlebot은 웹을 끊임없이 크롤링한다.

- **SEO (Search Engine Optimization)**: 검색엔진 최적화, 즉 검색엔진에서 찾기 쉽도록 사이트를 개선하는 프로세스입니다. 검색엔진 최적화 작업을 하는 사람의 직책을 의미하기도 합니다.
  - 예: 우리는 웹 인지도를 높이기 위해 검색엔진 최적화 전문가를 새로 고용했다.

**→ 용어 설명만 읽어봐도 어떤 방식으로 웹의 정보를 가져오는지 짐작 가능하다.**

## 04. 크롤링

### 크롤링이란

**컴퓨터를 활용 웹페이지를 자동으로 그대로 긁어서 가져옴**

### Parsing (파싱)

**Parsing이란?**
파싱은 **구문 분석**이란 말로, 특정하게 규칙이 정해져 있는 문장을 잘라내 분석하는 방법이다. 크롤링뿐만 아니라 프로그래밍 세계에선 두루 사용되는 데이터 처리 기법이다.

**"우리가 원하는 건 가격정보 이다"**

예시:
```html
<a href="https://prod.danawa.com/info/?pcode=17735156&amp;cate=112758" target="_blank"
   onmousedown="_trkEventLog('15상품리스트_가격')">
  <strong>1,899,000</strong>원
</a>
```

가격정보는 `<strong>` `</strong>` 블럭 안에 적혀있음.

### 크롤링과 파싱 — 전체 흐름

```
웹사이트 크롤링 → Parsing 통한 가격정보 획득 → 최종 결과를 DB에 저장 or 출력
```

### 크롤링과 파싱 — 비유

**포켓몬 빵 비유**:
```
포켓몬 빵 → 포장지와 빵은 분류 → 목표인 포켓몬 스티커만 획득
```

= 크롤링(빵 통째로 가져옴) → 파싱(필요한 부분만 추출) → 원하는 데이터 획득

### 크롤링 방지

**모든 사이트를 크롤링 할 수 있지 않으며, 검색 사이트 노출 방지 방법도 존재한다.**

### 크롤링의 목적

다양한 목적으로 크롤링은 활용된다:
- 시장가격 조사를 위한 데이터 모음 (ex. 쇼핑몰 판매)
- AI 훈련을 위한 데이터 확보
- 가격 비교 사이트

### 크롤링의 이슈

```
크롤링할 사이트 접속 → 사이트의 트래픽 발생 → 유지비용 증대 / 속도 저하
```

크롤링 때문에 생기는 트래픽 비용은 크롤링 하는 사람이 지불하는 것이 아닌 **웹 사이트 관리자가 부담**.

**많은 사람이 크롤링을 한다면 운영 비용이 증가함**

Good news for archivists, academics, researchers and journalists: Scraping publicly accessible data is legal, according to a U.S. appeals court ruling.

### 크롤링 이슈 — 법적 측면 (한국)

**'여기어때 무죄'로 본 '크롤링'…합법과 불법 사이**

크롤링(crawling·온라인상 정보 수집 및 가공)을 둘러싼 IT업계 소송이 잇따르고 있는 가운데, 최근 업계 1위인 '야놀자' 제휴 숙박업소 정보를 크롤링을 이용해 빼돌린 혐의를 받는 숙박 플랫폼 '여기어때'가 무죄를 확정받으면서 어디까지 합법이고 어디부터 불법인지 관심이 모아진다.

크롤링과 관련된 법조문은 **정보통신망법 제48조**에 규정된 '정보통신망 침해행위 금지' 규정이다. 해당 규정은 '누구든지 정당한 접근권한 없이 또는 허용된 접근권한을 넘어 정보통신망에 침입하여서는 아니된다'고 적시돼 있다.

출처: https://www.ajunews.com/view/20220515155129332

### 크롤링이 사라지지 않는 이유

**데이터의 가치가 그 어느때보다 크기 때문**

### 데이터에 진심 — 공식 데이터 제공 서비스

- **구글 트렌드 (Google Trends)** — 구글이 제공. 전 세계 사용자들의 검색어 동향을 살펴볼 수 있음
- **네이버 데이터랩 (NAVER DataLab)** — 네이버에서 제공. 쇼핑인사이트(분야별 인기 검색어), 인기분야 등 분석 가능

### 색인 (Index)

**데이터를 분류하여 쉽게 표시**

색인을 가장 쉽게 확인할 수 있는 건, 포털 서비스들의 웹 주소.

예시:
- 구글에서 `site:naver.com` 이라고 검색
- 네이버의 다양한 서비스들 검색됨 (네이버 고객센터, 네이버 부동산, 네이버 OGQ마켓, 네이버 클로바 등)
- 네이버 크롤링 이후 색인을 통해 분류하는 엔진의 기능이 있기 때문에 가능

### 색인 — 온라인 마케터 관점

**색인 과정에서 더 효율적인 노출이 될 수 있도록 작업**

이것이 SEO 작업의 핵심이며, 온라인 마케터의 역할 중 하나.

## 강의 핵심 메시지 요약

1. **검색엔진의 작동 = 크롤링 + 색인 + 검색결과 제공** (구글 3단계)
2. **크롤링 = 통째로 긁어오기, 파싱 = 원하는 정보만 추출** (포켓몬 빵 비유)
3. **SEO = 색인 과정에서 더 효율적으로 노출되도록 작업** (마케터의 영역)
4. **크롤링은 합법과 불법의 경계가 있는 회색지대** (정보통신망법 48조, 여기어때 판례)
5. **공식 API를 통한 데이터 활용도 있음** (구글 트렌드, 네이버 데이터랩)
