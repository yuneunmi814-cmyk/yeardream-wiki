## 01. 컴퓨터와 소통하는 방법, 0과 1

### 기계어

컴퓨터가 직접 알아듣는 언어는 **0 or 1**.

### 생활에서의 기계어

일상 속 모든 전자기기는 결국 0과 1로 동작:

- TV On / Off → 1 / 0
- 컴퓨터 On / Off → 1 / 0
- 에어컨 On / Off → 1 / 0

### 기계어의 실 예시

```
0000 1001 1100 0110 1010 1111 0101 1000
1010 1001 1100 0110 1010 1111 0101 1000
1100 0110 1010 1111 0101 1000 0000 1001
```

### 기계랑 소통하는 엔지니어

> **복잡한 기계어를 사람이 사용하기 쉽게 만들어 기계를 활용하게 한다**

## 02. 프로그래밍 언어로 소통하는 개발자

### 다양한 프로그래밍 언어

Javascript, Swift, Kotlin, C++, Go, C, Rust, Assembley(원문 표기), Java, C#, Python …
모두 **컴퓨터란 기계 위에서 실행**되기 위한 언어들.

### 프로그래밍 언어 vs 기계어

| 일반적인 코드 작성 예시 | 컴퓨터가 인식하는 기계어 예시 |
|---|---|
| `public void ShirtmeshChange(int i) { mainBody.sharedMesh = bodies[i]; mainShirts.sharedMesh = shirts[i]; mainShirts.material = shirtMaterials[i]; } public void PantsmeshChange(int i) { mainLegs.sharedMesh = legs[i]; mainPants.sharedMesh = pants[i]; mainPants.material = pantsMaterials[i]; }` | `0000 1001 1100 0110` <br> `1010 1001 1100 0110` <br> `1100 0110 1010 1111` |

### 컴퓨터를 위한 번역

```
Code 작성 → 인터프리터/컴파일러 → 기계어 번역 → 컴퓨터 인식
```

### Low level vs High level language

기계와 직접적으로 더 가까운 언어를 **Low level language** 라고 통칭.

> **메모**: High level 이라고 해서 고 차원의 언어라는 의미가 아닙니다. Low level 일수록 기계를 프로그래머가 직접적으로 컨트롤 할 수 있는 속도도 더 빠릅니다. 하지만 상대적으로 작성이 어렵고 실수하기가 쉽기 때문에 이러한 단점들을 보완하고자 나온것이 High level language 라고 이해하면 편합니다.

### 클라이언트와 서버 (Front-end 와 Back-end)

```
[배달앱]  ⇄  [서버]
 (클라이언트)    (서버)
 Web: 프론트엔드   Web: 백엔드

  → 배달음식 주문
  ← 주문완료/배달이동
```

## 03. 컨트롤이 다른 이유, 운영체제

### 운영체제란?

> **컴퓨팅 기계를 쉽게 다루기 위한 시스템 소프트웨어**

### 다양한 운영체제

- **MacOS** / **Windows** / **Linux** (데스크탑)
- **Android** / **iOS** (모바일)

아이폰에서 실행되는 → iOS 운영체제
안드로이드 폰에서 실행되는 → Android 운영체제

### 우리가 스마트폰을 사용하는 과정

```
스마트폰 공장제작 → 운영체제 설치 → App 제작 → 사용자 활용
```

(아이폰은 두달 뒤 OO월 OO일에 출시됩니다. 안드로이드 전격출시! — 출시 시점에 OS가 함께 배포된다는 맥락의 예시)

### 스마트폰의 카메라 활용

길이 측정을 위한 AR 어플리케이션(예: 0.12mm / 0.14mm 단위 측정) 같은 앱이 가능한 이유:

1. 스마트폰의 카메라 활용을 위해서 카메라에 접근 권한이 필요
2. 운영체제는 카메라에 접근할 수 있는 코드를 제공
3. 개발자는 이를 활용하여 어플리케이션 제작
   - 운영체제 별로 카메라 제공 코드가 다르기에 다르게 제작

### 스마트폰의 OS 예시

하드웨어 접근 권한을 사용자에게 알려주는 예시
(예: "AR Measure PRO AR Foundation이 사진을 찍거나 동영상을 녹화하도록 허용하시겠습니까? — 앱 사용 중에만 허용 / 이번만 허용 / 허용 안함")

> **운영체제는 이런 컨트롤을 사용자가 쉽게 할 수 있도록 도와주는 시스템 소프트웨어**
