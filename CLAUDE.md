## 페이지 네이밍 규칙

- **한글 + 하이픈** (현재 표준): `데이터베이스-기초.md`, `1주차-1일차-00-컴퓨터의-이해.md`
- 영어 표준 약어(HTTP·REST·API·SEO·SQL·SSH·DB·OS 등)는 **영어 유지**: `HTTP-프로토콜.md`, `REST-API.md`, `SEO.md`
- 한국어 표현이 어색한 CS 표준 용어는 영문 유지 가능 (예: 새 개념 등장 시 판단)
- 강의 페이지: `N주차-N일차-CH-주제.md` (예: `1주차-2일차-00-운영체제-소개.md`)
- 단계 종합: `syntheses/N단계-종합.md` 형식 (영문 `stage-NN-review.md`도 허용)
- 영문 alias·CS 표준 용어는 페이지 frontmatter `aliases:`에 명시

**🎯 이 부분의 핵심 메시지**: 한 줄 요약.

⚠️ **흔한 오해** (해당 시): 정정 + 보강.

(슬라이드 묶음마다 위 구조 반복)

**4섹션 규칙**:
- 🚀 예습: 5분 안에 큰 그림 잡기용. 짧고 가독성.
- 📖 본문: 슬라이드 5~10개씩 묶어 소제목. ✦ 마크는 LLM 보강 (검증 가능). 키워드는 [[위키링크]].
- 🔄 복습: 빈칸 회상 질문. ❓는 비워둠 (학습자가 채움).
- 💡 일상 비유: 강의 본문에 없는 친숙한 비유. 콘텐츠 자산으로도 활용.

## 워크플로우

### 1. Ingest (새 소스 추가)

사용자가 raw/에 파일 떨어뜨리고 "ingest" 요청하면:

1. 소스 전체 읽기
2. **핵심 takeaway 3-5개 먼저 사용자에게 제시** → 강조점 가이드 받기 (절대 생략 금지)
3. 적절한 카테고리에 요약 페이지 작성
4. **강의 자료라면 `lectures/` 페이지를 4섹션 구조(🚀 예습 / 📖 본문 / 🔄 복습 / 💡 일상 비유)로 작성** ([[wiki/_meta/학습-워크플로우]])
5. 신규 개념 → `concepts/` 신규 페이지 생성 + frontmatter에 stage 기록
6. 기존 개념 등장 → 해당 페이지의 sources +1, 새 맥락 누적
7. `wiki/인덱스.md` 갱신
8. `wiki/커리큘럼.md` 진도 체크 업데이트
9. `wiki/로그.md` 항목 추가
10. 마지막에 영향받은 파일 전체 리스트 보고

**강의 페이지 4섹션 구조 요건** (위 "강의 페이지 템플릿" 참조):
- 🚀 **예습** (5분용): 한 줄 요약 + 핵심 단어 5~7개([[위키링크]]) + 큰 그림
- 📖 **본문** (정독용): 슬라이드 5~10개씩 묶음. 강의 못 들은 학습자가 이것만 보고 따라잡을 수 있게. **✦ 마크는 LLM 보강** (검증 가능하게). 키워드는 [[위키링크]]. ⚠️ 흔한 오해 명시. 묶음마다 한 줄 핵심 메시지.
- 🔄 **복습** (회상용): 빈칸 질문 3~5개 + "헷갈렸던 것 ❓" 비워두기 (학습자가 채움)
- 💡 **일상 비유**: 카톡·인스타·맥북 등 친숙한 비유 1~2개

### 2. Query (질문 응답)

1. `wiki/인덱스.md` 먼저 읽고 관련 페이지 후보 탐색
2. 후보 페이지 정독
3. 답변 + 인용 (`[[페이지명]]` 형식)
4. 답변이 새 통찰이면 → `syntheses/` 페이지로 저장할지 사용자에게 물어보기

### 3. Lint (헬스체크)

"lint" 요청 시:
- 모순: 같은 개념의 다른 페이지 충돌
- 고아 페이지: 아무 데서도 링크 안 된 페이지
- 누락 개념: 본문에서 자주 언급되는데 페이지 없는 것
- 낡은 정보: 새 소스로 superseded
- 끊긴 링크
- **단계 누락**: 진도와 위키 페이지 수가 안 맞는 단계

리포트 형태로 제시, 우선순위는 사용자가 결정.

### 4. Synthesize (단계 종료시)

각 단계 끝나면 "synthesize stage N" 요청.
- 그 단계 모든 강의/과제/개념 페이지 종합
- 핵심 인사이트, 연결고리, 본인이 강해진 영역, 약한 영역
- 다음 단계 진입 전 보강할 것
- 인스타 콘텐츠로 풀 수 있는 토픽 5개 제안
- 프로젝트윤365 솔루션화 가능한 것 제안

## 절대 규칙

1. `raw/` 안의 파일은 어떤 경우에도 수정하지 않는다
2. 추측을 사실처럼 쓰지 않는다. 모르면 "추가 학습 필요" 명시
3. 강의에서 얕게 다룬 건 얕게 표기. 깊이 과장 금지
4. 한 ingest에서 만진 파일은 마지막에 리스트로 보고
5. 개념 페이지는 다시 쓰지 말고 누적 — 사용자의 학습 곡선을 반영
6. 모든 강의 페이지에 "프로젝트윤365 / 인스타 콘텐츠 적용 아이디어" 섹션 필수
7. 3단계 시작 전에 사용자에게 트랙 선택 확정 여부 확인
8. 캐글 진행 상황은 항상 별도로 추적 (월별 미니챌린지 누락 방지)
