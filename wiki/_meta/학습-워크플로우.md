# Study Workflow — 학습 운영 가이드

> ⚠️ **이 파일은 학습 운영 가이드입니다. raw 자료가 아닙니다.**
> 위키 운영·학습 사이클 자체에 대한 메타 문서.

---

## 핵심 원칙

**한 강의 페이지 = 4섹션 고정 구조.** 학습 모드에 따라 보는 섹션이 다름.

모든 강의 교재 페이지에 항상 4섹션이 있음:
- 🚀 **예습** (강의 듣기 전)
- 📖 **본문** (정독용, 강의 대체 가능)
- 🔄 **복습** (회상용)
- 💡 **일상 비유** (이해·콘텐츠용)

→ 학습 모드별로 페이지를 따로 두지 않음. **같은 페이지를 모드별로 다르게 활용**.

---

## 학습 사이클 4단계

### 1️⃣ 예습 (5분)

- **🚀 섹션만** 훑기
- "이 강의가 뭘 다루는지" 큰 그림
- 핵심 단어 5~7개 머릿속에 미리 깔기

→ 강의 시작 전 두뇌에 빈 폴더 만들기. 강의 흡수율 ↑.

### 2️⃣ 정독 (30~40분)

- **📖 섹션** 천천히 읽기
- 읽으며 마킹:
  - ❓ 헷갈리는 부분
  - ✅ 이해한 부분
  - ⭐ 중요한 부분
  - 💡 영감 떠오른 부분
- **강의 못 들은 사람에게 이 단계가 강의 듣기 대체**
- **짝 강의(이론↔실습) 교차 참조**도 이 단계에서
  - 예: OS 이론 [[lectures/1주차-2일차-01-프로세스|02 프로세스]] ↔ 리눅스 실습 [[lectures/1주차-2일차-07-프로세스와-네트워크|07 프로세스와 네트워크]]
  - 예: OS 이론 [[lectures/1주차-2일차-03-파일-시스템|04 파일 시스템]] ↔ 리눅스 실습 [[lectures/1주차-2일차-05-리눅스-파일-시스템|05 리눅스 파일 시스템]]

### 3️⃣ 회상 (15분)

- **🔄 섹션** 빈칸 자료 안 보고 채우기
- 능동 회상(Active Recall)로 기억 강화
- 막히는 게 정상 → ❓ 표시하고 다음 단계로

### 4️⃣ 막힌 거 정복 (20~30분)

- ❓ 부분만 골라 Claude한테 **일상 비유로** 질문
  - "이 개념을 카카오톡으로 비유해줘"
  - "인스타 추천 알고리즘으로 비유하면?"
  - "맥북 일상으로 풀어줘"
- Claude 답변을 자기 말로 풀어서 **위키 페이지의 💡 섹션에 추가**
- 누적된 ❓는 다음 복습 때 우선 확인

---

## 실습 강의 추가 단계

[강의자료] 시리즈(리눅스 실습·코딩 실습)는 위 4단계에 **터미널에서 손으로 쳐보기** 추가:

- 정독 중 명령어 등장 → 바로 macOS 터미널 또는 Linux 셸에서 실행
- 결과를 페이지 📖 섹션 또는 별도 메모에 캡처
- 안 되는 거 → ❓로 표시 → 4단계에서 Claude한테 디버깅 질문

→ **실습 안 해본 명령어는 기억에 안 남음**. "쳐봐야 안다"가 실습 강의 핵심.

## 페이지 4섹션 구조 상세

자세한 작성 규칙은 [`CLAUDE.md`](../../CLAUDE.md) "강의 페이지 (4섹션 구조)" 섹션 참조.

요약:
- **🚀 예습**: 한 줄 요약 + 핵심 단어 5~7개([[위키링크]]) + 큰 그림
- **📖 본문**: 슬라이드 5~10개씩 묶음, 묶음당 핵심 메시지 한 줄, ⚠️ 오해, ✦ LLM 보강 표시
- **🔄 복습**: 빈칸 3~5개 + ❓ 비워두기
- **💡 일상 비유**: 카톡·인스타·맥북 등 1~2개
