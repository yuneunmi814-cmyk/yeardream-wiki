## 🎯 14개 핵심 명령어 × 5예시 (2026-05-30 퀴즈 정리)

> 너 약점 14개 명령어 — 각 5개 실전 예시. 자주 쓰는 것부터.

### 1. `chmod` (change mode — 권한 변경)

```bash
chmod +x script.sh         # 1. 실행 권한 주기
chmod -w secret.txt        # 2. 쓰기(수정) 권한 빼기
chmod +r data.csv          # 3. 읽기 권한 추가
chmod -rwx 잠긴파일.txt    # 4. 모든 권한 빼기
chmod 755 server.py        # 5. 숫자 모드 (소유자=rwx, 그룹=rx, 기타=rx)
```
→ `+x` 실행, `+w` 수정, `+r` 읽기. 빼려면 `-`.

### 2. `chown` (change owner — 소유자 변경)

```bash
sudo chown yoon file.txt          # 1. 소유자 변경
sudo chown yoon:staff file.txt    # 2. 소유자 + 그룹 (콜론)
sudo chown -R yoon project/       # 3. 폴더 재귀 (-R)
sudo chown :admin file.txt        # 4. 그룹만
sudo chown root important.conf    # 5. root로 (위험)
```
→ 거의 항상 `sudo` 같이.

### 3. `ps` (process status — 프로세스 상태)

```bash
ps                              # 1. 내 프로세스만
ps aux                          # 2. 모든 사용자 (가장 흔함)
ps aux | grep python            # 3. 파이썬 프로세스 찾기
ps aux --sort=-%cpu | head -5   # 4. CPU 많이 먹는 Top 5
ps -u yoon                      # 5. 특정 사용자
```
→ `ps aux` 외워. 거의 매번 이거 + 파이프.

### 4. `top` (top — 실시간 모니터)

```bash
top              # 1. 기본 (q 종료)
top -o cpu       # 2. CPU 순
top -o mem       # 3. 메모리 순
top -U yoon      # 4. 특정 사용자
top -s 2         # 5. 2초마다 갱신
```
→ 실시간 화면. macOS는 `htop`이 더 예쁨.

### 5. `kill` (kill — 프로세스 종료)

```bash
kill 12345              # 1. 정중하게 종료 요청
kill -9 12345           # 2. 강제 종료 ⭐
killall -9 python       # 3. 이름으로 다 죽이기
lsof -i :8080           # 4-1. 포트 사용 PID 찾기
kill -9 12345           # 4-2. 죽이기
ps aux | grep python    # 5-1. PID 찾기
kill -9 12345 67890     # 5-2. 여러 개 한 번에
```
→ `-9` = 강제. 보통 `kill 12345` 먼저, 안 되면 `-9`.

### 6. `grep` (Global Regular Expression Print — 텍스트 검색) ⭐

```bash
grep "error" log.txt        # 1. 파일에서 찾기
grep -r "TODO" .            # 2. 재귀 (하위 폴더 다)
grep -n "import" main.py    # 3. 행 번호
grep -i "python" README.md  # 4. 대소문자 무시
ps aux | grep nginx         # 5. 파이프로 받기
```
→ `grep -rni "검색어" .` 가 만능.

### 7. `find` (find — 파일 검색)

```bash
find . -name "*.py"          # 1. 확장자별 검색
find . -name "main.py"       # 2. 정확한 이름
find . -size +1M             # 3. 1MB 이상
find . -mtime -1             # 4. 24시간 이내 수정
find . -type d -name "test*" # 5. 폴더만
```
→ `find 위치 -name "패턴"`. `.`=현재, `~`=홈, `/`=루트.

### 8. `which` (which — 어느 것?)

```bash
which python      # 1. python 위치
which python3     # 2. python3 따로
which git         # 3. git 위치
which brew        # 4. brew 위치 (mac)
which -a python   # 5. 여러 개 한 번에
```
→ "이 명령어 어디 깔렸어?" 디버깅용.

### 9. `man` (manual — 매뉴얼)

```bash
man grep    # 1.
man ls      # 2.
man chmod   # 3.
man ssh     # 4.
man man     # 5. man의 매뉴얼 (메타)
```
→ 화면 안에서 `q` = 나가기. **요즘은 `--help`가 더 간편**.

### 10. `head` (head — 앞부분)

```bash
head log.txt           # 1. 기본 (앞 10줄)
head -5 log.txt        # 2. 앞 5줄
head -n 100 big.csv    # 3. 앞 100줄
head -3 *.md           # 4. 여러 파일
ps aux | sort -k 3 | head  # 5. 파이프
```
→ 큰 파일 처음 좀만 볼 때. 기본 10줄.

### 11. `tail` (tail — 뒷부분) ⭐

```bash
tail log.txt              # 1. 기본 (뒤 10줄)
tail -20 log.txt          # 2. 뒤 20줄
tail -f /var/log/syslog   # 3. 실시간 추적 (-f = follow) ⭐
tail -f log1.txt log2.txt # 4. 여러 파일 동시
tail -n +50 data.csv      # 5. 50번째 줄부터 끝까지
```
→ **`tail -f` = 서버 로그 실시간 감시. 개발자 1순위**.

### 12. `wc` (word count — 단어·줄·바이트 세기)

```bash
wc file.txt          # 1. 기본 (줄·단어·바이트)
wc -l file.txt       # 2. 줄 수만 (-l = lines) ⭐
wc -w essay.md       # 3. 단어 수만
wc -c image.jpg      # 4. 바이트 수만
ls | wc -l           # 5. 현재 폴더 파일 개수
```
→ `wc -l` 가 가장 자주. `ls | wc -l` = 파일 개수.

### 13. `apt` (Advanced Package Tool — Ubuntu)

```bash
sudo apt update              # 1. 목록 업데이트 (필수)
sudo apt upgrade             # 2. 설치된 거 업그레이드
sudo apt install git         # 3. 설치
sudo apt remove python3      # 4. 삭제
apt search nginx             # 5. 검색
```
→ Ubuntu·Debian. macOS는 `brew`. 항상 `sudo` 같이.

### 14. `brew` (Homebrew — macOS) ⭐ 너 일상

```bash
brew install git              # 1. 설치 (sudo X)
brew install python uv node   # 2. 여러 개 한 번에
brew update && brew upgrade   # 3. 업데이트 + 업그레이드
brew uninstall git            # 4. 삭제
brew search docker            # 5. 검색
```
→ macOS의 `apt`. **sudo 없음**. 너 일상.

## 🤝 자주 조합되는 패턴 5개

```bash
# 1. 포트 8080 쓰는 프로세스 죽이기
lsof -i :8080         # PID 확인
kill -9 [PID]

# 2. 폴더 내 .py 파일 개수
find . -name "*.py" | wc -l

# 3. 로그 마지막 100줄 중 ERROR만
tail -100 log.txt | grep "ERROR"

# 4. 메모리 많이 먹는 Top 5
ps aux --sort=-%mem | head -6

# 5. 명령어 매뉴얼 안에서 검색
man grep
# (안에서 /검색어 입력 + Enter)
```
