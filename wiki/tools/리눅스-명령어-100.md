# 🐧 리눅스 기본 명령어 100선 (우선순위별)

## 한 줄 정의

**실무 빈도·학습 가치 기준으로 큐레이션한 리눅스 명령어 100개**. Tier 1~4 우선순위.

> ⚠️ **출처 주의**: 이 100선은 Claude 큐레이션 (sysadmin 실무 빈도·강의 우선순위 종합). 강사 강의 분류와 100% 일치 X. **다 외울 필요 ❌**. Tier 1 20개 + Tier 2 자주 쓰는 거 위주로 손에 익히면 됨.

## 🚰 파이프·리다이렉션 (명령 ❌ 문법, 알아야 100% 활용)

> 명령어 단독보다 **조합**이 진짜 힘. Tier 2 이상 무조건 알아야.

| 기호 | 의미 | 예시 |
|---|---|---|
| `\|` | 파이프 — 출력을 다음 명령 입력으로 | `ps aux \| grep python` |
| `>` | 덮어쓰기 | `ls > files.txt` |
| `>>` | 이어쓰기 | `echo "x" >> log.txt` |
| `<` | 입력 받기 | `mysql < dump.sql` |
| `2>` | 에러 출력 | `cmd 2> err.log` |
| `2>&1` | 에러 + 출력 합치기 | `cmd > all.log 2>&1` |
| `&&` | 앞 성공 시 실행 | `make && make install` |
| `\|\|` | 앞 실패 시 실행 | `cmd \|\| echo "fail"` |
| `;` | 순차 실행 | `cd ~; ls` |
| `&` | 백그라운드 | `python a.py &` |

→ [[1주차-2일차-06-리눅스-명령어]] 페이지에 상세.

## 🎯 은미 학습 우선순위 (큐레이션)

### Phase 1 — 이번 주말 ([[할-일]] 본인 결정 시)
**Tier 1 (20개)** 손코딩. Colab·로컬 터미널에서 하나씩 쳐보기.

### Phase 2 — Stage 2 진입 (6월~)
**Tier 2 (30개)** + 파이프 활용. `grep · find · xargs · tail -f` 매일.

### Phase 3 — 서비스개발자 트랙 가면
**Tier 3 (30개)** 네트워크·SSH·crontab 필수. [[미션-2-크롤링-자동화]] 도전 시 cron 필수.

### Phase 4 — Stage 4 / 졸업 후
**Tier 4 (20개)** 보안·서비스·디버깅. 실무 들어가면 자연스럽게.

## 🙋 외울 필요 없는 것

| 외워야 함 | 외울 필요 X (찾아서 쓰면 됨) |
|---|---|
| `cd · ls · pwd · cat · grep` | 옵션 디테일 (`-rn` `-l` `-i` 등) |
| 파이프 `\|` | sed·awk 복잡한 정규식 |
| `chmod 755` 기본 패턴 | `iptables` 규칙 문법 |
| sudo·apt 흐름 | strace 출력 해석 |

→ 강사 메시지 "프로그래밍 = 참조" 그대로 ([[Python]] 1-2 섹션).

## 💡 Mac 사용자 차이 (너 환경)

| Linux | macOS 차이 |
|---|---|
| `apt` (Ubuntu) | `brew` (Homebrew) |
| `ls --color` | macOS는 `ls -G` 또는 `alias` 필요 |
| `ifconfig` 비추 | macOS는 여전히 `ifconfig` OK |
| `sed -i 'expr'` | macOS는 `sed -i '' 'expr'` (빈 따옴표) |
| `journalctl` | macOS는 `log show` 또는 Console.app |

→ 너 일상 = macOS. **`brew · ls -la · cd · grep` 매일**.
