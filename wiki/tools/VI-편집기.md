# VI / Vim — 편집기

## 한 줄 정의

**터미널에서 작동하는 모달(modal) 텍스트 편집기**. 1976년 등장. Linux·macOS·Unix 어디서나 기본 설치되어 있어 **서버 작업의 표준**. 6개월 점진적 마스터 목표.

## 왜 배워야 하나 (사용자 케이스)

- **서버 접속 시 nano 없을 수 있음** — VI는 거의 100% 설치되어 있음
- **빠른 편집** — 키보드만으로 마우스보다 빠름
- **모든 개발자가 아는 기본기** — 클라이언트·동료와 협업 시 공통 도구
- **AI 시대에도 안 사라짐** — Claude Code·SSH 작업에서 매일 마주침

→ 강의 본 자료엔 nano만 있지만, 강의자 가이드대로 6개월 동안 점진적으로.

## 핵심 개념: 모드 4종

VI의 가장 큰 진입 장벽 = **모드(mode)**. 다른 에디터엔 없는 개념.

| 모드 | 역할 | 들어가는 키 |
|------|------|------------|
| **Normal** | 기본. 이동·삭제·복사 | `Esc` (다른 모드에서 빠져나옴) |
| **Insert** | 글자 입력 | `i`, `a`, `o` |
| **Visual** | 영역 선택 | `v`, `V`, `Ctrl+V` |
| **Command** | 저장·검색·종료 | `:` |

**핵심 흐름**:
```
Normal → i → Insert → 글자 입력 → Esc → Normal → : → Command → :wq
```

⚠️ **흔한 함정**: 처음 vi 열면 **Normal 모드**. 바로 글자 입력하면 안 됨. `i` 누르고 Insert 모드 들어가야 글자 쳐짐.

## 1단계 (1개월) — 최소 생존 명령어

이거만 알아도 일단 파일 편집·저장 가능.

| 명령 | 동작 |
|------|------|
| `vi 파일명` | VI 열기 |
| `i` | Insert 모드 (글자 입력 시작) |
| `Esc` | Normal 모드로 (편집 끝) |
| `:w` | 저장 (write) |
| `:q` | 종료 (quit) |
| `:wq` | 저장 + 종료 |
| `:q!` | 저장 안 하고 강제 종료 |

**실습 예시**:
```bash
vi test.txt          # 열기
# (Normal 모드에서) i 누름
hello world          # 글자 입력
# (Esc 눌러서 Normal 모드)
:wq                  # 저장 + 종료
```

→ 1개월 동안 이것만 익혀도 충분. nano 대신 VI로 가벼운 편집 시도.

## 2단계 (2~3개월) — 이동 + 기본 편집

### 이동 (Normal 모드)

| 명령 | 동작 |
|------|------|
| `h` `j` `k` `l` | ← ↓ ↑ → (화살표 대용) |
| `0` | 줄 시작 |
| `$` | 줄 끝 |
| `w` | 다음 단어 |
| `b` | 이전 단어 |
| `gg` | 파일 맨 위 |
| `G` | 파일 맨 아래 |
| `숫자G` | 숫자번째 줄로 (예: `42G` = 42번째 줄) |

→ **`hjkl` 대신 화살표 키도 작동**. 단 hjkl이 빠름 (손이 키보드 중앙에서 안 벗어남).

### 편집

| 명령 | 동작 |
|------|------|
| `x` | 현재 글자 삭제 |
| `dd` | 현재 줄 삭제 |
| `yy` | 현재 줄 복사 (yank) |
| `p` | 붙여넣기 (paste) |
| `u` | 실행 취소 (undo) |
| `Ctrl+r` | 다시 실행 (redo) |

### Insert 모드 진입 변형

| 명령 | 동작 |
|------|------|
| `i` | 커서 자리에 입력 |
| `a` | 커서 다음 자리에 입력 |
| `o` | 아래 새 줄 만들고 입력 |
| `O` | 위 새 줄 만들고 입력 |

## 3단계 (4~5개월) — 검색·치환·범위

### 검색

```vim
/패턴       # 앞으로 검색
?패턴       # 뒤로 검색
n           # 다음 결과
N           # 이전 결과
```

### 치환 (Command 모드)

```vim
:s/old/new          # 현재 줄 첫 매치 치환
:s/old/new/g        # 현재 줄 전체 치환
:%s/old/new/g       # 파일 전체 치환
:%s/old/new/gc      # 전체 치환, 매번 확인
```

→ 사용자가 위키에서 단어 일괄 변경할 때 유용.

### 줄 범위 명령

```vim
:5,10d              # 5~10번째 줄 삭제
:5,10y              # 5~10번째 줄 복사
:1,$s/old/new/g     # 1~끝 줄 치환 (= :%s/...)
```

## 4단계 (6개월+) — 효율화

### 단축어 (`.vimrc` 설정)

`~/.vimrc` 파일에 자주 쓰는 단축어 등록:

```vim
set number          " 줄 번호 표시
set tabstop=4       " 탭 = 4칸
set autoindent      " 자동 들여쓰기
syntax on           " 문법 강조
```

→ 사용자 맥북에서 `vi ~/.vimrc` 또는 `vim ~/.vimrc`로 설정.

### 매크로 / 마크 / 멀티 윈도우

(고급 — 6개월차 이후 점진적)
- `q문자` — 매크로 녹화
- `m문자` — 마크 설정
- `:split`·`:vsplit` — 여러 창 분할

## ⚠️ 흔한 함정

### 1. vi에서 못 나오겠다 😱

가장 흔한 초보 경험. 해결:
```
Esc Esc Esc        (확실히 Normal 모드로)
:q!                (저장 안 하고 강제 종료)
```

**`Esc + :q!` = 비상 탈출 주문**.

### 2. 글자가 안 쳐진다

Normal 모드. `i` 누르면 Insert 모드 → 그 다음 글자 입력.

### 3. 한글 입력 안 된다

기본 vi는 영문 위주. 한글 가능하지만 인코딩 이슈 종종. `vim`(VI의 개선판)이 더 안정.

→ macOS 기본은 `vi`라 치면 `vim` 실행됨 (alias).

## VI vs Vim vs nano

| | nano | vi | vim |
|--|------|------|------|
| 등장 | 1999 | 1976 | 1991 |
| 학습 곡선 | 매우 쉬움 | 가파름 | 가파름 |
| 단축키 화면 표시 | O | X | X |
| 기능 | 기본만 | 강력 | 매우 강력 |
| 설치 | 일부 서버엔 없음 | 거의 100% | 거의 100% |

→ **nano**로 시작 → **vi/vim**으로 마스터가 사용자 6개월 계획.
