## 핵심 정리

### 챕터 1. 현재 위치 확인

```bash
$ pwd                # 절대 경로 출력
$ cd ~/Desktop       # 디렉터리 이동
$ ls                 # 파일·폴더 목록
$ ls -al             # 숨김 파일 포함, 상세 정보
```

→ 강의 06강 챕터 1·2와 매핑. 강의에선 절대 경로 vs 상대 경로 비교가 더 깊음.

### 챕터 2. 폴더 만들기 및 이동

```bash
$ mkdir linux_practice                    # 폴더 생성
$ mkdir -p linux_practice/src/utils       # 중첩 폴더 한번에
$ cd linux_practice
$ cd ..                                   # 상위 폴더로
```

→ `-p` 옵션은 [[1주차-2일차-라이브-노트]]의 "폴더 만들 때도 리눅스 활용해서 연습" 메모와 직결.

### 챕터 3. 파일 만들기 및 vi 편집기

#### 파일 생성

```bash
$ touch hello.txt
$ echo "Hello, Linux!" > hello.txt        # 덮어쓰기
$ echo "두 번째 줄 추가" >> hello.txt      # 이어쓰기
$ cat hello.txt                           # 내용 확인
```

⚠️ **자료의 가이드**: "가급적이면 `>`는 사용 금지" — 실수로 덮어쓰는 사고 방지. [[1주차-2일차-라이브-노트]] 4번 항목과 동일 메시지.

#### vi 편집기 (자료 발췌)

> "vi 편집기: **개발자 필수 / 분석가 옵션 / 기획자는 공부하지 마세요!**"
> "개발자의 경우: 앞으로 markdown 작업할 때, vi 편집기로 하는 것 추천"

| 키 | 동작 |
|---|---|
| `i` | 삽입 모드 진입 |
| `ESC` | 일반 모드 복귀 |
| `:w` | 저장 |
| `:q` | 종료 |
| `:wq` | 저장 후 종료 |
| `:q!` | 저장 없이 강제 종료 |

→ 사용자 학습 노선 판단 근거: 사용자는 AI 분석가·서비스개발자 트랙 미정 상태. **분석가 = 옵션**이지만 사용자는 마크다운 작업·서버 작업이 많아 VI 6개월 마스터 노선 유지가 합리적. ([[VI-편집기]] 참조)

### 챕터 5. 복사·이동·삭제

```bash
$ cp hello.txt hello_backup.txt           # 파일 복사
$ cp -r linux_practice/ backup/           # 폴더 전체 복사
$ mv hello_backup.txt hello_copy.txt      # 이름 변경
$ mv hello_copy.txt archive/              # 이동
$ rm 파일명.txt                           # 파일 삭제
$ rm -r 폴더명                            # 폴더 삭제
$ rm -rf 폴더명                           # 강제 삭제 ⚠️
```

⚠️ **`rm -rf` 주의**: 확인 없이 즉시 삭제. 잘못된 경로 입력 시 복구 불가. 실무에서 `sudo rm -rf /` 사고가 종종 발생 → 절대 경로 / 와 함께 쓰지 말 것.
