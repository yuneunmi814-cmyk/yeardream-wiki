# 🥇 Tier 1 — 필수 150개 (빈도 240회 이상)

> 책에서 **240번 이상 등장**. 모르면 매 페이지에서 막힘.

## A. 운영체제 기본 (9개)

### 1. **system** — 시스템 (6,073회) 🏆 1위

> 책 제목의 핵심 단어. 모든 챕터에 등장.

**개념**: "여러 부품이 함께 작동하는 하나의 큰 묶음".

**비유**: 학교 시스템 = 교사 + 학생 + 교실 + 시간표 다 합친 것.
컴퓨터 시스템 = CPU + 메모리 + 디스크 + OS 다 합친 것.

```
[CPU] + [메모리] + [디스크] + [OS] = 컴퓨터 시스템
```

---

### 9. **kernel** — 커널 (1,398회)

**개념**: OS의 **심장**. 컴퓨터 자원(CPU·메모리·디스크)을 직접 다루는 부분.

**비유**: 식당의 **주방장**. 손님은 못 만남, 음식 만드는 데 집중.

```
사용자 (너) → Shell (홀 매니저) → Kernel (주방장) → 하드웨어
```

→ 너 위키 [[운영체제]] "쉘 vs 커널 식당 비유" 참조.

---

### 8. **user** — 사용자 (1,473회)

**개념**: 컴퓨터를 쓰는 사람. 또는 그 사람의 계정.

**비유**: 도서관 회원증. `whoami` 명령어로 "나 누구?" 확인.

---

### 10. **operating** — 운영하는 (1,389회)

**개념**: "운영하다"의 진행형. **operating system** 합쳐서 "운영체제".

---

### 29. **computer** — 컴퓨터 (736회)

**개념**: 계산·저장·통신하는 기계. CPU + 메모리 + 입출력으로 구성.

---

### 45. **hardware** — 하드웨어 (585회)

**개념**: 컴퓨터의 **만질 수 있는** 부품. CPU·RAM·키보드·디스크.

---

### 101. **software** — 소프트웨어 (338회)

**개념**: 컴퓨터의 **만질 수 없는** 프로그램·앱. 하드웨어와 반대.

```
하드웨어 = 몸     소프트웨어 = 영혼
```

---

### 73. **machine** — 기계 (444회)

**개념**: 컴퓨터를 부르는 또 다른 말. **virtual machine** (가상 컴퓨터)에 자주.

---

### 148. **architecture** — 아키텍처·구조 (243회)

**개념**: 컴퓨터·소프트웨어의 **전체 구조 설계**.

**비유**: 건축에서 건물의 도면. "이 컴퓨터는 어떻게 설계됐어?".

---

## B. 프로세스·스레드 (12개)

### 3. **process** — 프로세스 (2,432회) 🏆 3위 ⭐⭐⭐

**개념**: **실행 중인 프로그램**. 디스크의 프로그램이 메모리에 올라가서 돌아가는 상태.

**비유**: 요리책(프로그램) → 실제로 요리하는 중(프로세스).

```
디스크에 잠자는 .py 파일  →  실행하면  →  메모리에서 살아 움직임 (process)
```

→ `ps` 명령으로 실행 중인 프로세스 다 볼 수 있음.

---

### 5. **thread** — 스레드 (1,957회) ⭐⭐

**개념**: 프로세스 **안의 더 작은 실행 단위**. 한 프로세스 안에 여러 스레드.

**비유**: 식당(프로세스) 안의 직원들(스레드). 한 식당에서 여러 일 동시에.

```
크롬 (프로세스 1개)
  ├ 탭 1 그리기 (스레드)
  ├ 다운로드 (스레드)
  └ 음악 재생 (스레드)
```

---

### 17. **program** — 프로그램 (1,020회)

**개념**: 디스크에 저장된 명령어 묶음. 실행하면 **process**가 됨.

```
program (잠자는 책)  →  실행  →  process (살아 움직임)
```

---

### 27. **application** — 응용 프로그램·앱 (756회)

**개념**: 사용자가 직접 쓰는 프로그램. Chrome·카톡·VSCode 등.

---

### 36. **scheduling** — 스케줄링 (676회) ⭐

**개념**: "어떤 프로세스 다음에 CPU 줄까?" **순서를 정하는 것**.

**비유**: 학교 시간표 짜기. 1교시 누가? 2교시 누가?

```
프로세스 A·B·C 다 CPU 쓰고 싶어함
  ↓
스케줄러: "A → C → B 순서로 줘"
```

---

### 46. **task** — 작업 (581회)

**개념**: 컴퓨터가 처리할 일. 프로세스·스레드의 또 다른 말.

---

### 70. **priority** — 우선순위 (462회)

**개념**: "누가 먼저야?" 결정하는 숫자. 높을수록 먼저 처리.

**비유**: 응급실. 심각한 환자(높은 우선순위) 먼저.

---

### 74. **queue** — 큐·대기열 (440회)

**개념**: **줄 서기**. 먼저 온 게 먼저 나감 (FIFO — First In First Out).

**비유**: 매표소 줄. 먼저 줄 선 사람이 먼저 표 받음.

```
[A][B][C][D] ← 줄 끝
앞쪽 A부터 처리됨
```

---

### 99. **running** — 실행 중 (349회)

**개념**: 프로세스의 4가지 상태 중 하나. 지금 CPU 쓰고 있는 상태.

```
프로세스 상태: ready (준비) → running (실행) → waiting (대기) → terminated (종료)
```

---

### 121. **wait** / 126. **waiting** — 기다림 (287회·279회)

**개념**: 프로세스가 **뭔가를 기다리는 상태**. 디스크 읽기·사용자 입력 대기 등.

---

### 138. **deadlock** — 교착 상태 (254회) ⭐

**개념**: 두 프로세스가 **서로 기다리며 둘 다 멈춤**. 영원히.

**비유**: 좁은 골목에서 마주 본 두 자동차. 둘 다 비켜주지 않으면 영원히 못 움직임.

```
프로세스 A: B가 가진 자원 필요 → 기다림
프로세스 B: A가 가진 자원 필요 → 기다림
→ 둘 다 영원히 멈춤 (deadlock)
```

---

### 142. **mach** — Mach (251회)

**개념**: macOS 커널의 기반이 된 운영체제 (1985 Carnegie Mellon 대학).

→ 너 macOS = Mach 후손. 책에서 자주.

---

## C. 메모리·페이지 (11개)

### 4. **memory** — 메모리 (2,224회) 🏆 4위 ⭐⭐⭐

**개념**: 프로그램이 실행되는 **임시 작업 공간**. RAM. 전원 끄면 사라짐.

**비유**: 책상. 책꽂이(디스크)에서 책 꺼내 책상(메모리) 위에 놓고 작업.

```
디스크 = 책꽂이 (영구 저장, 느림)
메모리 = 책상 (작업 중, 빠름, 전원 OFF면 다 사라짐)
```

---

### 6. **page** — 페이지 (1,789회) ⭐⭐

**개념**: 메모리를 **조각낸 한 단위**. 보통 4KB.

**비유**: 큰 책을 페이지 단위로 나눔. 필요한 페이지만 메모리에 올림.

```
프로그램 (전체)
  ├ Page 1 ─ 메모리 ✅
  ├ Page 2 ─ 디스크 (필요할 때 가져옴)
  ├ Page 3 ─ 메모리 ✅
  └ Page 4 ─ 디스크
```

---

### 30. **address** — 주소 (713회) ⭐

**개념**: 메모리에서 **데이터가 있는 위치**. 집 주소처럼.

**비유**: 메모리 = 아파트. 주소 = 102동 305호.

---

### 28. **virtual** — 가상의 (737회) ⭐

**개념**: "진짜는 아니지만 진짜처럼 보이는". 컴퓨터가 만들어낸 환상.

**예**: **virtual memory** = 가상 메모리. RAM이 1GB여도 8GB처럼 쓸 수 있음 (디스크 일부를 메모리처럼).

---

### 71. **physical** — 물리적인 (460회)

**개념**: 진짜 하드웨어 상의. **virtual**의 반대.

- physical memory = 진짜 RAM
- virtual memory = 가짜(추상화된) 메모리

---

### 91. **cache** — 캐시 (376회) ⭐

**개념**: **자주 쓰는 데이터를 빠른 곳에 임시 저장**.

**비유**: 책상 위에 자주 보는 책 올려두기. 책꽂이 매번 다녀오기 귀찮으니.

```
CPU 캐시 (가장 빠름)  →  RAM (보통)  →  디스크 (느림)
```

→ 너 위키 [[캐시]] 페이지 참조.

---

### 98. **buffer** — 버퍼 (355회)

**개념**: 데이터를 **잠깐 모아두는 공간**. 한 번에 처리하려고.

**비유**: 우체통. 편지 한 통씩 보내는 게 아니라 모아서 한 번에.

---

### 97. **frame** — 프레임 (359회)

**개념**: 물리 메모리의 **page와 같은 크기 조각**. 페이지가 들어가는 자리.

```
page (가상)  →  복사  →  frame (물리)
```

---

### 133. **allocation** — 할당 (265회) ⭐

**개념**: 자원을 **나눠줌**. "이 메모리는 너 거" 결정.

**예**: 메모리 할당 = 프로세스에게 메모리 일부 떼어줌.

---

### 52. **space** — 공간 (550회)

**개념**: 메모리·디스크의 **빈 자리**. 주소 공간(address space)에 자주.

---

### 89. **shared** — 공유된 (378회)

**개념**: 여러 프로세스가 **같이 쓰는**. 공유 메모리·공유 자원.

---

## D. 파일·디렉토리·저장 (12개)

### 2. **file** — 파일 (2,874회) 🏆 2위 ⭐⭐⭐

**개념**: 디스크에 저장된 데이터 묶음. `.py`·`.md`·`.jpg` 등.

**비유**: 종이 한 장. 디렉토리(폴더) 안에 정리.

---

### 55. **directory** — 디렉토리·폴더 (534회) ⭐

**개념**: 파일들을 묶어두는 **폴더**. 그 안에 또 폴더 가능.

```
Documents/                   ← 디렉토리
  ├ yeardream-wiki/         ← 디렉토리
  │   ├ Python.md          ← 파일
  │   └ 변수.md             ← 파일
  └ photos/                ← 디렉토리
      └ cat.jpg            ← 파일
```

---

### 19. **block** — 블록 (960회) ⭐

**개념**: 디스크의 **읽기·쓰기 최소 단위**. 보통 4KB.

**비유**: 책의 페이지. 한 글자만 읽고 싶어도 한 페이지 통째 읽어야.

---

### 39. **disk** — 디스크 (640회)

**개념**: **영구 저장 장치**. HDD·SSD.

```
RAM (전원 OFF = 사라짐)  ↔  Disk (전원 OFF = 유지)
```

---

### 40. **storage** — 저장장치 (624회)

**개념**: **데이터를 저장하는 장치 전체**. 디스크·SSD·USB·클라우드 다 포함.

---

### 7. **data** — 데이터 (1,498회)

**개념**: 컴퓨터가 처리하는 **모든 정보**. 숫자·글자·이미지·소리 다.

---

### 21. **object** — 객체 (887회) ⭐

**개념**: 프로그래밍에서 **데이터 + 행동의 묶음**. class의 인스턴스.

**비유**: 붕어빵 = object. 붕어빵 틀 = class.

→ 너 위키 Day 4 메모 "class·메서드 = 붕어빵 틀" 참조.

---

### 50. **table** — 테이블·표 (554회)

**개념**: 행과 열로 정리된 데이터. **page table**·**file allocation table** 자주.

---

### 90. **entry** — 항목·기재 (377회)

**개념**: 표·리스트의 한 줄. 디렉토리 entry = 폴더 안 파일 하나.

---

### 75. **driver** — 드라이버 (428회)

**개념**: **하드웨어 장치를 제어하는 소프트웨어**. 프린터 드라이버·그래픽카드 드라이버.

---

### 11. **device** — 장치 (1,258회) ⭐

**개념**: 컴퓨터에 연결된 **모든 하드웨어**. 키보드·마우스·디스크·프린터.

---

### 144. **copy** — 복사 (248회)

**개념**: 파일·데이터를 **그대로 다른 곳에**. `cp` 명령.

---

## E. 동기화·통신 (8개)

### 66. **lock** — 잠금 (487회) ⭐

**개념**: **다른 프로세스가 못 쓰게 잠그기**. 동시 접근 막기.

**비유**: 화장실 문 잠금. 한 명이 들어가면 다른 사람 못 들어옴.

---

### 105. **mutex** — 뮤텍스 (329회) ⭐

**개념**: **mut**ual **ex**clusion = "상호 배제". lock의 일종.

→ 한 명만 쓰게 해주는 자물쇠.

---

### 141. **semaphore** — 세마포어 (252회)

**개념**: **여러 명까지 허용하는 신호등**. mutex의 확장.

**비유**: 주차장 입구 "현재 5/10대 주차 중". 차수 카운트.

---

### 56. **message** — 메시지 (533회)

**개념**: 프로세스끼리 주고받는 **데이터 패킷**.

---

### 153. **signal** — 시그널·신호 (240회)

**개념**: 프로세스에게 보내는 **짧은 알림**. `kill -9 PID`도 시그널 보내는 거.

---

### 72. **interrupt** — 인터럽트 (455회)

**개념**: **하던 일 중단시키는 신호**. 키보드 눌림·디스크 완료 등 외부 사건.

**비유**: 일하는 중 전화 옴. "잠깐 멈추고 받아".

---

**개념**: **8개 bit를 묶은 단위**. 한 글자(영문) 저장 가능.

```
1 byte = 8 bit
1 KB (킬로바이트) = 1,024 byte
1 MB (메가바이트) = 1,024 KB
1 GB (기가바이트) = 1,024 MB
```
