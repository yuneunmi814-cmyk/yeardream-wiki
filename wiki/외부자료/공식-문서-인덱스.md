## 4단계 공부법 (강사 권고)

```
1. 튜토리얼 ("Getting Started")
       ↓ 따라 치기만 해도 절반
2. 레퍼런스 (모르는 거 사전식 검색)
       ↓ AI 묻기 전 공식 먼저 ⭐
3. 가이드 ("Best Practices"·"How-to")
       ↓ 실무 패턴
4. 깊이 (GitHub 소스·이슈·PR)
       ↓ 컨트리뷰터·기여
```

**비유**:
- AI = 통역사 (편함, 100% 정확 X)
- 공식 문서 = 원어민 직접 강의 (시간 들지만 정확)

## 🔍 검색 키워드 팁

### Python 공식 검색 패턴

```
docs.python.org → 검색창에:
- "list comprehension" → 컴프리헨션 깊이
- "type hints"        → 타입 힌트
- "asyncio"           → 비동기
- "decorator"         → 데코레이터
```

### Claude API 검색 패턴

```
docs.anthropic.com → 검색창에:
- "messages api"      → 기본 호출
- "tool use"          → 도구 호출
- "prompt caching"    → 캐싱
- "vision"            → 이미지 분석
```

### Stack Overflow 활용

- 공식에 없으면 → stackoverflow.com 검색
- 영문 검색 필수 (한국어 검색은 답 적음)
- "[python] error message" 형식으로 태그 활용

### GitHub Issues 검색

- 라이브러리 버그·이슈 → `github.com/저장소/issues`
- "is:issue is:closed" 닫힌 거 = 해결 사례

## ⚠️ 주의

### 영어 부담 줄이는 법

- **한국어 공식이 있으면 무조건 한국어** (Python·React·Vue·Git·GitHub·MDN·GCP·Kubernetes 등)
- 영어만 있는 경우 → **Claude/Gemini로 번역** 또는 Chrome 자동 번역
- **API 레퍼런스**는 영어가 더 빠름 (코드 위주)

### 옛 버전 함정

- AI는 학습 데이터 기준 옛 버전 답할 수 있음
- 공식 = 항상 최신 (URL 그대로 가면 최신 버전)
- 라이브러리 메이저 버전 변경 시 = AI 답 ❌ 가능 → 공식 필수
