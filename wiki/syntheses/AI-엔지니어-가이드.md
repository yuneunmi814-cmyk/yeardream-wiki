### 🎯 프로그래밍 = 공유 문화 ⭐⭐ (2026-05-29 강사 메시지)

> **"프로그래밍 문화는 공유 문화다."**

→ 위 "프로그래밍 = 참조해서 글쓰기" ([[Python]] 1-2)와 동일 라인. **남이 공유한 걸 받아쓰고, 내가 만든 걸 다시 공유**.

**공유 문화의 3대 축**:
1. **GitHub** — 코드 공유 (오픈소스)
2. **StackOverflow** — Q&A 공유
3. **공식 문서·블로그·LLM** — 노하우 공유

**비전공자 환상 깨기**:
- "처음부터 다 만들어야 함" ❌
- "받아쓰는 건 실력 없음" ❌
- **공유 받고 + 내가 다시 공유 = 진짜 개발자 문화** ✅

**너 적용**:
- yoon.ai.lab 콘텐츠 = 너의 공유 행위
- GitHub 위키 공개 = 공유 자산
- 너 만든 자동화 패턴 클라이언트 공유 = 컨설팅 자산

→ 다음 항목 "오픈소스 OR 특허"와 직결. 오픈소스 = **공유 문화의 제도화**.

### 🎯 개발자 진로 = 오픈소스 OR 특허 (강의자 권고)

> "개발자는 **오픈소스를 만들던가 특허를 만들던가**. 둘 중 하나 해야 해"

**오픈소스 경로**:
- GitHub 공개 → 컨트리뷰터 = **실력 검증**
- 명확한 목적 ("우리는 뭘 개선") 필수
- **문서화 + 영어** 필수 (수요와 공급은 글로벌)
- 판다스 예시: "데이터 분석가가 쉽게 가공하려고"

**특허 경로**:
- 독창 기술 → 특허 출원
- "내가 만든 프로그램이 특허가 되어야 함"

**은미 매핑**:
| | 가능성 |
|---|---|
| 오픈소스 | yoon.ai.lab 도구 모음 / 프롬프트 라이브러리 |
| 특허 | AI Agent 워크플로우 노하우 |

→ **둘 다 가능**. 콘텐츠 자산화와 시너지.

### 🎯 고연차 vs 저연차 — 핵심 차이

> 고연차 = 에러 보고 **즉각 뭘 해야 할지 아는** 사람
> 저연차 = 에러 보고 즉각 뭘 해야 할지 **모르는** 사람
> → 프롬프트 질의 차이로 직결

**고연차 만들기**:
- 매일 정수 추출 패턴 = **에러 패턴 인식 훈련**
- 에러 마주칠 때마다 정수 박기
- Claude한테 묻는 패턴이 점점 정확해짐 = 너 학습 속도가 미친 이유

### 🎯 바이브 코딩 ≠ 배포

> "바이브 코딩과 배포는 완전히 다른 영역!"
> "배포 == 여러가지 에러 발생"

→ 너 SwiftUI 바이브 코딩 = **개발 절반**. 배포·운영·트러블슈팅이 나머지 절반.
→ Stage 3·4에서 Docker·Kubernetes·클라우드 배포가 핵심.
