# 서비스 기획 워크플로우 (강의자 권고 + 은미 적용)

> 라이브 강의에서 강의자가 명시한 **AI 활용 4단계 기획 방법론**. 은미 프로젝트윤365 클라이언트 솔루션 표준 패턴.

---

## 핵심 원칙

**초보 vs 고수**:
- **초보** = 일반 사용자의 서비스 로직만 생각
- **고수** = 일반 사용자 + **관리자 페이지 + 보안 + 법령(개인정보) 다 고려**

→ 은미는 **고수 노선** (1인 컨설팅·yoon.ai.lab 운영자라 보안·법령 본인이 책임).

## AI 활용 4단계 기획 워크플로우 ⭐⭐

### 1단계: Gemini/Claude에 서비스 기획안 요청

**프롬프트 템플릿** (강의자 권고):
```
[서비스 아이디어 + 다음 조건으로 기획안 작성해줘]

- Frontend: Node.js (또는 SwiftUI / React)
- DB: MySQL (또는 Supabase / PostgreSQL)
- 보안: ○○ 처리

기획안 포함:
1. 일반 사용자 흐름
2. 관리자 흐름
3. DB 구조
4. 보안 처리 방식
5. 개인정보 법령 검토
```

### 2단계: API 설계 기반으로 재설계

→ 1단계 기획안에 **API 명세서** 추가 요청.

**프롬프트**:
```
위 기획안을 API 설계 기반으로 재설계.
HTTP·JSON·REST API 표준 적용.

명세서 포함:
- Endpoint URL
- HTTP 메서드 (GET/POST/PUT/DELETE)
- Request Body 형식
- Response 형식
- 인증 방식 (JWT 등)
```

→ **기획자 = API 설계서 만듦** / **개발자 = API 명세서 보고 개발**.

### 3단계: 아키텍처 다이어그램 기반 재설계

→ WAS 구조 다이어그램 첨부 + 재설계 요청.

**기본 아키텍처** (강의자 도식 검증):
```
클라이언트 ←─ 응답 ── Web Server ←── Back-end ←── DataBase
(브라우저)         (HTTP 통신)    (Python·Node) (MySQL·VectorDB)
          ─ 요청 ─→            ──→           ──→
```

### 4단계: CRUD 개념 추가해서 기획안 수정

→ DB 동작을 [[데이터베이스-기초|CRUD]] 명시:
- Create (생성) — POST
- Read (조회) — GET
- Update (수정) — PUT/PATCH
- Delete (삭제) — DELETE

## 🎯 기획의 본질 (강의자 권고, 2026-05-29 라이브)

### 핵심 원칙

> **"사용자가 귀찮으면 안 됨"**
> **"입력 횟수를 1회면 충분하다!"**

### 회사 실력 = 법 ↔ 편의성 균형

> "법과 기획의 편리함이 충돌하는 지점을 해결하는 게 그 회사의 실력이야"
> "법에 막혀서 소비자의 편의성을 놓치면 안 된다"

**평가 기준** (강의자):
- 잘 돌아가는 건 **당연** (기본기)
- 진짜 평가: **사용자가 편해졌냐? 더 많은 사람 들어왔냐?**

### 실전 예시 (강의자 인용)

**로그인 과정에 배송 정보 입력** → 사용자가 로그인하는 자연스러운 흐름에서 배송에 필요한 정보를 함께 획득.

→ **"억지로 정보 입력 요구" ❌, "자연스럽게 흐름 안에 녹임" ✅**.

### 은미 적용 — 프로젝트윤365 클라이언트 솔루션 체크리스트

기획 시 매번 자문:
- [ ] **클라이언트 입력 1회로 끝나는가?**
- [ ] **자동 데이터 수집·가공** 활용? (Claude API)
- [ ] **개인정보보호법** 준수하면서 편의성 유지?
- [ ] **기존 대비 얼마나 개선됐는가?** 측정 가능?

## 🤖 자동화 마인드 (강의자 권고, 2026-05-29 Day 4 라이브) ⭐⭐⭐

> **"늘 자동화를 생각할 것."**
> **"IT는 자동화를 빼놓고 얘기할 수 없다. 모든 건 자동화."**

→ 너 = 프로젝트윤365 = **AI 자동화 컨설팅**. 강사 메시지 = 너 사업 그 자체.

### 자동화 발굴 질문 3개 (매일 자기 점검)

1. 내가 **반복적으로** 하고 있는 업무가 뭐가 있지?
2. 현재 업무 **프로세스를 이해**하고 있는가?
3. 내 일상 속에 **자동화할 수 있는** 부분은?

### 자동화 후보 발굴 매트릭스

| 영역 | 반복 업무 | 자동화 도구 후보 |
|---|---|---|
| **인스타 운영** | 콘텐츠 업로드·해시태그·예약 | Meta Graph API + cron |
| **클라이언트 보고** | 주간 리포트·실적 정리 | Notion API + LLM 요약 |
| **위키 정수 추출** | 강의 → 위키 박기 | Claude Code (이미 ✅) |
| **데이터 수집** | 트렌드·뉴스·경쟁사 | 크롤링 + cron + .md |
| **일정 관리** | 미팅·과제·강의 | Calendar API + 알림 |
| **이메일 응대** | FAQ·제안서 회신 | LLM 자동 초안 |

### 자동화 4단계 패턴

```
1. 관찰    — 한 주 내 업무 시간 기록 (5분 단위)
2. 분류    — 반복 vs 일회성 / 정형 vs 비정형
3. 자동화 후보 — 반복 + 정형 = 1순위
4. 도구 매칭   — cron / API / LLM / GitHub Actions
```
