# HTTP 상태코드 (Status Code)

## 한 줄 정의

**서버가 클라이언트 요청 결과를 알리는 3자리 숫자**. 첫 자리로 5종 분류.

## 핵심 아이디어

| 첫 자리 | 분류 | 의미 |
|---|---|---|
| **1xx** | 정보 | 받았고 처리 중 |
| **2xx** | 성공 | 정상 처리됨 |
| **3xx** | 리다이렉트 | 다른 데로 가라 |
| **4xx** | 클라이언트 오류 | 네가 잘못함 |
| **5xx** | 서버 오류 | 내가 잘못함 |

### 자주 보는 코드

**2xx 성공**:
- **200 OK** — 모든 정상 응답
- **201 Created** — 새 자원 생성 (POST 후)
- **204 No Content** — 성공했지만 응답 본문 없음 (DELETE 후)
- **206 Partial Content** — 일부 응답 (대용량 부분 다운로드)

**3xx 리다이렉트**:
- **301 Moved Permanently** — 영구 이동. 검색엔진 색인 갱신
- **302 Found** — 일시 이동. 검색엔진 색인 유지
- **304 Not Modified** — 캐시 그대로. 변경 없음
- **307/308** — 메서드 보존 버전

**4xx 클라이언트 오류** ⭐ 가장 자주 봄:

| 코드 | 이름 | 뜻 |
|---|---|---|
| **400** | Bad Request | 문법 오류 (JSON 깨짐 등) |
| **401** | Unauthorized | 인증 필요 (로그인 안 됨) |
| **403** | Forbidden | 권한 없음 (로그인은 됨) |
| **404** | Not Found | 페이지 없음 |
| **405** | Method Not Allowed | 메서드 불일치 |
| **409** | Conflict | 충돌 (동시 수정) |
| **413** | Payload Too Large | 요청 본문 너무 큼 |
| **429** | Too Many Requests | Rate Limit |
| **451** | Legal Reasons | 법적 차단 |

**5xx 서버 오류**:
- **500 Internal Server Error** — 일반 서버 에러
- **502 Bad Gateway** — 게이트웨이/프록시 문제
- **503 Service Unavailable** — 일시 과부하·서비스 중단
- **504 Gateway Timeout** — 게이트웨이 시간 초과

## 왜 중요한가

- **디버깅 즉시 판단** — 첫 자리로 책임 소재 (클라 vs 서버)
- **[[REST-API]] 응답 설계** — API 만들 때 적절한 코드 선택 중요
- **[[SEO]] 영향** — 301 vs 302, 404 처리 등

## 자주 헷갈리는 것

### 301 vs 302

- **301**: 영구 이동 → 검색엔진이 새 URL 색인. 사이트 이전 시 필수
- **302**: 임시 이동 → 검색엔진이 원래 URL 유지. 일시적 우회용

### 401 vs 403

- **401**: 인증 안 됨 ("너 누구?")
- **403**: 인증 됐지만 권한 부족 ("그건 못 해")

[[인증과-인가]] 참조.

### "500이면 서버 잘못"

절반만 맞음. 클라이언트가 이상한 데이터 보내서 서버 코드가 터지면 500이지만 **근본 원인은 클라이언트**일 수 있음.

## 일상 비유: 카페 주문 결과

- **2xx 성공** — "주문 받았어요"
- **3xx 리다이렉트** — "옆 매장 가세요"
- **4xx 손님 잘못** — 400 = 주문 이상, 401 = 회원 카드 가져오세요, 403 = 회원 맞지만 그 메뉴 권한 ❌, 404 = 그런 메뉴 없음
- **5xx 매장 잘못** — 500 = 매장 사정, 503 = 잠시 영업 ❌
