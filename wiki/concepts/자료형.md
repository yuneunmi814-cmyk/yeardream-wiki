# 자료형 (Data Type)

## 한 줄 정의

**자료의 종류와 그에 따른 동작 규칙**. Python 4대 기본 자료형 = 숫자·문자열·리스트·논리.

## 핵심 아이디어

### Python 4대 기본 자료형

| 자료형 | 영문 | 예시 | 변경 가능 |
|---|---|---|---|
| **숫자** | `int`·`float`·`complex` | `3`, `3.14`, `3+4j` | (불변) |
| **문자열** | `str` | `"Hello"`, `'안녕'` | ❌ 불변 |
| **리스트** | `list` | `[1, 2, 3]` | ✅ 가변 |
| **논리** | `bool` | `True`, `False` | (불변) |

### 숫자형 3종

```python
3       # int (정수)
3.14    # float (실수)
3+4j    # complex (복소수, 실무에선 드묾)
```

### 문자열 (str)

```python
'Hello!'       # 작은 따옴표
"Hello!"       # 큰 따옴표 (둘 다 OK)
'3.14'         # 따옴표 안 숫자도 문자열
```

⚠️ `"3.14"` ≠ `3.14`. 전자는 str, 후자는 float. 연산 결과 완전히 다름.

### 리스트 (list)

```python
[]                       # 빈 리스트
['a', 'b', 'c']          # 문자열들
['a', 2, True, [1,2]]    # 다른 자료형 섞임도 OK
```

→ 자세한 메서드는 [[리스트]] 페이지.

### 논리 (bool)

```python
True       # 참
False      # 거짓
```

⚠️ 첫 글자 대문자. `true` ❌.

### `type()` — 자료형 확인

```python
print(type(3))           # <class 'int'>
print(type(3.14))        # <class 'float'>
print(type("hi"))        # <class 'str'>
print(type([1,2]))       # <class 'list'>
print(type(True))        # <class 'bool'>
```

→ 디버깅 첫 도구. "이거 왜 안 되지?" 할 때.

### 형 변환 (Type Casting)

| 함수 | 변환 | 예시 |
|---|---|---|
| `int()` | 정수로 | `int("100")` → `100` |
| `float()` | 실수로 | `float("3.14")` → `3.14` |
| `str()` | 문자열로 | `str(100)` → `"100"` |
| `list()` | 리스트로 | `list("abc")` → `['a','b','c']` |
| `bool()` | 논리로 | `bool(0)` → `False`, `bool(1)` → `True` |

```python
"3" + 5             # TypeError
int("3") + 5        # 8
str(3) + "5"        # "35"
```

## 왜 중요한가

- **모든 연산의 기초** — 자료형 모르면 `"3" + 5` 같은 에러 매일 만남
- **`input()` 결과 가공 필수** — input은 항상 str → 숫자로 쓰려면 int 변환
- **API 응답 처리** — JSON 파싱 결과가 dict·list 자료형

## 자주 헷갈리는 것

### `"3"` vs `3`

```python
"3" + "5"           # "35"   (문자열 이어붙이기)
3 + 5               # 8      (정수 덧셈)
"3" + 5             # TypeError
```

### Python의 동적 타입

같은 변수에 다른 자료형 재할당 가능:
```python
x = 10         # int
x = "hi"       # str로 변경 OK
```

→ 편하지만 실수 잦음. 큰 코드에선 type hint 권장 (강의 미커버).

### "리스트 안에 다른 자료형"

```python
mix = ['a', 2, True, 3.14, [1,2]]    # OK
```

→ Python 리스트는 **이질적**. Swift Array처럼 한 타입 강제 안 함.

### 불변 vs 가변

- **불변 (immutable)**: int·float·str·tuple·bool — 한 번 만들면 못 바꿈
- **가변 (mutable)**: list·dict·set — 원소 변경 가능

```python
s = "hello"
s[0] = "H"          # TypeError (str 불변)

L = [1, 2, 3]
L[0] = 10           # OK (list 가변)
```

## 일상 비유: 그릇 종류

- **숫자 그릇** = 저울 위 (산술 가능)
- **문자열 그릇** = 종이 (이어붙이기 가능, 한 번 쓰면 못 지움)
- **리스트 그릇** = 큰 박스 (다른 그릇들도 담음, 변경 가능)
- **논리 그릇** = 스위치 (켜짐/꺼짐 둘 중 하나)
