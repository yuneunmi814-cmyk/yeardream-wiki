# TCP / UDP

## 한 줄 정의

**TCP** = 신뢰성 있는 데이터 전송, **UDP** = 빠른 데이터 전송. [[OSI-7계층]] 4층(전송) 양대 프로토콜.

## 핵심 아이디어

인터넷의 거의 모든 통신이 TCP 또는 UDP 둘 중 하나. **신뢰성(TCP) vs 속도(UDP) 트레이드오프**.

### TCP — "신뢰성 우선"

```
   클라이언트                  서버
       │  ─── SYN ─────────→  │      "연결할래?"
       │  ←── SYN+ACK ────── │      "응 가능, 너도 OK?"
       │  ─── ACK ─────────→  │      "OK 시작"
       === 3-way handshake 완료 ===
       │   --- 데이터 송수신 ---
       │  ─── FIN ─────────→  │      "끊자"
       │  ←── ACK ────────── │
       │  ←── FIN ────────── │
       │  ─── ACK ─────────→  │
       === 4-way handshake 완료 ===
```

특징:
- **연결형** (3-way handshake로 연결)
- **신뢰성 보장** (패킷 분실 시 재전송, 순서 보장)
- **흐름·혼잡 제어**
- **Full-Duplex** (양방향 동시)
- **속도 느림**

### UDP — "속도 우선"

악수 없이 그냥 던짐. 도착 보장 ❌, 순서 보장 ❌, 빠름 ✅.

### 비교

| | TCP | UDP |
|---|---|---|
| **연결** | 연결형 (3-way) | 비연결형 |
| **순서** | 보장 | 보장 ❌ |
| **수신 확인** | 함 | 안 함 |
| **통신** | 1:1 | 1:1, 1:N, N:N |
| **신뢰성** | 높음 | 낮음 |
| **속도** | 느림 | 빠름 |
| **실사용** | 웹(HTTP)·메일·SSH·파일전송 | 음성통화·게임·영상스트리밍·DNS |

## 왜 중요한가

- **모든 인터넷 통신의 기초** — 어떤 앱·서비스든 TCP·UDP 중 하나 사용
- **실시간 vs 정확성** 선택 기준 — 영상 통화는 UDP, 메시지는 TCP

## 자주 헷갈리는 것

### "3-way handshake가 왜 3번?"

2번이면 송신자만 확인, 4번이면 낭비. **3번이 양쪽 확인하는 최소 횟수**.

### "UDP는 쓸모없는 거 아닌가?"

영상 통화·게임은 패킷 한두 개 잃는 게 1초 지연보다 나음 → UDP 합리적. DNS도 UDP (응답 짧고 빠름).

## 일상 비유: 등기 vs 일반 우편

- **TCP (등기)**: "받았어요" 확인, 분실 시 재발송, 순서 보장. **느림·확실**
- **UDP (일반 우편)**: 그냥 던짐, 도착 모름. **빠름·불확실**
