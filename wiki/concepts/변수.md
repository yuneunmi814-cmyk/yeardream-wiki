# 변수 (Variable)

## 한 줄 정의

**자료를 담는 "그릇" + 그릇에 붙인 라벨**. `이름 = 자료` 형태로 만듦. 모든 코드의 시작.

## 핵심 아이디어

```python
num = 10                      # 숫자 그릇
name = "Michael"              # 문자열 그릇
grade = ['A+', 'B+', 'A0']    # 리스트 그릇
```

→ **`=`는 "같다" ❌, "할당" ✅**.

### 변수 이름 규칙 4가지 ⚠️

```python
# 1. 숫자로 시작 ❌
1name = "Coke"        # SyntaxError

# 2. 숫자만 ❌
1234 = 5678           # SyntaxError

# 3. 예약어 사용 ❌
print = "Hello"       # 가능하지만 print() 함수 망가짐
                      # for, if, def, return, class 등도 ❌

# 4. 공백·연산자 ❌
prod name = "milk"    # 공백 안 됨
prod+date = "..."     # +, -, % 등 ❌
```

### 표기법 2종

```python
# 스네이크 (Python 권장)
user_name = "yoon"
order_count = 5

# 카멜 (Swift·JS 흔함, Python 비권장)
userName = "yoon"
orderCount = 5
```

→ Python은 **PEP 8 표준**으로 스네이크 권장. 사용자 SwiftUI 경험상 카멜에 익숙 → **의식적으로 스네이크** 연습.

### 재할당

```python
x = 10
x = 20        # 이전 10 사라짐
print(x)      # 20
```

→ 변수는 **마지막 할당값**만 기억.

### Python 동적 타입

```python
x = 10           # int
x = "hello"     # str로 바뀌어도 OK (Python 동적 타입)
x = [1, 2, 3]   # list로 또 바뀜
```

→ Swift·Java 같은 정적 타입 언어와 큰 차이. 편하지만 실수 잦음.

## 왜 중요한가

- **모든 코드의 시작점** — 변수 없이는 아무것도 못 함
- **가독성** — 좋은 변수 이름은 주석보다 강력 (`x` 보다 `user_age`)
- **디버깅** — 변수 추적이 디버깅의 90%

## 자주 헷갈리는 것

### `=` vs `==`

- `=` — **할당** (왼쪽 그릇에 오른쪽 값)
- `==` — **비교** ("같은가?")

```python
x = 5         # 할당
x == 5        # True (비교)
```

### "변수 = 값"이 아니라 "변수 = 값에 대한 참조"

```python
a = [1, 2, 3]
b = a            # b는 a와 같은 리스트를 가리킴
b.append(4)
print(a)         # [1, 2, 3, 4]   ← a도 바뀜!
```

→ 가변 자료형(list·dict) 할당 시 **얕은 복사** 됨. 진짜 복사하려면 `b = a.copy()` 또는 `b = list(a)`.

## 일상 비유: 라벨 붙인 상자

- **상자 (메모리)** + **라벨 (변수 이름)** + **물건 (자료)**
- 같은 상자에 새 물건 넣으면 이전 물건 사라짐
- 라벨 규칙: 숫자 시작 ❌, 공백 ❌, 예약어 ❌

## 좋은 변수 이름 vs 나쁜 변수 이름

```python
# 나쁨 (의미 불명)
x = 1234567890
n = 25
arr = ["apple", "banana"]

# 좋음 (의미 명확)
user_phone = 1234567890
user_age = 25
fruit_list = ["apple", "banana"]
```

→ 6개월 후 본인이 봐도 이해되는 이름. **명확성 > 짧기**.

## 🙋 강의 직접 검증 (2026-05-29 Day 4 라이브)

### 강의 자료 = 너 위키와 100% 일치 ✅

| 강의 자료 | 너 위키 박힌 거 |
|---|---|
| 선언 방법: `변수명 = 값` | ✅ |
| 이름 규칙 4가지 (숫자·예약어·공백·연산자) | ✅ |
| 표기법 권장: 스네이크 (`my_name`) | ✅ |
| 코드 예: `prod_name = "milk"` / `prod_date` | ✅ 동일 패턴 |

### 강사 메모 풀이 (2026-05-29)

> "프로그래밍 양식 파일명 붙여서 쓰기. 안 그러면 언더바_쓰기"

→ **변수명·파일명 모두 같은 규칙**:
- 공백 ❌
- 단어 구분 필요 → 언더바 (`_`) = 스네이크
- 예시: `prod_name.py`·`crawl_books.py`·`check_env.py`

→ 강사 PRISMA-KB도 `compile_auto.py`·`cron_path_test.py` 다 스네이크 ✅.

### NameError — Day 4 가장 자주 만남 ⭐

```python
score = 50
print("바뀐 점수:", b)   # ❌ b는 정의 안 됨
# NameError: name 'b' is not defined
```

**원인**: 변수 이름 오타 / 정의 안 한 변수 호출.

**비유**: 카톡에서 친구 A 검색해놓고 친구 B한테 답장 누른 격.

**해결**: 변수 이름 정확히 일치 (대소문자·언더바 포함).

### 동적 타이핑 보강 (강의 검증, 2026-05-29)

> "상자 하나에 숫자(50) 넣었다가 글자('ddfaddf')로 덮어써도 에러 ❌"

```python
score = 50              # 정수
score = "ddfaddf"       # 문자열로 재할당 OK
score = [1, 2, 3]       # 리스트로도 OK
```

**Python 자유 vs Swift·Java 엄격**:
- Python = 백종원 식당 (재료 자유, 셰프 책임)
- Swift = 미슐랭 (재료 표준, 시스템 강제)

**⚠️ 너 주의**: SwiftUI 정적 타입 익숙 → Python 진입 시 `type()` 자주 확인.
