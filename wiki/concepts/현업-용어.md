# 현업 IT 용어 묶음

> **묶음 페이지 안내**: 「현업에서의 용어 노하우」(32p)는 5개 하위 주제가 한 자료에 묶여 있어 일단 묶음 페이지로 작성. 후속 강의에서 동일 개념이 다시 등장하면 별도 페이지로 분리·누적([[CLAUDE]] 절대 규칙 5).

## 한 줄 정의

현업 IT 환경에서 비전공자가 자주 마주치는 5대 핵심 용어·도구의 작동 원리: **Version Control / 기획·플로우차트 / 오픈소스 / 클라우드 / SaaS**.

---

## 1. Version Control (Git)

### 한 줄
프로젝트 코드의 변경 이력을 시점별로 관리하고, 협업·롤백·브랜칭을 가능하게 하는 도구.

### 핵심
- **Git이 사실상 표준** — 리누스 토발즈(리눅스 OS 제작자) 작품
- 대표 서비스 두 곳:
  - **GitHub** (Microsoft가 8조원에 인수)
  - **Bitbucket** (Atlassian — 공동창업자가 호주 부호 6위)
- "설탕 1·2·3스푼" 비유: 어느 시점으로든 이동 가능
- 협업·테스트·출시·제작 등 **버전 분리 작업** 용이
- Git 공식 자기 소개 (강의 인용): "free and open source distributed version control system" — 분산형, 빠른 성능, Subversion·CVS·Perforce·ClearCase 같은 SCM 도구 능가

### 실 예시 — 카카오톡 v9.9.0 업데이트 노트

```
• 오픈채팅 홈 아이콘 추가
• 내프로필 QR 개선
• 오픈채팅 송금 기능 추가
• 이미지 편집기 개선
• 보이스룸 공감 아이템 추가
• 이모티콘 즐겨찾기 개수 확대 (최대 120개)
```

→ 각 항목 = 한 버전에 들어간 변경사항. **출시 노트가 곧 Version Control의 사용자 측면 표현**.

### 왜 중요한가
사용자는 이미 Claude Code 일상 사용자 → Git 작동 지식은 있을 것. 강의에선 비전공자 입문 톤.

## 2. 기획과 플로우차트

### 한 줄
서비스의 핵심 흐름을 데이터와 함께 시각화한 도식. 기획-개발 간 공통 언어.

### 강의 예시 1 — 쇼핑몰 핵심 로직

```
[상품 페이지 접속] → [상품 구매] → [상품 배송]
                                ↑
                    배송 정보가 없네? → 로그인/회원가입 추가

[상품 페이지] → [로그인] → [구매] → [배송]
                  ↑
        이름·전화·주소 자연스럽게 획득
                            ↑
                 간편결제(카카오페이/네이버페이)·카드·무통장
                            ↓
                  배송사 API 레퍼런스 체크
```

### 강의 예시 2 — 미스터리 박스 (분기 플로우)

```
[결제] → [미스터리 박스 구매 완료] ──오픈──→ [상품 당첨] ─┬─→ [배송]
                  │                                      │
                  ↓                                      └─→ [환불] → [상품 가격 포인트 환불]
            [100% 환불]                                       ↓
                  │                                     [마이페이지]
                  └────── 마이페이지 정보저장 ──────→         ↑
                                                              마이페이지 정보저장
```

→ 분기(당첨/환불/100% 환불)가 모두 **마이페이지로 정보 저장 수렴**. 단일 흐름이 아닌 여러 경로 처리 패턴.

### 프로토타입 이후의 작업

- 기획팀의 **상세 기획** 시작
- 개발자는 **레퍼런스·라이브러리·API 등 점검** ([[라이브러리-vs-프레임워크]], [[REST-API]])

### 왜 중요한가
1인 컨설팅에서 클라이언트한테 "이렇게 동작합니다" 보여줄 때 그대로 사용 가능. **프로젝트윤365 방법론 자산**으로 풀 만한 1번 케이스.

### 자주 헷갈리는 것
- 와이어프레임(화면 단위) vs 플로우차트(흐름 단위) — 다른 도구

## 3. 오픈소스

### 한 줄
프로젝트 전체 코드가 공개되어 누구나 활용·기여할 수 있는 소프트웨어. 단 **무료가 곧 자유는 아님** — 라이선스로 관리됨.

### 핵심
- **OLIS** (오픈소스 SW 라이선스 종합정보 시스템): https://www.olis.or.kr/license/compareGuide.do
- 안드로이드도 오픈소스 OS
- 대표 오픈소스: Firefox, OpenOffice, Blender, PHP, Java, MySQL, Debian, WordPress, Apache, Ubuntu, Linux
- 어플리케이션 메뉴에 "사용 중인 오픈소스 SW" 명시 = 라이선스 규칙 준수
- 대표 라이선스 카테고리:
  - **GPL** (전염성 강함, 파생물도 공개 의무)
  - **MIT / Apache** (상대적으로 자유)

### 실 예시 — 카카오톡 OSS Notice

라이선스 규칙에 따라 카카오톡이 자사가 사용하는 오픈소스를 안드로이드 메뉴에 기재:

```
• abseil-cpp (Apache License 2.0) — github.com/abseil/abseil-cpp
• android-platform-cts (Apache License 2.0)
• android-platform-frameworks-base (Apache License 2.0)
• android-platform-frameworks-ex (Apache License 2.0)
• android-platform-frameworks-multidex (Apache License 2.0)
• android-platform-frameworks-opt-calendar
• ... (계속)
```

→ **사용자가 1인 컨설팅에서 클라이언트 앱 빌드 시 동일한 OSS Notice 작성 의무** 발생. Apache License 2.0이 대세.

### 왜 중요한가
**프로젝트윤365 클라이언트 솔루션 빌드 시 직접 영향**. GPL 코드 사용하면 클라이언트 코드도 공개해야 하는 함정. 콘텐츠 후보 1번 → "오픈소스 라이선스 모르고 쓰면 큰일나는 케이스 3가지".

## 4. 클라우드

### 한 줄
실물 서버 없이 인터넷을 통해 서버·소프트웨어를 빌려 쓰는 모델. 사용한 만큼 과금, 유연한 확장.

### 핵심
- **기존 서버의 단점**: 용량 확장 경직, 유지비 고정, 계산 틀리면 추가비용 부담
- **클라우드의 장점**:
  - 유연한 셋업
  - **사용량 기반 과금** (pay-as-you-go)
  - 365일 정상 가동 운영 부담 감소
- 트래픽 증감 → 증설·폐기 클릭 몇 번 (강의 명시 시나리오: 증가 → 증설 → 감소 → 폐기)
- **강의 명시 대표 서비스 3대**:
  - **AWS** (Amazon Web Services)
  - **Microsoft Azure**
  - **Google Cloud**
- (+ Cloudflare 등 사용자가 이미 쓰고 있는 서비스)

### 컨테이너화 (2일차 [[운영체제]] 마지막 슬라이드)

> "실행에 필요한 모든 환경과 패키지를 포함하여 하나로 묶는 기술. 내가 사용하는 OS에 상관없이 동일한 개발환경을 구성할 수 있음."

- Docker 추정. 클라우드 워크로드의 표준
- **1인 컨설팅 직결**: 클라이언트마다 다른 환경 차이로 인한 디버깅 시간 절감
- 자세히는 [[운영체제]] 컨테이너화 섹션 참조 (강의에선 한 슬라이드로 짧게)

### 왜 중요한가
사용자는 이미 **Cloudflare Pages** 사용자 → 클라우드 작동 지식 있음. 강의는 입문 톤일 듯. 1인 사업자 인프라 의사결정 멘탈 모델로 콘텐츠화 가능.

## 5. SaaS (Software as a Service)

### 한 줄
클라우드 기반의 구독형 소프트웨어 서비스. 설치·운영 부담 없이 브라우저·앱으로 바로 사용.

### 핵심
- 파일 공유 등 **공유의 유연성**
- 설치 과정 없이 **즉시 사용**
- **Adobe→Figma 28조원 인수** = SaaS 위력 사례
- 지속 확장 중인 핵심 시장

### 왜 중요한가
프로젝트윤365 자체가 잠재적 SaaS 비즈니스 후보. 인스타 콘텐츠 1번 후보 → "Figma는 왜 28조에 팔렸나 — SaaS 한 줄 설명".

---

## 자주 헷갈리는 것 (전체)

- **클라우드 vs SaaS**: 클라우드는 인프라 모델, SaaS는 그 위에서 돌아가는 소프트웨어 형태
- **IaaS / PaaS / SaaS** 3단계 구분 (강의 보충 후 갱신)

## 🙋 은미 정수 (2026-05-29) — 클라우드 깊이 풀이

### 클라우드 = "남의 서버 빌려 쓰는 것"

| | 옛 방식 | 클라우드 |
|---|---|---|
| **서버** | 직접 사서 사무실에 둠 | AWS·Google이 미리 사놓은 거 빌림 |
| **관리** | 직원이 직접 | 클라우드 회사가 자동 |
| **확장** | 트래픽 늘면 새 서버 구입 | 클릭 한 번에 |
| **비용** | 초기 큰 투자 | 사용한 만큼 |
| **비유** | **집 사기** | **호텔 묵기** |

### 클라우드가 빌려주는 거

- **서버** (EC2·GCE) — 컴퓨터 자체
- **DB** (DynamoDB·Aurora·Cloud SQL)
- **저장소** (S3·Cloud Storage) — 파일 보관
- **CDN** (CloudFront·Cloudflare) — 콘텐츠 가까이
- **AI 모델** (Bedrock·Vertex AI)
- 인증·결제·로깅 등 다

### 3대 클라우드 + 중소

**대형 3대장**:
- **AWS** (Amazon, 1위, 가장 큼)
- **GCP** (Google)
- **Azure** (Microsoft)

**중소 (은미 친숙)**:
- **Cloudflare** ⭐ (네 Pages) — CDN·웹앱·DNS
- **Vercel** — 프론트엔드 호스팅
- **Netlify** — 비슷

### ⚠️ 구글 클라우드 vs 구글 드라이브 — 완전 다름

| | 구글 드라이브 | 구글 클라우드 (GCP) |
|---|---|---|
| **누구용** | 일반 사용자 | **개발자·기업** |
| **목적** | 개인 파일 저장·공유 | 서버·DB·AI 인프라 임대 |
| **비유** | 너 개인 옷장 | **거대 백화점** |
| **가격** | 무료 15GB | 사용량만큼 ($수십~수만/월) |

→ 같은 "구글"이지만 **완전 별개 제품**.

### 너 이미 클라우드 사용자

**Cloudflare Pages 쓴다 = 이미 클라우드 사용 중**. 클라우드 진입 ❌ 불필요. 다음 단계는 **AWS·GCP 어디까지 확장하느냐**.

## 🙋 은미 정수 2 (2026-05-29) — Day 1 05강 정리

### 한국 기업 클라우드 활용

| 기업군 | 사용 | 이유 |
|---|---|---|
| **당근·배민·토스·쿠팡** | AWS (절대다수) | 트래픽 폭증 대응 1위 |
| **이마트·롯데온** | Azure | "아마존은 유통 경쟁사" |
| **넷마블·엔씨소프트** | AWS + GCP 멀티 | 글로벌 |

→ 한국 스타트업·중견 = **AWS 메인** 거의 표준.

### Git vs GitHub — 일기장 vs 블로그

- **Git** = 내 컴퓨터 안 **타임머신 일기장** (게임 세이브 슬롯)
  - Commit 누르면 그 시점 스냅샷 박제
  - 망쳐도 그 슬롯으로 롤백
- **GitHub** = Git 일기장 올리는 **블로그·네이버 카페** (공유·협업)

→ Git=기술 / GitHub=공간.

### 플로우차트 — 시각화로 빈틈 발견

말로 풀면 뇌가 대충 넘김 → **도형·화살표로 그려야 빈틈 발견**.

로그인 예시:
- 글: "사용자 로그인 → 메인 이동"
- 플로우차트: 마름모(조건) 그려야 함
  - 아이디 틀리면?
  - 비밀번호 5번 틀리면?
  - 인터넷 끊겼으면?
- → "갈 곳 없는 화살표" 발견 = 버그 사전 예방

### 오픈소스 라이선스 상세

- **OLIS** = 한국저작권위 오픈소스 정보 검색 포털
- **OSS Notice** = 우리 앱에 "이 오픈소스 썼어요" 명시. 안 하면 저작권 소송

라이선스 4종은 [[라이브러리-vs-프레임워크]] 참조 (MIT·Apache·GPL·BSD).

### SaaS

**완성된 SW를 설치 없이 웹에서 구독형 사용**.

대표:
- **Figma** (디자인) — 어도비 28조 인수 시도 → 독과점 무산
- **Notion**·**Slack**·**Gmail**·**Canva**

**클라우드 vs SaaS**:
- 클라우드 = 서버 하드웨어 임대 (AWS·GCP)
- SaaS = 완성된 SW 구독 (그 위에 동작)

### 피그마 핵심 4기능

| 기능 | 역할 |
|---|---|
| **Canvas + Frame** | 무한 도화지 + 화면 크기 틀 |
| **Component + Instance** | 원본·복사본. 원본 바꾸면 복사본 다 자동 변경 |
| **Prototype** | 화살표로 화면 연결 → 진짜 앱처럼 시뮬 |
| **Dev Mode** | 디자인 → CSS 코드 자동 추출 |

### 캔바 vs 피그마

| | 캔바 | 피그마 |
|---|---|---|
| **타깃** | 비디자이너 (마케터·학생) | 디자이너·기획자·개발자 |
| **목적** | 템플릿 기반 빠른 결과물 | 앱·웹 설계·협업 |
| **현업** | 마케팅·인사·영업 | 프로덕트팀 |

**트렌드**: 서로 영역 침범
- 피그마 → FigJam·Figma Slides (대중 시장)
- 캔바 → Affinity 인수 (프로 디자이너 시장)

→ 디자인 종합 플랫폼 경쟁.

## 🙋 은미 정수 3 (2026-05-29) — 리눅스·우분투·Shell·권한 (Day 2 04강)

### 오픈소스 vs API — 노래 비유 (은미 정답)

| | 오픈소스 | API |
|---|---|---|
| **본질** | 코드 통째 공개 (수정 OK) | 데이터·기능만 빌려옴 |
| **노래 비유** | **작곡 프로젝트 파일** (트랙 다 공개) | 가사·멜로디 조회 |
| **활용** | 가사 개사·드럼 비트 변경 → 새 곡 | 화면에 가사 표시만 |

### GNU + Linux 커널 — 자전거 비유

- **Linux 커널** = 자전거 프레임·페달·체인 (구동계)
- **GNU** = 핸들·안장·브레이크 (사람 제어 도구)
- **합치면** = 우리가 쓰는 "리눅스 운영체제"

### 우분투 (Ubuntu) = 리눅스 OS의 완성차 브랜드

- 리눅스 커널 + GNU + 예쁜 GUI = 우분투
- 가정용 자전거(현대) vs 짐차(우분투, 가볍고 24시간 안 멈춤)

**왜 쓰나?**:
- 클라우드 서버(AWS·GCP)는 거의 다 우분투
- 무료·가볍·안정 → 24시간 운영 적합
- 개발자가 만든 프로그램이 배포될 곳

### ⚠️ 맥 ≠ 리눅스

**맥 = UNIX 계열**. 리눅스와 사촌 관계. 명령어(영어)는 비슷.

### 터미널·Shell·명령어 위계

```
[터미널] — 검은 창 (껍데기)
   ↓
[Shell] — 글자를 컴퓨터에 통역 (번역가)
   - Bash (영어, 리눅스 표준)
   - zsh (영어 후속, Mac 기본)
   - PowerShell (중국어, 윈도우 전용)
   ↓
[명령어] — 사용자가 치는 말 (cd·ls·pwd 등)
```

**Git Bash = 윈도우에 설치하는 영어 통역기** (은미 정답).

### 기초 명령어 4종 ⭐

```bash
whoami     # 나 누구? (계정 이름)
pwd        # 지금 어디? (현재 폴더 경로)
ls         # 여기 뭐 있어? (목록)
cd 폴더명   # 거기 가 (이동)
```

⚠️ pwd ≠ 비밀번호. **P**rint **W**orking **D**irectory.

### GUI vs CLI

| | GUI | CLI |
|---|---|---|
| **풀이름** | **G**raphical **U**ser **I**nterface | **C**ommand **L**ine **I**nterface |
| **방식** | 마우스·아이콘 | 텍스트 명령어 |
| **예** | 윈도우·맥 바탕화면 | 터미널·CMD·Git Bash |

→ **VS Code ≠ CLI**. IDE (소스코드 편집기).

### sudo·apt·root — 가족 비유 ⭐⭐

| | 가족 비유 | 설명 |
|---|---|---|
| **root** | 할아버지 | 절대 권력자. 시스템 통째로 변경 가능 |
| **일반 계정** | 손자 | 사고 방지. 제한적 권한 |
| **sudo** | 엄마한테 받는 할아버지 카드 | "내가 할아버지 권한으로!" + 비밀번호 인증 |
| **apt** | 심부름꾼 삼촌 | 앱 다운로드·설치 전문 |

**세트 명령**: `sudo apt install 프로그램명` ("할아버지 권한으로 + 삼촌아 + 설치해 + 이거")

### ⭐ 로컬 영향 여부로 명령어 분류 (은미 정답)

| 분류 | 명령어 | 로컬 영향 |
|---|---|---|
| **구경·이동** | cd·pwd·ls·whoami | ❌ 없음 (안전) |
| **시스템 변경** | sudo·apt·rm·chmod | ✅ 있음 (대공사) |

→ 변경 작업은 항상 sudo 권한 + 신중.

### 다른 OS 깔아 쓰기 — 가상화 3종

| 방식 | 설명 | 예 |
|---|---|---|
| **VM** | 컴퓨터 안에 가짜 컴퓨터 통째로 | VirtualBox·VMware·UTM |
| **WSL2** | 윈도우 + 리눅스 한 몸 (윈도우 표준) | Windows 안의 우분투 |
| **부트캠프** | 디스크 나눠 깔기 | 옛 Intel 맥에서만 (M1+ ❌) |

### 맥에서 윈도우 쓰기

| 방법 | 비용 | 추천도 |
|---|---|---|
| **Parallels Desktop** | 유료 ($100+) | 가장 편함, 맥과 동시 사용 |
| **UTM** | 무료 | 가성비 |
| **Boot Camp** | 무료 | Intel 맥만 (M1+ ❌) |

→ 은미 맥 M 시리즈 → Parallels 또는 UTM만 가능.
