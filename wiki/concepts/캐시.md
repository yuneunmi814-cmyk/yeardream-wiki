# 캐시 (Cache)

## 한 줄 정의

**자주 쓰는 데이터를 빠른 곳에 임시 저장**. 매번 원본까지 안 가도 됨 → 속도 ↑·부담 ↓.

## 핵심 아이디어

```
[원본 DB (책장)] ───→ [캐시 (책상 위 메모지)] ───→ [요청]
                          ↑                          ↓
                          └── 자주 보는 거만 둠 ←─────┘
                          → 다음엔 책장 안 가도 됨
```

### Hit vs Miss

```
요청 → 캐시 확인
       ├── 있음 (Hit)  → 즉시 응답 (빠름)
       └── 없음 (Miss) → 원본에서 가져옴 → 캐시 저장 → 응답
```

### 캐시 종류 (가까운 곳부터)

| 위치 | 이름 | 예 |
|---|---|---|
| **CPU 안** | CPU 캐시 (L1·L2·L3) | 가장 빠른 메모리, CPU 자체 내장 |
| **서버 RAM** | 메모리 캐시 | **Redis**·Memcached |
| **너 컴퓨터** | 브라우저 캐시 | 이미지·CSS·JS |
| **너 근처 서버** | CDN 캐시 | **Cloudflare**·CloudFront |

## 왜 중요한가

- **서버 부담 줄이는 핵심 도구** — DB 부하 ↓
- **사용자 응답 속도 ↑** — 매번 DB 안 가도 됨
- **글로벌 서비스 필수** — CDN으로 전 세계 사용자에게 빠르게

## 일상 예

- **유튜브 이미 본 영상 빨리 뜸** = 브라우저 캐시
- **인스타 자주 보는 프로필 빨리 뜸** = 서버의 Redis 캐시
- **Cloudflare Pages** = 너 사이트가 전 세계 CDN 캐시에 분산됨
- **`Cmd+Shift+R`** = 캐시 무시하고 새로 받기 (강제 새로고침)

## 자주 헷갈리는 것

### 캐시 vs 쿠키

| | 캐시 | 쿠키 |
|---|---|---|
| **목적** | 성능 (빠른 로딩) | 식별·인증 |
| **데이터** | 페이지·이미지·CSS | 사용자 정보 |
| **삭제** | 수동·만료 | 만료 시 자동 |

### 캐시 vs DB

- 캐시 = **임시** (전원 끄거나 만료되면 사라짐, 보통 메모리)
- DB = **영구** (디스크 저장, 끄러도 살아 있음)

→ 보통 **DB(영구) + Redis(캐시)** 같이 씀.

### "캐시는 영원 저장"

❌. **TTL (Time To Live, 만료 시간)** 있음. 잘못 설정하면 오래된 데이터 응답.

### ⚠️ 캐시 삭제 ≠ 알고리즘 리셋 (2026-05-29 추가)

**가장 흔한 오해**. 둘은 다른 영역:

| | 캐시 삭제로 영향 | 영향 ❌ |
|---|---|---|
| 이미 본 영상 다시 뜸 | ✅ | |
| 페이지 로딩 잠시 느림 | ✅ | |
| 로그아웃 가능 (쿠키 같이) | ✅ | |
| **유튜브·인스타 추천 알고리즘** | | ✅ (서버 저장) |
| 계정·구독·시청 기록 | | ✅ |
| 검색 기록 | | ✅ |

→ 추천 알고리즘은 **서버에 저장된 너의 행동 데이터 기반**. 캐시 삭제 안 함.

**알고리즘 진짜 리셋**:
- 유튜브: 시청 기록 모두 삭제 + 일시 중지
- 인스타: 검색 기록·광고 관심사 삭제
- **시크릿 모드** (Incognito) = 새 계정처럼
- **새 계정 만들기** = 가장 확실

### 캐시 삭제 방법 (2026-05-29 추가)

**브라우저 Chrome**:
- Mac: `Cmd + Shift + Delete`
- 설정 → 개인정보 보호 → 인터넷 사용 기록 삭제

**iOS Safari**: 설정 → Safari → 방문 기록 및 웹사이트 데이터 지우기

**iOS 앱**: 앱 삭제 → 재설치 (앱별 캐시 직접 삭제 ❌)

**강제 새로고침** (현재 페이지 캐시만):
- Mac: `Cmd + Shift + R`
- Windows: `Ctrl + Shift + R`

## CDN (Content Delivery Network)

**캐시의 지리적 분산 버전**:

```
Origin Server (서울)
       ↑
   ┌───┼───────────────────┐
[Tokyo Cache] [Singapore] [LA Cache]
   ↓           ↓           ↓
일본 사용자  싱가폴 사용자  미국 사용자
```

**대표 CDN**:
- **Cloudflare** ⭐ (너 사용 중)
- AWS CloudFront
- Netflix Open Connect (자체)

**장점**: 글로벌 로딩 속도 ↑, 원본 서버 비용 ↓, DDoS 방어
**단점**: SPOF — Cloudflare 다운 시 전 세계 사이트 마비 (2022년 실제 발생)

## Redis 실제 사용 서비스 (2026-05-29 추가)

| 서비스 | Redis 용도 |
|---|---|
| 인스타 | 피드 캐싱·좋아요 카운트 |
| 트위터(X) | 타임라인 캐싱 |
| 카카오톡 | 채팅 메시지 임시 저장 |
| 배달의민족 | 실시간 주문 상태 |
| GitHub | 세션·Rate Limit |
| Discord | 실시간 채팅·온라인 상태 |
| 게임 (롤) | 실시간 랭킹 |

**일상 예**:
- 카톡 친구 목록 빨리 뜸 = Redis
- 인스타 "팔로워 1,234명" = Redis (DB 매번 안 셈)
- 라이브 방송 시청자 수 = Redis

### Redis vs 디스크 DB — 저장 위치 계층 (2026-05-29 추가)

**핵심**: Redis는 "가벼운" 게 아니라 **"빠른 위치(RAM)에 있음"**.

```
빠름 ↑                                          용량 ↓
CPU 캐시 → RAM (Redis) → SSD → HDD → 테이프
                                                용량 ↑
느림 ↓
```

| | Redis (인메모리) | PostgreSQL·MySQL (디스크) |
|---|---|---|
| 저장 | RAM | 디스크 |
| 속도 | 매우 빠름 | 느림 |
| 용량 | 작음·비쌈 | 큼·쌈 |
| 휘발성 | 전원 끄면 위험 | 영구 |
| 비유 | **책상 메모지** | **책장** |

→ 보통 **같이 씀**: PostgreSQL(영구) + Redis(캐시).
