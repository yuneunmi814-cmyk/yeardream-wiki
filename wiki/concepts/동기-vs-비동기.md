# Synchronous vs Asynchronous (동기·비동기)

## 한 줄 정의

- **동기 (Synchronous)**: 통신의 결과와 함께 다음 동작이 움직임 — **결과를 기다린다**
- **비동기 (Asynchronous)**: 통신의 결과와 상관없이 다음 동작이 움직임 — **기다리지 않는다**

## 핵심 아이디어

### 동기 (Synchronous) 사례

```
[로그인 후] ⇄ [Web Server]
   저장된 명함정보 요청 (GET) →
   ← 저장된 명함정보 전달
```

특징:
- 강의 목록 → 상세 페이지 이동
- 저장된 명함 정보 가져오는 페이지
- **일반적인 웹 페이지 이동**
- 정보와 함께 제공되어야 하는 화면

핵심: **결과가 와야 다음 화면이 보임** → 사용자가 기다림

### 비동기 (Asynchronous) 사례

> "통신의 결과와 상관없이 독립적인 결과를 가져옴" (강의 본문)

특징 (강의 본문):
- 페이지 이동이 없는 댓글 달기
- SNS 피드백

### 영업사원 비유 ⭐ (강의 핵심 비유)

```
[영업사원] → [명함전달] → [수령후 입력] → [디지털 명함]
    ↓
[영업활동]   (디지털 명함이 없어도 영업 활동은 독립적으로 가능)
```

→ 명함 디지털화는 백그라운드로 진행되고, 영업사원은 그동안 다른 미팅·연락을 계속함. **상호 의존 없는 병렬 진행**이 비동기의 본질.

## 왜 중요한가

- **AI Agent·자동화 워크플로우 설계의 핵심 결정 포인트**
  - "LLM 호출 끝날 때까지 기다릴까 vs 백그라운드로 던지고 다음 작업 갈까"
  - Claude Code 등 일상 도구가 이미 비동기 — 명시적 멘탈 모델로 정리 필요
- **사용자 경험 직결**: 동기로 짜면 사용자가 기다림 → 비동기로 풀면 UI 부드러워짐
- Stage 3 AI 트랙 (분석가·서비스개발자 둘 다) 일상 도구

## 자주 헷갈리는 것

- **비동기 = 빠르다 X**: 비동기는 "기다리지 않는다"이지 "빠르다"가 아님. 작업 자체 시간은 동일.
- **비동기 = 병렬(parallel) X**: 비동기는 호출 패턴, 병렬은 실제 동시 실행. 단일 스레드에서도 비동기는 가능 (JavaScript의 `async/await`).
- **콜백 / Promise / async-await**: 모두 비동기 표현 방법. JavaScript 기준 진화 순서.
- **동기 vs 블로킹**: 동기 ≈ 블로킹이지만 엄밀히는 다른 축. (강의는 깊이 다루지 않음, 후속 학습 영역)

## 🙋 은미 정수 2 (2026-05-29) — OS와의 상관관계 ⭐

### 가상메모리 ↔ 비동기 — Stage 3 수준 연결

**가상메모리에서 데이터 가져오는 순간 = 페이지 폴트 + I/O 작업 (느림)**.

**동기 방식이면**: CPU가 디스크 응답 기다리며 **멈춤** (Blocking) → 컴퓨터 렉

**비동기 방식**: CPU가 디스크 요청 던져두고 **다른 일 처리** → 데이터 도착 시 알림 → 매끄럽게

→ **가상메모리의 느려짐을 비동기로 감춤**. OS 효율성의 핵심.

### 카페 비유 (가상메모리 + 비동기)

매장(RAM) 좁아서 원두 재고를 지하 창고(가상메모리/디스크)에 둠.

- **동기**: 알바생(CPU)이 창고 갔다 올 때까지 카운터 멈춤. 손님 줄 멈춤. → **렉**
- **비동기**: 창고에 요청만 던져두고, 그 사이 다음 손님 음료 만듦. 원두 오면 그때 그 손님 음료 만듦. → **매끄러움**

자세히 → [[운영체제]] 페이지의 "가상메모리 ↔ 비동기" 섹션.

## 🙋 은미 정수 (2026-05-29) — 인스타 비유로 통일

**핵심**: 기다리느냐 / 시켜놓고 다른 거 하느냐.

| | 동기 (Sync) | 비동기 (Async) |
|---|---|---|
| **방식** | 끝까지 기다림 | 시켜놓고 다른 거 |
| **순서** | 순차적 | 동시 진행 가능 |
| **인스타 비유** | DM 1개 보내고 "전송 중" 빙글빙글 → 완료 뜸 → 그 다음 DM | **인스타 영상 4개 동시 업로드** → 백그라운드 진행 → 끝나는 순서대로 알림 |
| **게임 비유** | RPG 턴제 전투 (내 차례 → 적 차례 → 내 차례) | 롤 매칭 큐 (잡아두고 그 동안 채팅·유튜브) |

**전화 vs 카톡**:
- **전화 = 동기**: 상대 응답 기다림. 못 받으면 끝
- **카톡 = 비동기**: 메시지 보내고 다른 거 함. 상대가 나중에 답함
