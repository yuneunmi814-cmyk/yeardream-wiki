# SEO (검색엔진 최적화)

## 한 줄 정의

검색엔진의 **색인·랭킹 단계에서 내 페이지가 더 잘 노출되도록** 페이지 구조·메타·콘텐츠를 작업하는 것. 마케터·콘텐츠 운영자의 영역.

## 핵심 아이디어

### 구글 검색엔진 3단계

```
크롤링 → 색인 생성 → 최종 검색결과 제공
```

- [[크롤링]] — 웹페이지를 통째로 가져오는 단계 (Googlebot)
- [[색인]] — 가져온 페이지를 검색 가능한 구조로 저장
- 검색결과 제공 — 사용자 질의에 맞춰 색인에서 순위화하여 반환

**SEO의 본질** = "색인 단계에서 더 효율적으로 노출되도록 작업" — 강의 본문 원문 표현.

### 검색엔진 역사 (강의 명시)

- **1996년** 스탠포드 대학원에서 검색엔진 연구 시작
- 연구 프로세스: `웹사이트 다운로드 → DB 저장 → 구성요소 분류·분석`
- **래리 페이지(Larry Page) · 세르게이 브린(Sergey Brin)** — 1998년 구글 창립
- 현재 웹사이트 규모: **114억 개** (강의 시점)

### 검색엔진 용어 (구글 공식 가이드 인용)

| 용어 | 정의 |
|------|------|
| 색인 (Index) | 알고 있는 모든 웹페이지를 저장. 각 항목은 콘텐츠와 URL 명시 |
| 크롤링 (Crawling) | 신규·업데이트 웹페이지를 찾는 프로세스 (링크 추적·사이트맵 등) |
| 크롤러 (Crawler) | 크롤링 후 색인을 생성하는 자동 소프트웨어 |
| Googlebot | 구글 크롤러의 일반적 이름 |
| SEO | 검색엔진에서 찾기 쉽도록 사이트 개선하는 프로세스 / 그 직책 |

### 검색엔진에 사이트 알리는 방법

- 검색엔진이 가만히 찾아주길 기다리지 않음
- **네이버 Search Advisor** — 네이버 사이트 등록·관리
- **구글 Search Console** — 구글 사이트 등록·관리
- 해시태그(#크리스마스 #맛집 #오운완 등)도 SEO 시도의 일종

## 왜 중요한가

- **사용자 본인 채널 직결**: yoon.ai.lab / prompt.by.yoon 인스타 외에 블로그·랜딩 만들 때 즉시 적용
- **1인 컨설팅 자산**: 클라이언트가 만든 사이트 SEO 진단·개선이 그대로 서비스
- **콘텐츠 후보 다수**: "site:naver.com 검색 트릭", "검색엔진 3단계 한 장 설명" 등 비전공자 친화 토픽

## 자주 헷갈리는 것

- **SEO vs SEM**: SEO는 자연 검색 노출 최적화, SEM은 광고 포함 검색엔진 마케팅. 사용자는 Meta 광고 경험으로 SEM 쪽 친숙
- **온페이지 vs 오프페이지 SEO**: 페이지 내부 요소(메타·구조·콘텐츠) vs 외부 요소(백링크·도메인 권위)
- **색인되는 것 ≠ 상위 노출되는 것**: 색인 = 검색 결과에 등장할 자격, 랭킹 = 그 안에서의 순위

## 코드/실습 예시

### `site:` 검색으로 색인 확인

```
구글 검색창에 → site:naver.com
```

→ 구글이 색인한 naver.com 하위 페이지들 노출 (네이버 고객센터, 부동산, OGQ마켓 등). 본인 도메인에 적용하면 **어떤 페이지가 색인됐는지 즉시 확인 가능**.

## 🙋 은미 정수 (2026-05-29)

### PageRank — 구글 핵심 알고리즘

**아이디어**: 많은 사람이 링크하는 페이지 = 더 중요. **학교에서 인기 측정** 비유.

- 페이지 A → B 링크 = "A가 B 추천"
- 많은 페이지가 B 링크 → B 중요
- 또 중요한 페이지가 B 링크 → B 가중치 ↑↑

**만든 사람**: 래리 페이지 + 세르게이 브린, 1996 스탠포드. 1998 구글 창업 핵심 무기. 야후보다 정확해서 폭발.

**현재**: PageRank만 ❌. 200+ 신호 종합 (BERT·MUM 등 AI 추가).

### Search Console / Search Advisor — 사이트 운영자 도구

**역할**:
- 내 사이트 등록 ("검색 결과 보여주세요")
- 사이트맵 제출
- 검색 통계 (키워드별 노출·클릭률)
- 에러 진단·모바일 친화도

**도구**:
- **Google Search Console** (구글)
- **네이버 Search Advisor**
- **Bing Webmaster Tools**

**은미 적용**: yoon.ai.lab을 Google Search Console에 등록 → 키워드 통계 → 콘텐츠 전략. Meta 광고(유료) vs Search Console(무료 검색).

### Google Trends · 네이버 데이터랩 — 검색 키워드 트렌드

**용도**:
- 어떤 키워드 많이 검색?
- 시간별·지역별·연관 키워드
- 트렌드 비교 ("ChatGPT vs Claude")
- **콘텐츠 기획·시장 조사·계절성**

**은미 적용**:
- yoon.ai.lab 콘텐츠 → "AI 부트캠프"·"비전공자 AI" 검색량 보고 인기 시점 발행
- 프로젝트윤365 마케팅 → 클라이언트 업종 키워드 트렌드

**비유**: **인스타 인사이트** = 본인 게시물 통계 / **Google Trends** = 검색 시장 전체 통계.

### SEO ≠ 광고

| | SEO | 광고 (Meta·Google Ads) |
|---|---|---|
| **비용** | 무료 (시간 투자) | 유료 |
| **노출** | 자연 검색 결과 (Organic) | 광고 결과 (Paid) |
| **속도** | 느림 (수개월) | 즉시 |
| **지속성** | 영구적 (한 번 상위 가면) | 광고비 끊으면 사라짐 |

→ 은미 Meta 광고 경험 = 유료. SEO = 무료. **둘 다 운영하면 시너지**.

### BERT · MUM — 구글 검색 AI (2026-05-29 추가)

| | BERT (버트) | MUM (멈) |
|---|---|---|
| **출시** | 2018 구글 | 2021 구글 |
| **풀이름** | Bidirectional Encoder Representations from Transformers | Multitask Unified Model |
| **강력함** | 1배 | **BERT × 1000배** |
| **할 수 있는 것** | 문장 맥락 이해 | 다국어·이미지·텍스트 종합 |

**옛 검색 vs BERT/MUM**:
- 옛: 단어 일치만 ("내일 비" → "내일"·"비" 단어 찾기)
- BERT/MUM: **의미 이해** ("내일 비온대" = 우산 챙길지 알려줘)

→ **ChatGPT·Claude LLM의 조상격**. 구글이 검색에 LLM 적용.

### 구글 AI 계보 + Gemini 관계 (2026-05-29 추가)

```
2017 Transformer (구글 발명)     ← 모든 모던 AI의 토대
   ↓
2018 BERT                       ← 검색 특화
   ↓
2021 MUM                        ← BERT × 1000
   ↓
2023 PaLM, PaLM 2               ← 거대 LLM
   ↓
2023 Bard (베타)
   ↓
2024 Gemini ⭐                   ← 멀티모달 LLM
```

**BERT/MUM vs Gemini**:
- **BERT/MUM** = 구글 검색에 박힌 AI (특화)
- **Gemini** = 일반 사용 LLM (ChatGPT 같은 거)
- 같은 구글, 같은 Transformer 기반, **다른 제품·다른 목적**

**비유**: BERT/MUM = 도서관 사서 (검색 전문) / Gemini = 만능 어시스턴트.

### SEO = 색인 + 랭킹 둘 다 (2026-05-29 추가)

```
1단계: 색인에 들어가기   ← 후보 자격
   ↓
2단계: 랭킹 상위 가기    ← 클릭률
```

- 색인 안 들어가면 → 노출 자체 불가
- 색인 들어가도 100p 뒤면 클릭률 0%

→ SEO 작업 = 둘 다 신경.

### Google Search Console 등록 가이드 (2026-05-29 추가)

```
1. search.google.com/search-console 접속
2. 구글 계정 로그인
3. "속성 추가" → URL 입력 (yoon.ai.lab)
4. 소유권 확인:
   - DNS 레코드 추가 ⭐ (Cloudflare 사용자 쉬움)
   - HTML 파일 업로드
   - HTML 메타 태그
5. Cloudflare 대시보드 → DNS → TXT 레코드 추가
6. Search Console에서 "확인"
7. 며칠 후 데이터 보임
```

**볼 수 있는 것**: 키워드별 노출·클릭·평균 순위·검색 노출 페이지·모바일 친화도·에러.

### Google Trends 정확히 (2026-05-29 추가)

**Trends ≠ 실시간 검색어**.

| | 네이버 실시간 검색어 (폐지) | Google Trends |
|---|---|---|
| 본질 | 현재 순위 스냅샷 | **시간별 트렌드 그래프** |
| 비유 | 인스타 "지금 핫한 거" | 인스타 **인사이트 그래프** |

**가능한 것**:
- ChatGPT 검색량 1년 추이
- AI 부트캠프 지역별 분포
- 다이어트 vs 운동 비교
- 12월 인기 키워드
