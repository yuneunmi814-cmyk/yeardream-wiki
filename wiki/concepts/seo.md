# SEO (검색엔진 최적화)

## 한 줄 정의

검색엔진의 **색인·랭킹 단계에서 내 페이지가 더 잘 노출되도록** 페이지 구조·메타·콘텐츠를 작업하는 것. 마케터·콘텐츠 운영자의 영역.

## 핵심 아이디어

### 구글 검색엔진 3단계

```
크롤링 → 색인 생성 → 최종 검색결과 제공
```

- [[crawling]] — 웹페이지를 통째로 가져오는 단계 (Googlebot)
- [[indexing]] — 가져온 페이지를 검색 가능한 구조로 저장
- 검색결과 제공 — 사용자 질의에 맞춰 색인에서 순위화하여 반환

**SEO의 본질** = "색인 단계에서 더 효율적으로 노출되도록 작업" — 강의 본문 원문 표현.

### 검색엔진 역사 (강의 명시)

- **1996년** 스탠포드 대학원에서 검색엔진 연구 시작
- 연구 프로세스: `웹사이트 다운로드 → DB 저장 → 구성요소 분류·분석`
- **래리 페이지(Larry Page) · 세르게이 브린(Sergey Brin)** — 1998년 구글 창립
- 현재 웹사이트 규모: **114억 개** (강의 시점)

### 검색엔진 용어 (구글 공식 가이드 인용)

| 용어 | 정의 |
|------|------|
| 색인 (Index) | 알고 있는 모든 웹페이지를 저장. 각 항목은 콘텐츠와 URL 명시 |
| 크롤링 (Crawling) | 신규·업데이트 웹페이지를 찾는 프로세스 (링크 추적·사이트맵 등) |
| 크롤러 (Crawler) | 크롤링 후 색인을 생성하는 자동 소프트웨어 |
| Googlebot | 구글 크롤러의 일반적 이름 |
| SEO | 검색엔진에서 찾기 쉽도록 사이트 개선하는 프로세스 / 그 직책 |

### 검색엔진에 사이트 알리는 방법

- 검색엔진이 가만히 찾아주길 기다리지 않음
- **네이버 Search Advisor** — 네이버 사이트 등록·관리
- **구글 Search Console** — 구글 사이트 등록·관리
- 해시태그(#크리스마스 #맛집 #오운완 등)도 SEO 시도의 일종

## 왜 중요한가

- **사용자 본인 채널 직결**: yoon.ai.lab / prompt.by.yoon 인스타 외에 블로그·랜딩 만들 때 즉시 적용
- **1인 컨설팅 자산**: 클라이언트가 만든 사이트 SEO 진단·개선이 그대로 서비스
- **콘텐츠 후보 다수**: "site:naver.com 검색 트릭", "검색엔진 3단계 한 장 설명" 등 비전공자 친화 토픽

## 자주 헷갈리는 것

- **SEO vs SEM**: SEO는 자연 검색 노출 최적화, SEM은 광고 포함 검색엔진 마케팅. 사용자는 Meta 광고 경험으로 SEM 쪽 친숙
- **온페이지 vs 오프페이지 SEO**: 페이지 내부 요소(메타·구조·콘텐츠) vs 외부 요소(백링크·도메인 권위)
- **색인되는 것 ≠ 상위 노출되는 것**: 색인 = 검색 결과에 등장할 자격, 랭킹 = 그 안에서의 순위

## 코드/실습 예시

### `site:` 검색으로 색인 확인

```
구글 검색창에 → site:naver.com
```

→ 구글이 색인한 naver.com 하위 페이지들 노출 (네이버 고객센터, 부동산, OGQ마켓 등). 본인 도메인에 적용하면 **어떤 페이지가 색인됐는지 즉시 확인 가능**.
