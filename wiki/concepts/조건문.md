# 조건문 (Conditional)

## 한 줄 정의

**특정 조건에 따라 실행되는 명령이 달라지는 구문**. Python에서는 `if`·`elif`·`else` + 들여쓰기로 분기.

## 핵심 아이디어

### 기본 `if`

```python
if 조건 :
    <명령>
```

```python
i = 1
if i == 1 :
    print("일")          # 조건 True일 때만
```

### `if-else`

```python
if 조건 :
    <명령>
else :
    <명령>
```

```python
score = 78
if score >= 60 :
    print("합격")
else :
    print("불합격")        # ← 출력
```

### `if-elif-else`

```python
if 조건1 :
    <명령>
elif 조건2 :
    <명령>
elif 조건3 :
    <명령>
else :
    <명령>
```

```python
score = 78

if score >= 90 :
    print("A")
elif score >= 75 :
    print("B")           # ← 출력
elif score >= 60 :
    print("C")
else :
    print("F")
```

**실행 로직**: 위에서 아래로 검사, **첫 True에서 멈춤**. 이후 elif·else 건너뜀.

### 비교 연산자 6종

| 연산자 | 의미 |
|---|---|
| `==` | 같다 |
| `!=` | 다르다 |
| `>` `<` | 크다 / 작다 |
| `>=` `<=` | 크거나 같다 / 작거나 같다 |

### 논리 연산자 3종

```python
# AND: 모두 True
age >= 18 and age < 65

# OR: 하나라도 True
day == "토요일" or day == "일요일"

# NOT: 반대로
not is_logged_in
```

### 중첩 조건문

```python
if is_member :
    if level == "VIP" :
        discount = 30
    else :
        discount = 10
else :
    discount = 0
```

→ 들여쓰기 2단으로 표현. 너무 깊으면 가독성 ↓ → 일찍 return·`elif` 활용으로 평탄화 권장.

## 왜 중요한가

- **프로그램의 첫 "지능"** — 조건문 없이는 항상 같은 흐름만
- **모든 로직의 기초** — 회원/비회원, 로그인 성공/실패, A/B 테스트 등
- **API 응답 처리** — 상태코드별 분기 (`if response.status_code == 200`)

## 자주 헷갈리는 것

### `=` vs `==` ⚠️⚠️

```python
x = 5         # 할당 (변수에 저장)
x == 5        # 비교 (같은가?)

if x = 5 :    # SyntaxError ← 할당은 조건문에 못 씀
if x == 5 :   # OK
```

→ **가장 흔한 초보 실수**.

### Python 들여쓰기

```python
if x > 0 :
    print("양수")        # 들여쓰기 (보통 스페이스 4)
print("끝")              # 들여쓰기 풀면 if 밖
```

→ Python은 들여쓰기가 **문법의 일부**. SwiftUI·JavaScript처럼 `{}` 안 씀.

### `if` 여러 개 vs `elif`

```python
# 독립 검사 (둘 다 실행 가능)
if score >= 90 : print("A")
if score >= 75 : print("B")          # 90점이면 A·B 둘 다 출력

# 배타적 (첫 True만)
if score >= 90 : print("A")
elif score >= 75 : print("B")        # 90점이면 A만 출력
```

### "True / False만 조건이 되는 건 아님"

Python은 **Truthy / Falsy** 개념:
```python
# Falsy로 취급되는 값:
0, 0.0, "", [], {}, None, False

# 그 외는 모두 Truthy
if my_list :              # 비어있으면 False, 있으면 True
    print("리스트 있음")
```

### 짧은 조건문 (한 줄)

```python
result = "양수" if x > 0 else "음수 또는 0"      # 삼항 연산자
```

→ 강의 미커버지만 실무에서 자주.

## 🙋 은미 정수 (2026-05-29 Day 4 실습)

### 자주 만나는 함정 3가지

#### 1. `and`·`or` 양쪽 비교 대상 명시 필수

```python
if 18.5 <= bmi and < 23:        # ❌ SyntaxError
if 18.5 <= bmi and bmi < 23:    # ✅
```

**원인**: 컴퓨터는 `and < 23`만 보면 "무엇을 비교?" 모름.

#### 2. `else` 뒤 조건 ❌ → `elif` 사용

```python
if x > 10:
    ...
else x > 5:        # ❌ SyntaxError
    ...

# 올바른 거
if x > 10:
    ...
elif x > 5:        # ✅
    ...
else:              # ✅ else는 조건 없음
    ...
```

#### 3. **연속 부등호 (Chained Comparison)** ⭐ — Python 특별 문법

```python
# Python 직관 문법 (수학 공식처럼)
if 18.5 <= bmi < 23:
    print("정상")

# 위와 동일 (and 사용)
if 18.5 <= bmi and bmi < 23:
    print("정상")
```

→ **Python·Ruby만 지원** (Swift·Java·JS ❌).

**비유**: 수학 공식 `18.5 ≤ bmi < 23` 그대로 코드로.

### BMI 분류 예시 (실전)

```python
weight = 70.0
height_cm = 175.0

height_m = height_cm / 100
bmi = weight / (height_m ** 2)
print("BMI:", round(bmi, 2))

if bmi < 18.5:
    print("저체중")
elif 18.5 <= bmi < 23:    # 연속 부등호
    print("정상")
elif 23 <= bmi < 25:
    print("과체중")
else:                      # 25 이상
    print("비만")
```

### ⚠️ AI 답 검증 습관

> Gemini가 알려준 BMI 코드에 들여쓰기 오류 있었음 (else가 elif 안에 박힘)

→ **AI 답 100% 신뢰 ❌**. 직접 실행해서 검증.
→ 강사 "AI 묻기 전 공식 먼저" + "실행해서 확인" 패턴.

## 🙋 강사 핵심 메시지 (2026-05-29 Day 4 라이브)

> **"프로그래밍은 코딩이 아니라 논리적 사고."**
> **"코딩 = 문법 문제. 기초 문법 1순위 = 조건문."**
> **"문법보다 어느 순간 조건문을 써야 하는지 한글로 표현할 줄 알면 돼."**

### 로지컬 씽킹 훈련법 — 실생활을 조건문으로 ⭐

**핵심**: "이쯤에서 조건문 써야겠다" 감각 키우기. 코드 X, **자연어로 분기 표현** 연습.

#### 예시 1: 출근 루틴

```
한글 사고:
  만약 비가 오면 → 우산 챙기기
  아니면 만약 미세먼지 나쁨 → 마스크 챙기기
  아니면 → 그냥 출발

코드 변환:
  if weather == "rain":
      take("umbrella")
  elif pm25 > 75:
      take("mask")
  else:
      go_out()
```

#### 예시 2: 카페 주문

```
한글 사고:
  만약 잔액 ≥ 5000원 → 라떼
  아니면 만약 잔액 ≥ 3000원 → 아메리카노
  아니면 → 물

코드 변환:
  if balance >= 5000:
      order = "라떼"
  elif balance >= 3000:
      order = "아메리카노"
  else:
      order = "물"
```

#### 예시 3: 인스타 콘텐츠 업로드 (프로젝트윤365 직결)

```
한글 사고:
  만약 클라이언트 피드백 = "수정" → 다시 작업
  아니면 만약 피드백 = "OK" AND 예약 시간 도달 → 자동 업로드
  아니면 → 대기

코드 변환:
  if feedback == "수정":
      revise()
  elif feedback == "OK" and now >= scheduled_time:
      upload()
  else:
      wait()
```

#### 예시 4: 알람 끄기 (너 일상)

```
한글 사고:
  만약 평일 AND 시간 = 7시 → 알람 울림
  아니면 만약 주말 AND 일정 있음 → 알람 울림
  아니면 → 자기

코드 변환:
  if is_weekday and hour == 7:
      ring()
  elif is_weekend and has_schedule:
      ring()
  else:
      sleep()
```

#### 예시 5: 음식 배달 결정

```
한글 사고:
  만약 냉장고 비어있음 → 배달
  아니면 만약 시간 없음 → 배달
  아니면 만약 요리 의욕 있음 → 요리
  아니면 → 라면

→ if·elif 4단 구조 자연스럽게 떠오름
```

### 훈련 패턴

| 단계 | 행동 |
|---|---|
| 1 | 일상 행동을 **"만약 ~면"·"아니면"** 으로 말로 풀기 |
| 2 | **분기 개수 세기** (if 1개? elif 몇 개? else 필요?) |
| 3 | **조건 우선순위** 결정 (위에서 아래) |
| 4 | 그제서야 코드로 변환 |

→ **문법은 외우면 됨. 분기 감각은 훈련**. 강사 강조: "기초 문법 1순위 = 조건문" 이유.

## 일상 비유: 알람 시계 메뉴

- "오전 6시?" → 일어나기 (if)
- "아니면 오전 9시?" → 운동 (elif)
- "그도 아니면?" → 자유 (else)

조건은 위에서 아래로 순서대로 확인. **한 번 맞으면 끝**.
