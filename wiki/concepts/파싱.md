# Parsing (파싱)

## 한 줄 정의

**구문 분석** — 규칙이 정해져 있는 문자열·문서를 잘라내 분석해서 **원하는 부분만 추출**하는 데이터 처리 기법.

## 핵심 아이디어

### 크롤링과 짝

[[크롤링]]이 "통째로 가져오기"라면, 파싱은 "필요한 부분만 골라내기".

```
웹사이트 크롤링 → Parsing 통한 가격정보 획득 → 최종 결과를 DB에 저장 또는 출력
```

### 강의 예시 — 다나와 가격 추출

크롤링 결과 HTML 중:

```html
<a href="https://prod.danawa.com/info/?pcode=17735156&cate=112758">
  <strong>1,899,000</strong>원
</a>
```

→ 원하는 가격(1,899,000)은 `<strong>` 태그 사이에 있다. **`<strong>` 블록 안의 텍스트만 뽑아내는 것**이 파싱.

### 포켓몬 빵 비유 (강의 본문)

```
포켓몬 빵 통째 → 포장지·빵 분류 → 스티커만 획득
  = 크롤링         = 파싱
```

### 크롤링뿐만 아니라

> "크롤링뿐만 아니라 프로그래밍 세계에선 두루 사용되는 데이터 처리 기법이다." (강의 원문)

- **JSON 파싱**: API 응답 ([[REST-API]])
- **HTML 파싱**: 웹 스크래핑
- **로그 파싱**: 서버 로그에서 패턴 추출
- **CSV 파싱**: 데이터 분석 입구
- **XML 파싱**: 설정 파일·RSS·SOAP API 등

## 왜 중요한가

- **거의 모든 데이터 처리 흐름의 입구**. AI 학습 데이터 준비·자동화 워크플로우의 첫 단계
- 사용자(프로젝트윤365)가 클라이언트 데이터를 다룰 때 90%는 "이 데이터 어떻게 뽑냐"가 파싱 문제로 환원됨
- Stage 3 AI 트랙(서비스개발자/분석가 무관) 모두에서 일상적으로 사용

## 자주 헷갈리는 것

- **파싱 vs 크롤링**: 자주 묶여 다니지만 다른 단계. 크롤링이 없어도 파싱은 가능 (이미 가지고 있는 파일에서 추출).
- **파싱 vs 정규표현식**: 정규표현식은 파싱의 **도구 중 하나**. HTML 같은 구조화된 문서는 정규식보다 전용 파서(BeautifulSoup, lxml)가 안전.
- **파싱 vs 토큰화**: 토큰화(tokenization)는 자연어 처리에서 문장을 단위로 쪼개는 것. 파싱은 그 뒤 구조 분석까지 포함. NLP·컴파일러에서 둘은 인접 단계.

## 🙋 은미 정수 (2026-05-29) — JSON vs XML + 프로그래밍 언어 관계

### 은미 표현 ✅ 정답

- **크롤링** = 빵 통째로 가져옴
- **파싱** = 스티커만 골라냄

### JSON vs XML — 둘 다 데이터 형식

같은 정보를 다르게 적는 법.

**JSON** (현재 90%, 가벼움):
```json
{ "name": "마리오", "level": 99, "items": ["버섯", "꽃"] }
```

**XML** (옛 표준, 무거움):
```xml
<character>
  <name>마리오</name>
  <level>99</level>
</character>
```

| | JSON | XML |
|---|---|---|
| 무게 | 가벼움 | 무거움 |
| 주석 | ❌ | ✅ |
| 현재 표준 | ⭐ 90% | 옛 시스템 |
| 인스타 비유 | 카톡 메시지 (가볍게 데이터만) | 한컴 한글 문서 (구조 빡빡) |

→ 신규 API는 거의 다 **JSON**. XML은 정부·은행·MS Office 내부에서.

### JSON·XML(재료) + 프로그래밍 언어(레시피) 상관관계

**프로그래밍 언어 = 명령** (해라!), **JSON·XML = 데이터** (이거 있음).

| | 프로그래밍 언어 | JSON·XML |
|---|---|---|
| **본질** | 명령 (해라!) | 데이터 (정보 자체) |
| **실행** | 실행되면 동작 | 그냥 정보, 동작 ❌ |
| **비유** | 레시피 | 재료 라벨 |
| **예** | `print("hi")` 실행됨 | `{"name":"hi"}` 데이터만 |

**관계**: 프로그래밍 언어가 JSON·XML을 **읽고·만들고·수정함**. 인스타 흐름:
1. 앱이 서버에 "내 피드 줘" 요청 (프로그래밍 언어)
2. 서버 응답: `{"posts":[...]}` ← JSON
3. 앱이 JSON 받아서 파싱 (프로그래밍 언어로)
4. 화면 표시 (프로그래밍 언어로 UI)

→ JSON·XML = **정보 포장지**. 프로그래밍 언어가 그 포장지 만들고 풀고 화면에 보여줌.
