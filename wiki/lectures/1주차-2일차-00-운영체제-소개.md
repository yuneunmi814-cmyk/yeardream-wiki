## 🚀 예습 (5분용)

### 한 줄 요약

**운영체제(OS) = 하드웨어 리소스를 제어하고 프로그램을 실행해주는 시스템 소프트웨어**. CPU·메모리·디스크·네트워크를 앱과 사용자에게 안전하게 빌려주는 중간 관리자.

### 핵심 단어 5~7개

- [[운영체제]] — OS의 정의·기능 통합 페이지
- [[컴퓨터-기초]] — 하드웨어(CPU·메모리·주변장치) 기초
- **리소스(Resource)** — CPU·메모리·디스크·네트워크 등 OS가 분배하는 자원
- **프로세스** — 실행 중인 프로그램 (다음 강 [[lectures/1주차-2일차-01-프로세스|01 프로세스]])
- **가상 메모리** — "안심 번호" 비유. 다음 강 [[lectures/1주차-2일차-02-메모리|02 메모리]]
- **파일 시스템** — 디스크 추상화. 다음 강 [[lectures/1주차-2일차-03-파일-시스템|03 파일 시스템]]

### 챕터 1. 컴퓨터 하드웨어 복습 (슬라이드 1~3)

**키워드 (강의 본문)**:
- **CPU**: 중앙 처리 장치. 데이터·명령어 처리. 정보는 **메모리에 위치**
- **메모리**: CPU가 처리할 데이터·명령어 저장. **주소 부여** + **전원 차단 시 삭제**
- **주변장치**: 입력(마우스·키보드), 출력(모니터·스피커), 보조 기억(SSD·HDD)

✦ **강의자 풀이 추정**
하드웨어 3대장. OS가 관리할 대상들:
- **CPU**는 "두뇌". 1초에 수십억 번 계산
- **메모리(RAM)**는 "책상". 빠르고 좁음, 전원 끄면 사라짐
- **저장장치(SSD)**는 "책장". 느리지만 영구
- **주변장치**는 입출력의 길

→ Day 1 [[lectures/1주차-1일차-00-컴퓨터의-이해|00 컴퓨터의 이해]]에서 짧게 다룬 내용 복습. 이 강의의 토대.

**🎯 핵심 메시지**: OS가 관리할 하드웨어 = CPU·메모리·주변장치. 메모리는 휘발성.

⚠️ **흔한 오해**: "메모리 = 저장장치" → 다름. 메모리(RAM) = 휘발성·빠름, 저장장치(SSD) = 영구·느림.

### 챕터 2. 운영체제의 개념 (슬라이드 4~9)

**키워드 (강의 본문)**:
- **OS 정의**: 하드웨어를 포함한 **리소스를 제어**하고 **프로그램을 실행**해주는 시스템 소프트웨어
- **사장 비유**: 작은 회사 = 사장이 직원 직접 관리 / 큰 회사 = 사장 혼자 관리 못 함 → OS 같은 매니저 필요
- **시스템 리소스**: 어떤 목적에 이용할 수 있는 하드웨어 포함 모든 것
- **리소스 부족 메시지**: "리소스가 부족하여 이 작업을 완료할 수 없습니다" (메모리 부족)
- **리소스 격리**: 각 응용프로그램마다 할당된 리소스, 다른 프로그램 리소스 강제 사용 불가

✦ **강의자 풀이 추정**
OS의 두 가지 동시 책임:
1. **리소스 제어** — CPU·메모리·저장 등을 누가 얼마나 쓸지 분배
2. **프로그램 실행** — 깔린 앱들이 잘 돌아가게

**왜 OS가 필요한가**(사장 비유):
- 옛 컴퓨터(1950년대) = 하드웨어 단순, 사용자 전문가 → 사장 혼자 OK
- 현대 컴퓨터 = 하드웨어 복잡, 사용자 일반인 → 매니저(OS) 필수

**리소스 격리**의 의미:
- A 앱이 B 앱 메모리 못 만짐 → 보안·안정성
- 한 앱 망가져도 다른 앱 정상
- iOS·Android 카메라 권한 다이얼로그가 격리의 일상 사례

**🎯 핵심 메시지**: OS = 하드웨어·자원을 앱·사용자에게 안전하게 빌려주는 중간 관리자.

⚠️ **흔한 오해**: "OS = Windows·macOS 같은 거" → 부분적 맞음. 폰(iOS·Android)·자동차·IoT·스마트워치도 다 OS 있음.

### 챕터 3. 운영체제의 기능 — 4대 (슬라이드 10~18)

**키워드 (강의 본문)**:
- **프로세스 관리**: 실행 중인 프로그램 = 프로세스. 리소스 할당, 상태 관리, 멀티프로세싱
- **메모리 관리**: 메모리에 정보 올리고 내림. **가상 메모리**로 실제 메모리 적어도 실행 가능
- **가상 메모리**: 실제 주소가 아닌 **가상의 주소**로 메모리를 관리
- **파일 시스템 관리**: 저장장치에 파일 읽고 쓰는 방법. 어떤 방식이든 프로그램이 고민 없이 사용
- **네트워크 관리**: 다양한 **프로토콜** 지원. **포트** 관리 (중복 방지)

✦ **강의자 풀이 추정**
**OS 4대 기능 = 다음 4강의 구조 그대로**:

| OS 기능 | 본 시리즈 강의 | 한 줄 |
|---------|---------------|------|
| 프로세스 관리 | [[lectures/1주차-2일차-01-프로세스|01]] | 실행 중 프로그램 관리, 5상태 전이 |
| 메모리 관리 | [[lectures/1주차-2일차-02-메모리|02]] | 가상 메모리, 페이징, 세그멘테이션 |
| 파일 시스템 | [[lectures/1주차-2일차-03-파일-시스템|03]] | inode, 링크, 마운트 |
| 네트워크 관리 | (간략) + 리눅스 [[lectures/1주차-2일차-07-프로세스와-네트워크|07]] | 포트·프로토콜, SSH |

**가상 메모리 = 안심 번호 비유** ⭐⭐ (강의 핵심 비유):
- 택배기사(프로세스)는 안심 번호(가상 주소)만 봄
- 통신사(OS)가 안심 번호 ↔ 진짜 번호(물리 주소) 변환
- 진짜 번호는 통신사만 앎 → 정보 보호
- 안심 번호는 연속된 깔끔한 형태 → 사용 편함

**🎯 핵심 메시지**: OS 4대 기능 = 프로세스·메모리·파일·네트워크. 다음 4강이 각각 깊이.

⚠️ **흔한 오해**: "가상 메모리 = 가짜 메모리" → 가상이지만 실제로 작동. 안심 번호처럼 추상화일 뿐.

📎 자세히: [[운영체제]] 가상 메모리 섹션, 다음 강의 [[lectures/1주차-2일차-01-프로세스|01 프로세스]]

### 챕터 4. 운영체제의 종류 (슬라이드 19~22)

**키워드 (강의 본문)**:
- **Windows**: Microsoft 제작. 일반 사용자, 최고 점유율
- **macOS**: **유닉스 기반**, Apple 제작. **터미널 명령어 사용 가능**, 개발 환경 적합. **상용**
- **Linux**: **무료·안정**, 개발·서버 적합. **오픈소스**. Ubuntu·Debian 등 배포판. **안드로이드도 리눅스 기반**
- **컨테이너화**: 실행에 필요한 모든 환경·패키지를 **하나로 묶는 기술**. OS 무관 동일 환경

✦ **강의자 풀이 추정**
**3대 OS 비교**:

| OS | 만든 곳 | 강점 | 사용자 |
|----|---------|------|--------|
| Windows | Microsoft | 게임·기업 SW 호환 | 일반·기업 |
| macOS | Apple | Unix + GUI | 디자이너·개발자 |
| Linux | 오픈소스 | 무료·서버 | 개발자·서버 운영 |

→ **사용자(은미)는 macOS 사용자** = Unix 명령어 익히면 Linux 서버도 90% 호환.

**컨테이너화 (Docker)**:
- "내 컴퓨터에선 되는데 다른 데선 안 되는" 영원한 문제 해결
- 앱 + 라이브러리 + 설정을 한 박스에 담음
- 어디서 풀어도 같은 환경

→ Day 1 [[lectures/1주차-1일차-05-현업-용어|05 현업 용어]] 클라우드 섹션과 연결.

**🎯 핵심 메시지**: OS 3종(Windows·macOS·Linux) + 컨테이너화. 사용자 macOS는 Unix 계열 = Linux와 호환성 ↑.

⚠️ **흔한 오해**: "컨테이너 = VM" → 다름. VM은 OS 전체 가상화(무거움), 컨테이너는 앱+필요부품만(가벼움).

## 💡 일상 비유

### 비유 1 — 호텔 매니저 (OS 전체)

호텔에 손님(프로세스)들이 체크인. 매니저(OS)가:
- 빈 객실(메모리) 배정 (할당)
- 룸 키(권한) 발급 (격리)
- 체크아웃 시 객실 회수 (해제)
- 청소·세탁·식사 등 부대 서비스(주변장치) 연결

손님 1명이 다른 손님 방 못 들어감 → 격리. 매니저 없으면 호텔 카오스 → OS의 본질.

### 비유 2 — 카카오톡 안심번호 (가상 메모리)

당근마켓 거래 시 카카오톡 안심번호로 통화 → 거래자는 진짜 번호 모름. 통신사가 안심번호 ↔ 실제 번호 변환.

가상 메모리 정확히 같은 구조:
- 프로세스(거래자)는 가상 주소(안심번호)만 봄
- OS(통신사)가 가상↔물리 매핑
- 실제 물리 주소(진짜 번호) 보호

→ 콘텐츠 후보: "OS의 가상 메모리 = 카카오톡 안심번호" (yoon.ai.lab 1컷)
