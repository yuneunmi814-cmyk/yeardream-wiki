**키워드**: `print()`·`type()`·`int`·`float`·`str`·`list`·`bool`·변수·`+`·`-`·`*`·`/`·`//`·`%`·`**`·인덱싱·슬라이싱.

✦ **강의자 정답 패턴**:
```python
# 콤마로 여러 값 한 줄 출력 — print 가장 자주 쓰는 패턴
print("이름:", "Alice", "나이:", 20)
print(100, "+", 200, "=", 100 + 200)
```

→ `print("문자", 변수)` 양식이 디버깅 출력의 표준.

✦ **연산 정답 패턴**:
```python
print(3 / 5)   # 0.6     ← 항상 float
print(13 // 5) # 2       ← 몫 (int)
print(13 % 5)  # 3       ← 나머지
print(2 ** 4)  # 16      ← 제곱
```

→ [[자료형]] 페이지 연산 매트릭스와 일치 ✅.

✦ **인덱싱·슬라이싱 양수/음수 비교**:
```python
word = "Ready"
# 양수:  R(0) e(1) a(2) d(3) y(4)
# 음수:  R(-5) e(-4) a(-3) d(-2) y(-1)
print(word[-1])     # y  (뒤에서 첫 번째)
print(word[-2])     # d

beta = [2, 4, 6, 8, 10, 12, 14]
print(beta[2:5])    # [6, 8, 10]   2 이상 5 미만
print(beta[:3])     # [2, 4, 6]    처음~3 미만
print(beta[4:])     # [10, 12, 14] 4 이상~끝
print(beta[:])      # 전체 복사
```

**🎯 핵심 메시지**: 시작은 **이상**, 끝은 **미만**. `[:]` = 전체 복사 패턴.

✦ **중첩 리스트 접근** (실습 5-3 종합 도전):
```python
student_info = ["홍길동", 20, "컴퓨터공학과", [90, 85, 100, 78, 92]]
student_info[3][0]    # 90  (내부 리스트의 첫 원소)
student_info[3][-1]   # 92  (마지막 점수)
student_info[3][:3]   # [90, 85, 100]
```

⚠️ **흔한 오해**: 중첩 리스트 접근 = `[][]` 두 번. JSON 응답 처리 시 매일 만나는 패턴.

### 챕터 2. 조건문 정답본 (input·논리·if/elif/else)

**키워드**: `input()`·형 변환·`==`·`!=`·`<`·`>`·`and`·`or`·`not`·`if`·`elif`·`else`·`in`.

✦ **input 형 변환 강조**:
```python
var1 = "100"
var2 = "300"
print(var1 + var2)            # "100300"   ← 문자열 이어붙임
print(int(var1) + int(var2))  # 400        ← 정수 덧셈
```

→ [[Python]] 자주 만나는 에러 4번 (input TypeError) 직결.

✦ **계산기 시나리오 — 중첩 조건문 정답**:
```python
num1, op, num2 = 10.0, "/", 3.0

if op == "+":
    print(num1 + num2)
elif op == "-":
    print(num1 - num2)
elif op == "*":
    print(num1 * num2)
elif op == "/":
    if num2 == 0:                   # ← 중첩 조건
        print("0으로 나눌 수 없습니다!")
    else:
        print(num1 / num2)
else:
    print("지원하지 않는 연산자")
```

**🎯 핵심 메시지**: **0 나누기 방어** = 실무 필수 패턴. 중첩 조건문 자연스럽게 등장.

✦ **계절 판별 — `in` 연산자 활용**:
```python
month = 8
if month in [3, 4, 5]:
    print("봄")
elif month in [6, 7, 8]:
    print("여름")
```

→ `in` = "리스트 안에 있나?" 한 줄로 표현. 길게 `or` 쓰는 것보다 깔끔.

### 챕터 3. 리스트 정답본 + 시퀀스 공통

**키워드**: `append`·`insert`·`remove`·`sort(reverse=True)`·`in`·`len`·`+`·`*`·str vs list.

✦ **`sort(reverse=True)` — 내림차순**:
```python
g = [6, 2, 4, 1, 3, 5]
g.sort(reverse=True)
print(g)    # [6, 5, 4, 3, 2, 1]
```

→ [[리스트]] "🙋 list.sort() 깊이 파기" 섹션과 일치 ✅.

✦ **str vs list 결정적 차이**:
```python
my_str  = "Python"
my_list = ['P','y','t','h','o','n']

my_list.append('!')    # ✅ 가능
# my_str.append('!')   # ❌ AttributeError (str 불변)
```

⚠️ **흔한 오해**: 문자열도 append 될 것 같지만 안 됨. str은 불변. 새 문자열 만들려면 `my_str + "!"`.

### 챕터 4. 시나리오 4종 ⭐⭐ (실전 미니 프로젝트)

> Day 4 모든 개념의 종합. **너 콘텐츠 자산화 직결** — yoon.ai.lab "비전공자가 일주일에 만든 Python 미니 프로그램".

#### 시나리오 1: 학생 성적 관리 시스템

```python
student_names  = ["Alice", "Bob", "Charlie", "David", "Eve"]
student_grades = [85, 72, 90, 65, 88]
pass_score = 70

for i in range(len(student_names)):       # ← Day 5 예상 (for 반복문)
    if student_grades[i] >= pass_score:
        status = "합격"
    else:
        status = "불합격"
    print(f"  {student_names[i]:<10} {student_grades[i]}점 → {status}")   # ← f-string

sorted_grades = student_grades[:]         # 원본 보존 복사
sorted_grades.sort()
top3 = sorted_grades[-3:]                 # 상위 3개
```

**🎯 핵심**: **인덱스 동기화** = 두 리스트(이름·점수)의 같은 인덱스가 같은 학생. 실무에서 dict로 가는 다리.

#### 시나리오 2: 편의점 재고·매출 관리

```python
products = ["콜라", "과자", "라면", "물", "커피"]
prices   = [1500, 1200, 900, 800, 2000]
stocks   = [10, 5, 3, 20, 7]

for i in range(len(products)):
    if stocks[i] == 0:
        status = "품절"
    elif stocks[i] <= 3:
        status = "부족"
    else:
        status = "정상"

# 판매 시 재고 감소
sold_idx = 2; sold_qty = 2
if stocks[sold_idx] >= sold_qty:
    stocks[sold_idx] -= sold_qty       # ← += / -= 누적 연산자
    revenue = prices[sold_idx] * sold_qty
```

**🎯 핵심**: `-=` 누적 연산자 = `stocks[i] = stocks[i] - 2` 단축형. **실무에서 매일**.

#### 시나리오 3: 숫자 맞히기 게임

```python
answer = 47
guesses = [30, 60, 45, 50, 47]
history = []

for attempt, guess in enumerate(guesses, start=1):   # ← enumerate 등장
    if guess == answer:
        result = "CORRECT!"
    elif guess < answer:
        result = "UP"
    else:
        result = "DOWN"
    history.append((attempt, guess, result))         # ← 튜플 등장
    if guess == answer:
        break                                         # ← break 등장
```

**🎯 핵심**: `enumerate` = 인덱스+값 동시. `(a, b, c)` = **튜플** (불변 리스트, Day 4 미커버).

#### 시나리오 4: 로그인 + 메뉴 시스템

```python
if user_id == CORRECT_ID and user_pw == CORRECT_PW:
    is_logged_in = True
elif user_id == CORRECT_ID:
    is_logged_in = False   # 비번만 틀림
elif user_pw == CORRECT_PW:
    is_logged_in = False   # 아이디만 틀림
else:
    is_logged_in = False   # 둘 다 틀림

if is_logged_in:
    menu_items = ["회원 정보 조회", "비밀번호 변경", "로그아웃"]
    for idx, item in enumerate(menu_items, start=1):
        print(f"  {idx}. {item}")

    valid_choices = list(range(1, len(menu_items) + 1))
    if choice in valid_choices:
        selected = menu_items[choice - 1]    # 인덱스 - 1 주의
```

**🎯 핵심**:
- 복합 조건 `and` 활용 (4분기)
- `range(1, N+1)` = 1부터 N까지 (끝 미만 주의)
- **인덱스 0부터 → 메뉴는 1부터 → `-1` 보정**

## 💡 일상 비유

### 시나리오 1: 학생 명부와 점수표
- **두 리스트 = 한 학생 정보** = 학교 명부의 "출석번호 5번 = 이름·점수·반"
- 인덱스 0부터 시작 = 출석번호 1번이 인덱스 0
- 정렬 = 줄 세우기. `[:]` = 명부 복사본 만들기

### 시나리오 2: 편의점 알바
- **재고 -= 판매** = 실제 알바 출입 시 컴퓨터가 하는 일
- 재고 ≤ 3 = "부족" → 점장 호출 (조건문 = 매뉴얼)
- 매출 += 단가 × 수량 = 영수증 자동 계산

### 시나리오 3: 술자리 숫자 야구
- **UP/DOWN 게임** = 1~100 사이 숫자 맞히기
- enumerate = "몇 번째 시도?" 자동 카운트
- 튜플 `(시도, 추측, 결과)` = 기록부 한 줄

### 시나리오 4: 카톡 로그인
- 아이디·비밀번호 검증 = 매일 만나는 패턴
- 4단 elif = "비번만 틀림 vs 아이디만 틀림 vs 둘 다" 친절한 에러 메시지
- 메뉴 인덱스 = 카톡 햄버거 메뉴
