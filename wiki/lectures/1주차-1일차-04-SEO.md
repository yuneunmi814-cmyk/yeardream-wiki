# 04 / 검색엔진 최적화 SEO — 교재 버전

## 🚀 예습 (5분용)

### 한 줄 요약

**검색엔진 3단계 = 크롤링 → 색인 → 검색결과**. SEO = 색인 단계에서 더 잘 노출되게 작업하는 것 (마케터의 영역).

### 핵심 단어 5~7개

- [[SEO]] — 통합 노트
- [[크롤링]] — 통째로 가져오기 (Googlebot)
- [[파싱]] — 원하는 부분 추출 (포켓몬 빵 비유)
- [[색인]] — 색인 (`site:` 트릭)
- **PageRank** (보강) — 구글의 핵심 알고리즘
- **Google Search Console / 네이버 Search Advisor** — 사이트 등록 도구
- **Google Trends / 네이버 데이터랩** — 크롤링 대안

### 큰 그림

```
검색엔진 3단계:
   ① 크롤링 (Googlebot이 페이지 가져옴)
        ↓
   ② 색인 (DB로 정리)   ← SEO의 작업 대상
        ↓
   ③ 검색결과 (랭킹화)

SEO = ② 단계 노출 효율을 높이는 모든 활동
```

→ Day 1 시리즈 **5강**. 사용자 채널 운영에 즉시 적용. 콘텐츠 자산화 직결.

## 📖 본문 (정독용)

### 챕터 1. SEO 개념 (슬라이드 1~4)

**키워드 (강의 본문)**:
- 핵심 질문: "어떻게 우리 쇼핑몰이 잘 보이게 할까?"
- 유입 경로: **검색(네이버·구글)** + **SNS 광고**
- 상위 노출 = 알고리즘 이해
- **SEO = 알고리즘** 이해

✦ **강의자 풀이 추정**
**1페이지 1~3위 = 트래픽의 70% 이상**. 11위 이하는 99% 안 봄. 따라서 모든 SEO 작업의 목표 = 상위 10위.

알고리즘이 보는 신호:
- 콘텐츠 품질 + 검색어 관련성
- 백링크 (다른 사이트 인용 수·권위)
- 사용자 행동 (클릭율·체류 시간)
- 기술적 요인 (로딩 속도·모바일 친화)
- 신뢰성 (HTTPS·도메인 권위)

→ 사용자 인스타도 같은 종류 (좋아요·체류·해시태그·저장).

**🎯 핵심 메시지**: SEO = 검색엔진 알고리즘 이해 + 그에 맞춘 최적화. 상위 1~3위가 트래픽 압도.

---

### 챕터 2. 검색엔진의 시작 (슬라이드 5~8)

**키워드 (강의 본문)**:
- **1996년** 스탠포드 대학원에서 시작
- 연구: 웹사이트 다운로드 → DB 저장 → 분류·분석
- 서버 증설 → 거대 서버 팜
- **Google** 창립 (1998): **래리 페이지·세르게이 브린**

✦ **강의자 풀이 추정**
**PageRank** = 구글의 결정적 알고리즘:
> 사이트의 중요도는 그 사이트를 링크한 다른 사이트들의 중요도로 정해진다

학술 논문의 인용 관계에서 영감. 단순한 발상이 야후·라이코스를 압도.

→ **백링크가 여전히 SEO 중요한 이유** = PageRank의 유산.

검색엔진 운영 = 거대 인프라 회사. 구글·네이버·카카오 같은 거대 기업만 가능.

**🎯 핵심 메시지**: 현대 검색 30년 안 됨. PageRank가 결정적. 백링크의 토대.

---

### 챕터 3. 검색엔진 최적화 + 3단계 ⭐ (슬라이드 9~13)

**키워드 (강의 본문)**:
- **114억 개** 웹사이트 (114억!)
- **해시태그**: SNS 검색 최적화 시도의 일종
- **사이트 직접 등록**: 네이버 Search Advisor, 구글 Search Console
- **구글 검색엔진 3단계**: 크롤링 → 색인 생성 → 검색결과 제공
- **색인 정의**: "검색 성능 속도를 높이기 위해 데이터들을 저장할 때 별도 내용을 저장하는 방법"
- **용어**: 색인(Index)·크롤링(Crawling)·크롤러(Crawler)·Googlebot·SEO

✦ **강의자 풀이 추정**
**구글 3단계 = SEO의 척추**. 이것만 외우면 SEO 절반 끝.

각 단계 SEO 작업:
- **크롤링** → 구글봇이 잘 돌게 (sitemap.xml·robots.txt)
- **색인** → 콘텐츠를 정확히 이해하게 (메타 태그·구조화 데이터)
- **검색결과** → 다른 사이트보다 위로 (콘텐츠 품질·백링크)

해시태그가 인스타·SNS SEO의 일상.

**🎯 핵심 메시지**: 검색엔진 3단계 = 크롤링·색인·검색결과. SEO 작업은 각 단계에 대응.

⚠️ **흔한 오해**: "해시태그 많을수록 좋다" → 인스타 30개까지 가능하지만 너무 많으면 스팸 분류. 5~10개가 적정.

📎 자세히: [[SEO]], [[크롤링]], [[색인]]

---

### 챕터 4. 크롤링 + 파싱 + 포켓몬 빵 비유 ⭐ (슬라이드 14~18)

**키워드 (강의 본문)**:
- **크롤링** = 컴퓨터로 웹페이지 자동으로 그대로 긁기
- 결과: HTML 통째 (다나와 가격 페이지 예시)
- **파싱** = 구문 분석. 규칙 있는 문장 잘라 분석. **크롤링뿐 아니라 프로그래밍 전반**
- 다나와 예시: `<strong>1,899,000</strong>원`에서 가격만 추출
- **포켓몬 빵 비유** ⭐:
  ```
  포켓몬 빵 → 포장·빵 분류 → 스티커만 획득
  ```
- 전체 흐름: `크롤링 → 파싱 → DB 저장`

✦ **강의자 풀이 추정**
크롤링·파싱의 본질:
- 크롤링 = 빵 통째로 사기 (HTML 전체)
- 파싱 = 포장 뜯고 스티커만 꺼내기 (필요 부분 추출)

```python
# BeautifulSoup 파싱 예시
soup = BeautifulSoup(html, 'html.parser')
price = soup.find('strong').text  # "1,899,000"
```

→ 사용자가 AI 트랙 가면 데이터 수집 작업에서 매일 등장.

**🎯 핵심 메시지**: 크롤링+파싱 = 데이터 수집의 표준. 포켓몬 빵 비유로 한 번에.

⚠️ **흔한 오해**: "robots.txt만 잘 설정하면 크롤링 못 막음" → 거짓. **신사 협정**일 뿐. 악성 크롤러는 무시. 진짜 차단은 IP·CAPTCHA·로그인.

📎 자세히: [[크롤링]], [[파싱]]

---

### 챕터 5. 크롤링 합법·불법 ⭐ (슬라이드 19~23)

**키워드 (강의 본문)**:
- 크롤링 트래픽 비용은 **크롤링 당하는 쪽이 부담** (운영비 폭증)
- **목적**: 시장가격 조사 + **AI 훈련 데이터** + 가격 비교
- **미국 (2022)**: "Scraping publicly accessible data is legal" — 항소법원 판결
- **한국**: '여기어때' 무죄 (vs 야놀자) — 정보통신망법 제48조 "정당한 접근권한"
- 사라지지 않는 이유: **데이터의 가치**

✦ **강의자 풀이 추정**
**합법·불법 경계 = "정당한 접근권한"**:
- 공개 페이지 = 일반적 합법
- 로그인 후 비공개 = 권한 우회 = 불법

→ **AI 회사들(OpenAI·Anthropic) 학습 데이터의 토대**가 이 판례. 사용자가 우리(Claude)와 대화하는 것도 그 판례 위에서.

→ 1인 컨설팅에서 클라이언트한테 "경쟁사 가격 크롤링"은 합법이라도 평판 리스크 가능.

**🎯 핵심 메시지**: 공개 데이터 크롤링 합법 (한·미 둘 다). 단 평판·저작권은 별개.

⚠️ **흔한 오해**: "공개 데이터 = 무한 사용 가능" → 거짓. **저작권은 별개**. 공개 글 통째 복사·재사용은 침해 가능.

📎 자세히: [[크롤링]] 법적 이슈

---

### 챕터 6. 공식 데이터 + 색인 (슬라이드 24~27)

**키워드 (강의 본문)**:
- 공식 데이터 채널:
  - **Google Trends** — 전 세계 검색 동향
  - **네이버 데이터랩** — 쇼핑인사이트·인기 검색어
- **색인** = 데이터 분류·표시
- **`site:naver.com` 검색 트릭** — 그 도메인의 색인 확인
- **SEO 본질**: "색인 과정에서 더 효율적 노출 작업 = 마케터의 역할"

✦ **강의자 풀이 추정**
**크롤링 대안 = 공식 채널**:
- 합법·정제·무료
- 사용자 콘텐츠 기획에 즉시 활용 (Google Trends에서 "AI 자동화" 검색 추이)
- 공공데이터포털(data.go.kr)도 동일 카테고리

**`site:내도메인` 트릭** = 색인 상태 진단의 가장 빠른 방법:
```
구글 검색창 → site:yoon-ai-lab.com
```
→ 내 사이트에서 구글이 아는 페이지만 표시.

**🎯 핵심 메시지**: 공식 데이터 = 크롤링 대안. SEO 본질 = 마케터의 색인 노출 효율 작업.

⚠️ **흔한 오해**: "색인됨 = 검색 1위" → 거짓. 색인 = 결과에 등장할 자격. 랭킹(순위)은 별개.

📎 자세히: [[색인]]

## 💡 일상 비유

### 비유 1 — 인스타 #카페 (해시태그 = SEO)

게시물 #카페·#일상·#먹스타그램 달기 = SNS SEO. 사용자가 매일 하고 있는 SEO. 단지 그게 SEO라는 자각이 없을 뿐.

### 비유 2 — 도서관 사서 (검색엔진 3단계)

사서가 ① 신간 받음 (크롤링) → ② 분류·라벨 정리 (색인) → ③ 손님이 "추리소설" 묻자 추천 (검색결과). 색인 없으면 100만 권 있어도 못 찾음.
