# 02 / 메모리 — 교재 버전

> **4섹션 구조** → [[_meta/학습-워크플로우]]

---

## 🚀 예습 (5분용)

### 한 줄 요약

**메모리 관리 = 할당·해제 + 격리**. 단편화(내부·외부) 문제 → **가상 메모리(안심 번호)** + 페이징 + 세그멘테이션으로 해결. 현대 표준은 둘 결합.

### 핵심 단어 5~7개

- [[운영체제]] — OS 통합
- **단편화** — 내부 vs 외부
- **가상 메모리** — 안심 번호 비유
- **페이지 / 페이지 테이블** — 4KB 단위 매핑
- **요구 페이징** — 필요할 때만 RAM에
- **세그멘테이션** — 의미 단위 분할
- **캐시 (Cache)** — 빠른 곳에 자주 쓰는 거 복사

### 큰 그림

```
메모리 관리 = OS의 2책임
   할당·해제 + 격리
        ↓
   문제: 단편화 (내부·외부)
        ↓
   해결: 가상 메모리 + 페이징 + 세그멘테이션
        ↓
   현대 표준 = 페이징 + 세그멘테이션 결합
```

→ Day 2 8강 중 **3강**. OS 4대 기능 두 번째.

## 📖 본문 (정독용)

### 챕터 1. 메모리 관리 + 단편화 (슬라이드 1~8)

**키워드 (강의 본문)**:
- **관리** = 할당·해제 + 리소스 보호 + 여러 프로세스 동시 실행
- 메모리 주소: 0번부터 일렬
- **연속 할당** (한 손님 = 연속 방번호) vs **불연속 할당**
- **고정 분할** = 미리 길이 정함. **외부 단편화** (공간 있어도 못 줌)
- **내부 단편화** = 할당받았지만 안 쓰는 공간
- **가변 분할** = 손님마다 다른 길이. 내부 단편화 X, 외부 단편화 여전

✦ **강의자 풀이 추정**
**두 단편화의 차이**:
- **내부**: 블록 안 낭비 (30L 박스에 작은 물건)
- **외부**: 블록 사이 빈 공간 흩어짐 (영화관 60석 비어있지만 6명 가족 같이 앉을 자리 X)

**🎯 핵심 메시지**: 메모리 관리 = 할당·해제 + 격리. 단편화 2종 (내부=블록 안 / 외부=블록 사이).

⚠️ **흔한 오해**: "남은 공간 = 즉시 사용 가능" → 외부 단편화 때문에 X.

---

### 챕터 2. 가상 메모리 ⭐ (슬라이드 9~16)

**키워드 (강의 본문)**:
- 필요 이유: 잘못된 주소에 데이터 삽입·복잡
- **정의**: 실제 물리적 주소가 아닌 **가상의 주소(논리적 주소)** 사용
- 3대 장점:
  1. **모든 프로세스가 고정된 주소**
  2. **물리 주소는 OS만 접근** → 리소스 보호
  3. **보조 기억 장치 일부를 메모리처럼 사용** ⭐
- **바인딩** = 가상↔물리 주소 변환
- **페이지 테이블** = 매핑표

✦ **강의자 풀이 추정**
**카카오톡 안심 번호 비유** (1강 복습):
- 택배기사(프로세스) = 안심번호만 봄
- 통신사(OS) = 안심번호 ↔ 진짜번호 변환
- 진짜 번호(물리 주소) = OS만 앎

진짜 위력 (장점 3):
- RAM 16GB에서 30GB짜리 작업도 (느리지만) 가능
- 자주 안 쓰는 페이지 → SSD로 잠시 (swap)

**CPU의 MMU 칩**이 변환을 매우 빠르게 처리. TLB(캐시)도 있음.

**🎯 핵심 메시지**: 가상 메모리 = 안심 번호 + SSD 활용. 3대 장점 (격리·보안·확장).

⚠️ **흔한 오해**: "SSD를 RAM처럼 쓰면 빠름" → SSD가 RAM보다 100~1000배 느림. "느려도 안 멈춤"의 의미.

---

### 챕터 3. 페이징 + 캐시 (슬라이드 17~25)

**키워드 (강의 본문)**:
- **페이지** = 가상-물리 매핑 단위. **4KB 또는 8KB**
- **페이지 테이블** = 가상 페이지 ↔ 물리 페이지 매핑표
- **스와핑** = 안 쓰는 프로세스/페이지를 디스크로
- 옛 방식: 프로세스 통째 스와핑 (비효율)
- **요구 페이징** = 페이지 단위 스와핑. 당장 필요한 페이지만 RAM
- **캐시** = 자주 쓰는 데이터를 빠른 곳에 복사. 손바닥 메모 비유
- **RAM = 디스크의 캐시** (요구 페이징 관점)
- 페이징 문제: **내부 단편화** (4KB 배수 아닌 데이터)

✦ **강의자 풀이 추정**
**메모리 계층** (보강):
```
빠름·작음 ↑  CPU 레지스터
            CPU 캐시 (L1·L2·L3)
            RAM
느림·큼  ↓  SSD / HDD
```

각 단계가 위 단계의 캐시. 요구 페이징 = RAM이 디스크의 캐시.

**🎯 핵심 메시지**: 페이지 4KB 단위 매핑. 요구 페이징 = RAM이 디스크 캐시.

⚠️ **흔한 오해**: "캐시 = 항상 빠름" → 캐시 미스 시 오히려 느림. **캐시 적중률(Hit Rate)**이 성능 키.

---

### 챕터 4. 세그멘테이션 + 결합 (슬라이드 26~29)

**키워드 (강의 본문)**:
- **세그멘테이션** = 논리적으로 같은 역할 부분끼리 자르기
- 페이징(일정 크기) vs 세그멘테이션(의미 단위)
- 장점: 코드 공유 가능 + 내부 단편화 해결
- 단점: **외부 단편화** 재발생
- **현대 표준 = 페이징 + 세그멘테이션 결합** (먼저 의미 단위 → 그 안에서 페이징)

✦ **강의자 풀이 추정**
세그먼트 = [[lectures/1주차-2일차-01-프로세스|2강]] 프로세스 4영역(코드·데이터·힙·스택)이 정확히 세그먼트의 예.

| | 페이징만 | 세그멘테이션만 | 결합 |
|--|---------|---------------|------|
| 내부 단편화 | 발생 | 해결 | 페이징 부분만 |
| 외부 단편화 | 해결 | 발생 | 해결 |
| 코드 공유 | 어려움 | 쉬움 | 쉬움 |

**🎯 핵심 메시지**: 세그멘테이션 = 의미 분할. 페이징과 결합이 현대 표준.

⚠️ **흔한 오해**: "페이징·세그멘테이션 중 하나 골라야" → 현대 OS는 결합 사용 (2단계 변환).

## 💡 일상 비유

### 비유 1 — 영화관 좌석 (외부 단편화)

영화관 100석 중 60석 비었지만 6명 가족이 같이 앉을 자리 없음. 60석이 흩어져 있어서. 전체 공간 OK, 연속 공간 X.

### 비유 2 — 책상 + 책장 (가상 메모리 SSD 활용)

지금 안 보는 책 책장으로 옮김. 다시 필요하면 책상으로. 책상(RAM) 좁아도 책장(SSD) 덕에 많은 책 작업 가능. 단 책 가지러 가는 시간 추가.
