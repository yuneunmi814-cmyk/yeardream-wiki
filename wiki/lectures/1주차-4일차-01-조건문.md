# 01 / 조건문 (진실! 혹은 거짓?) — 교재 버전

> **4섹션 구조** → [[_meta/학습-워크플로우]]
> 00강 자료형·변수 위에 **input → 형 변환 → 논리 → 분기**.

---

## 🚀 예습 (5분용)

### 한 줄 요약

**input으로 자료 받고 → 형 변환으로 가공 → 비교·논리 연산으로 판단 → if/elif/else로 분기**. 프로그램의 첫 "지능".

### 핵심 단어 5~7개

- **`input`** — 사용자 입력 받기 (항상 문자열)
- **형 변환** — `int()`·`float()`·`str()`·`list()`
- **논리 자료형** — `True` / `False`
- **비교 연산자** — `== != > < >= <=`
- **논리 연산자** — `and` · `or` · `not`
- [[조건문]] ⭐ — `if` · `elif` · `else`
- **들여쓰기** — Python 분기 구조의 핵심

### 큰 그림

```
input ───→ 문자열로 입력
   ↓
형 변환 (int·float)
   ↓
비교 연산자 (==·!=·>·<·>=·<=)
   ↓
논리 자료형 (True/False)
   ↓
논리 연산자 (and·or·not)
   ↓
조건문 (if·elif·else) → 분기 실행
```

→ Day 4 3강 중 **2강**. 1강(자료형)을 받고 2강(분기)에 활용. 다음 강(리스트)·후속(반복문)으로 이어짐.

## 📖 본문 (정독용)

> 31 슬라이드 → 4챕터.

### 챕터 1. 입력 — input() (슬라이드 1~8)

**키워드 (강의 본문)**:
> input — 사용자가 컴퓨터에 정보 전달
> `변수 = input()` — 터미널에 입력 + Enter
> 무엇을 입력하든 **문자열로 입력 받아짐** ⚠️
> 가공 필요 — 자료형 변환(형 변환)으로
> 형 변환: int·float·str·list

✦ **강의자 풀이 추정** (LLM 보강)

**`input()` = 사용자 입력 받기**:

```python
var = input("숫자를 입력하세요 : ")
# 터미널: 숫자를 입력하세요 : 10
print(var)              # 10
print(type(var))        # <class 'str'>   ← 문자열!
```

⚠️ **가장 흔한 함정** ⭐: `input`은 **항상 문자열 반환**. 숫자 입력해도 `"10"` (str).

```python
var1 = input("숫자: ")    # 100 입력
var2 = input("숫자: ")    # 300 입력
print(var1 + var2)        # 100300   ← 문자열 이어붙이기!
```

→ 더하기 하려면 **형 변환** 필요.

**형 변환 (Type Casting)**:

| 변환 | 함수 | 예시 | 결과 |
|---|---|---|---|
| 정수로 | `int()` | `int("1008")` | `1008` |
| 실수로 | `float()` | `float("3.14")` | `3.14` |
| 문자열로 | `str()` | `str(3.14)` | `"3.14"` |
| 리스트로 | `list()` | `list("1008")` | `["1","0","0","8"]` |

```python
var1 = "8"
var2 = "1000"

print(var1 + var2)              # 81000  (문자열)
print(int(var1) + int(var2))    # 1008   (정수 덧셈)
```

**`type()` — 자료형 확인**:
```python
a = "345"
b = int("345")

print(type(a))   # <class 'str'>
print(type(b))   # <class 'int'>
```

→ 디버깅의 첫 도구. "이거 왜 안 되지?" 할 때 `type()` 찍어봄.

**🎯 핵심 메시지**: `input`은 **무조건 문자열**. 숫자로 쓸 거면 **반드시 형 변환**.

---

### 챕터 2. 논리형 자료와 비교 연산 (슬라이드 9~18)

**키워드 (강의 본문)**:
> 논리 자료형 (Boolean): True·False만 존재
> 비교 연산자 — `==` 같다, `!=` 다르다, `>` `<` `>=` `<=`
> 논리 연산자: AND (모두 True) · OR (하나라도 True) · NOT (반대)

✦ **강의자 풀이 추정** (LLM 보강)

**논리 자료형** (`bool`) — Python의 4번째 자료형:

```python
print(True)         # True
print(False)        # False
print(type(True))   # <class 'bool'>
```

⚠️ **주의**: 첫 글자 대문자. `true`(소문자)는 에러.

**비교 연산자 6종** ⭐:

| 연산자 | 의미 | 예시 | 결과 |
|---|---|---|---|
| `==` | **같다** | `3 == 3` | True |
| `!=` | 다르다 | `3 != 3` | False |
| `>` | 왼쪽이 크다 | `3 > 5` | False |
| `<` | 오른쪽이 크다 | `3 < 5` | True |
| `>=` | 왼쪽이 크거나 같다 | `3 >= 3` | True |
| `<=` | 오른쪽이 크거나 같다 | `3 <= 5` | True |

⚠️ **가장 흔한 오해** ⭐⭐: `=` vs `==`
- `=` — **할당** (오른쪽 값을 왼쪽 그릇에)
- `==` — **비교** ("같은가?")

```python
x = 5          # 할당
x == 5         # True (비교)
```

**논리 연산자 3종** — 여러 비교를 결합:

```python
# AND : 모두 True 인가요?
print(3 == 3 and 4 <= 5 and 6 > 2)
# True and True and True → True

# OR : 하나라도 True 인가요?
print(3 == 4 or 4 <= 5 or 6 < 2)
# False or True or False → True

# NOT : 반대로
print(not 3 == 4)
# not False → True
```

**진리표 정리**:

| A | B | A and B | A or B |
|---|---|---|---|
| T | T | T | T |
| T | F | F | T |
| F | T | F | T |
| F | F | F | F |

**🎯 핵심 메시지**: `=` 는 **할당**, `==` 는 **비교**. 논리 연산자로 여러 조건 결합.

⚠️ **흔한 오해**: "and·or 대신 `&·|` 써도 됨". Python에서 `&·|`는 **비트 연산자** (다른 용도). 논리는 반드시 `and·or·not`.

---

### 챕터 3. 조건문 — if (슬라이드 19~25)

**키워드 (강의 본문)**:
> 조건문 = 특정 조건에 따라 실행되는 명령이 달라지는 구문
> `if 조건 :` 다음 줄에 들여쓰기로 명령
> 조건이 True일 때만 명령 실행

✦ **강의자 풀이 추정** (LLM 보강) — [[조건문]] 페이지 ⭐

**`if` 문 = "만약 ~라면"**:

```python
if 조건 :
    <수행할 명령>
    <수행할 명령>
    ...
```

⚠️ **Python 핵심 문법**: **들여쓰기로 블록 구분** (다른 언어는 `{}` 사용).

```python
i = 1

if i == 1 :
    print(i)        # 들여쓰기 (보통 스페이스 4칸 또는 탭)
# 출력: 1
```

⚠️ **흔한 함정**:
- 콜론 `:` 빠짐 → SyntaxError
- 들여쓰기 깜빡 → IndentationError
- 들여쓰기 안 맞춤 → IndentationError

→ Python은 **들여쓰기가 문법의 일부**. SwiftUI/JavaScript와 큰 차이.

**복잡한 조건 (논리 연산자 활용)**:
```python
age = 25
if age >= 18 and age < 65 :
    print("성인입니다")
```

**🎯 핵심 메시지**: `if 조건 :` + **들여쓰기**가 Python 분기의 골격.

---

### 챕터 4. else / elif (슬라이드 26~31)

**키워드 (강의 본문)**:
> if-else — 조건 True면 if, False면 else
> if-elif — 여러 조건 차례로 검사
> elif는 if 다음, else 앞에. 여러 개 가능

✦ **강의자 풀이 추정** (LLM 보강)

**`if-else` = "그렇지 않으면"**:

```python
if 조건 :
    <수행할 명령>
else :
    <수행할 명령>
```

```python
i = 1

if i == 1 :
    print("일")
else :
    print("일이 아님")
# 출력: 일
```

**`if-elif-else` = "아니면 ~라면"** — 여러 갈래:

```python
if 조건1 :
    <수행할 명령>
elif 조건2 :
    <수행할 명령>
elif 조건3 :
    <수행할 명령>
else :
    <수행할 명령>
```

**성적 예시**:
```python
score = 78

if score >= 90 :
    print("A")
elif score >= 75 :
    print("B")      # ← 여기 실행
elif score >= 60 :
    print("C")
else :
    print("F")
```

**실행 로직**:
- 조건1 True → A 실행, **이후 elif·else 건너뜀**
- 조건1 False + 조건2 True → B 실행, 이후 건너뜀
- 모든 조건 False → else 실행

⚠️ **흔한 오해**: "if 여러 개 = elif랑 같다". 다름.
```python
# A: if 여러 개 (각각 독립 검사)
if score >= 90 : print("A")
if score >= 75 : print("B")    # 90점이면 A·B 둘 다 출력

# B: elif (배타적)
if score >= 90 : print("A")
elif score >= 75 : print("B")  # 90점이면 A만 출력
```

→ B처럼 **배타적 분기**가 필요할 땐 elif.

**🎯 핵심 메시지**: `if-elif-else`는 **위에서 아래로 첫 번째 True**만 실행. 배타적.

## 💡 일상 비유

### 비유 1: `input()` = 손님 주문

- 카페에서 손님이 뭐라 말하든 **점원은 일단 받아 적음** (input은 무조건 문자열)
- "아메리카노 4잔" → 4잔은 문자, 계산하려면 `int("4")`로 가공
- → 형 변환 안 하면 "4 + 5 = 45" (이어붙이기)

### 비유 2: `if-elif-else` = 알람 시계 메뉴

- "오전 6시 ?" → 일어나기 (if)
- "아니면 오전 9시 ?" → 운동 (elif)
- "그도 아니면 ?" → 자유 (else)

조건은 **위에서 아래**로 순서대로 확인. 한 번 맞으면 끝.

### 비유 3: 들여쓰기 = 메뉴 단계

- 일반 줄 = 메인 메뉴
- 들여쓰기 = 서브 메뉴 (`if` 안의 내용)
- 들여쓰기 풀면 = `if` 끝, 메인 메뉴 복귀

```python
if 조건 :
    들여쓰기 안 → if 안 (실행 대상)
    들여쓰기 안 → if 안
들여쓰기 풀면 → if 끝 (밖 코드)
```
