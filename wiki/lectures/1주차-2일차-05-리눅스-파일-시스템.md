## 🚀 예습 (5분용)

### 한 줄 요약

**리눅스 = / 단일 트리**. `tree` 시각화, **권한 9자(rwx×3)** + `chmod`·`chown`, **파일 명령어** (mkdir·rm·cp·mv·cat·find), **nano** 에디터.

### 핵심 단어 5~7개

- [[운영체제]] — 파일 시스템 이론 연결
- **트리 구조** — / 아래 디렉토리
- **권한 9자 rwx×3** — 소유자·그룹·기타
- **chmod·chown** — 권한·소유자 변경
- **mkdir·touch·rm·cp·mv·cat·find** — 파일 명령어
- **rm -rf** ⚠️ — 휴지통 없음
- **nano** — CLI 에디터

### 큰 그림

```
리눅스 파일 시스템:
   / (루트)
   ├── home/사용자/
   ├── etc/ (설정)
   ├── usr/bin/ (명령어)
   ├── var/log/ (로그)
   └── tmp/

각 파일/폴더:
   권한(rwx×3) + 소유자 + 그룹 + 크기 + 시간 + 이름
        ↓
   chmod / chown 으로 변경
```

→ Day 2 8강 중 **6강**. 이론 4강(파일 시스템) + 입문 5강 합쳐 실무 작업.

## 📖 본문 (정독용)

### 챕터 1. 파일 시스템 + 디렉토리 구조 (슬라이드 1~8)

**키워드 (강의 본문)**:
- 파일 시스템 = 저장장치 내 데이터 읽고 쓰는 약속
- 리눅스 = **루트(/) 아래 계층적 트리**
- 파일 = 데이터 묶음 / 디렉토리 = 분류 이름 공간
- **트리(tree) 명령**으로 시각화. 없으면 `sudo apt install tree`

✦ **강의자 풀이 추정**
**모든 게 / 아래 단일 트리** = Windows의 C:·D:·E: 분리와 차이.

표준 디렉토리:
- `/home/사용자명` = 개인 홈
- `/etc` = 시스템 설정
- `/usr/bin` = 일반 명령어
- `/var/log` = 로그
- `/tmp` = 임시

→ macOS도 비슷 (`/Users/yoon`). 한 번 익히면 둘 다 OK.

**🎯 핵심 메시지**: 리눅스 = / 단일 트리. tree 명령으로 시각화.

📎 **실습 ✨**: 터미널에서 `tree -L 2 ~` (홈 폴더 2단계까지). macOS는 `brew install tree`.

---

### 챕터 2. 소유권과 권한 ⭐ (슬라이드 9~13)

**키워드 (강의 본문)**:
- `ls -al` 9칸: 유형·권한·링크·소유자·그룹·크기·시간·이름
- 파일 유형: `-`(파일), `d`(디렉토리), `l`(링크) 등
- **권한 9자 = rwx × 3그룹** (소유자·그룹·기타)
- 권한 숫자: r=4, w=2, x=1. 합산
- **chmod**: `chmod 777 elice.txt` = 모든 사용자 모든 권한
- **chown**: 소유자·그룹 변경. **root 권한 필요**

✦ **강의자 풀이 추정**
**권한 9자 분해**:
```
rwx | rwx | rwx
↑     ↑     ↑
소유자  그룹  기타
```
- 7 = rwx (4+2+1)
- 6 = rw- (4+2)
- 5 = r-x (4+1)
- 4 = r-- (4)

표준:
- 파일: 644 (소유자 rw, 나머지 r)
- 디렉토리: 755 (소유자 rwx, 나머지 r-x)
- 셸 스크립트: 755 (실행 권한 추가)

**문자 표기**:
```bash
chmod u+x script.sh   # 소유자 실행 권한 추가
chmod g-w file        # 그룹 쓰기 제거
chmod a+r file        # 모두 읽기
```

**🎯 핵심 메시지**: 권한 9자 = rwx×3. r=4·w=2·x=1 숫자 합산. chmod로 변경.

⚠️ **흔한 오해**: "777이 좋은 권한" → 거짓. **777 = 누구나 다 만짐 = 보안 최악**. 일반은 644·755.

📎 **실습 ✨**: 빈 파일 만들고 `chmod 644 test.txt`·`ls -l test.txt` 확인.

---

### 챕터 3. 파일 명령어 ⭐ (슬라이드 14~26)

**키워드 (강의 본문)**:
- `mkdir 폴더` — 폴더 생성. `mkdir -p a/b/c`로 중간 자동 생성
- `touch 파일` — 빈 파일 생성 (또는 시간 갱신)
- `rmdir 폴더` — 빈 폴더만 삭제
- `rm` — 파일·폴더 삭제
  - `-r`: 재귀 (폴더 안 다)
  - `-f`: 강제 (확인 X)
  - `-rf`: **가장 위험**
- `cp` — 복사. `-r` 재귀, `-p` 메타데이터 보존
- `mv` — 이동 + 이름 변경
- `cat 파일` — 내용 출력
  - `>`: 덮어쓰기 / `>>`: 이어쓰기
- `find` — 강력한 검색
- `head·tail·diff·which·wc` 등

✦ **강의자 풀이 추정**
실무 자주 쓰는 형태:
```bash
mkdir -p ~/projects/yoon/new-app   # 깊은 경로 한 줄
touch app.py                        # 새 파일
rm -rf old-folder/                  # 폴더 통째 삭제 (위험)
cp -r src/ backup/                  # 폴더 백업
mv old.txt new.txt                  # 이름 변경
cat file.txt | grep "error"         # 파이프 (다음 강)
find . -name "*.py"                 # 모든 .py
tail -f log.txt                     # 실시간 로그 추적
```

**🎯 핵심 메시지**: 파일 명령어 7대 = mkdir·touch·rm·cp·mv·cat·find. `rm -rf` 위험.

⚠️ **흔한 오해 / 위험**:
- `rm -rf /` → **시스템 전체 삭제. 절대 금지**
- `rm -rf $variable` → 변수 비어있으면 위 같은 효과
- Linux의 `rm`은 **휴지통 없음**. 즉시 영구

📎 **실습 ✨**: 테스트 폴더에서 안전하게 시도. `mkdir test && cd test && touch a.txt && ls && cd .. && rm -rf test`.

---

### 챕터 4. nano editor (슬라이드 27~29)

**키워드 (강의 본문)**:
- **nano** = UNIX 호환 간단 텍스트 에디터
- `nano 파일명`으로 실행
- 단축키 (Ctrl+):
  - `O`: 저장
  - `X`: 종료
  - `W`: 검색

✦ **강의자 풀이 추정**
CLI 에디터 3대장:
- **nano**: 초보자 친화. 단축키 화면 표시
- **vim**: 강력하지만 학습곡선 가파름
- **emacs**: 거대 에디터

설정 파일 빠르게 수정할 때 nano. 사용자가 Claude Code 일상이면 nano 거의 안 만남.

**🎯 핵심 메시지**: nano = 초보 친화 CLI 에디터. Ctrl+O 저장, Ctrl+X 종료.

📎 **실습 ✨**: `nano test.txt` 열고 글자 입력 → `Ctrl+O` 저장 → `Ctrl+X` 종료.

## 💡 일상 비유

### 비유 1 — 인스타 공개 설정 (권한)

- "전체 공개" = 모든 사용자 r (읽기)
- "친구만" = 그룹 r
- "비공개" = 소유자만 r

Linux 권한 모델이 정확히 그 구조.

### 비유 2 — macOS 휴지통 없는 rm (rm -rf 위험)

macOS에서 ⌘+Delete = 휴지통으로 (복구 가능).
Linux `rm` = 영구 삭제, 휴지통 없음. 그래서 `rm -rf`는 신중.

## 이해 체크 (은미, 2026-05-29)

- [x] **트리 구조** — 폴더 안의 폴더 안의 폴더. 최상위 = `/` (루트 디렉토리). 모든 폴더는 / 아래
- [x] **권한 9자리 = rwx × 3** (소유자·그룹·기타)
  - **r** = 읽기 (Read)
  - **w** = 쓰기 (Write)
  - **x** = **실행 (eXecute)** — 단순 텍스트 아니라 프로그램으로 실행 가능
- [x] **소유자·그룹·기타** = 가족 비유
  - **소유자** = 파일 만든 주인 (보통 rwx 다)
  - **그룹** = 같은 팀·형제들
  - **기타 (Others)** = 외부 손님
- [x] **rwxrwxrwx = chmod 777** = 모두에게 다 권한 (위험·실무 거의 ❌)
- [x] **그룹 만들고 사용자 추가**:
  - `sudo groupadd dev_team` (그룹 생성)
  - `sudo usermod -aG dev_team chulsoo` (철수를 그룹에 추가)
  - **usermod** = User Modification
  - **-aG** = **a**ppend + **G**roup (기존 그룹 유지 + 새 그룹 추가) ⭐ 너 직감 정답
- [x] **chmod (Change Mode)** — 권한 바꾸기
  - 숫자 = r4 + w2 + x1 합산
  - **755** = 주인 rwx(7) + 그룹 r-x(5) + 기타 r-x(5) → 프로그램용
  - **644** = 주인 rw-(6) + 그룹 r--(4) + 기타 r--(4) → 일반 문서·웹 파일
  - **777** = 모두 rwx (위험)
- [x] **chown (Change Owner)** — 소유자 바꾸기
  - `sudo chown chulsoo report.pdf` (퇴사한 사람 파일 받기)
  - 보통 sudo 같이 씀
- [x] **명령어 순서 = 영어 문장 (동사+조건+목적어)** ⭐
  - `chmod (바꿔) 755 (이렇게) mygame.sh (이거를)`
  - `chown chulsoo report.pdf`
