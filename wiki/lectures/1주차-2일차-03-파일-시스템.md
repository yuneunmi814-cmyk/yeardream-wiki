## 🚀 예습 (5분용)

### 한 줄 요약

**파일 = 논리적 저장 단위**. OS의 디스크 추상화(NTFS·APFS·ext4). 할당 4방식, **inode·심볼릭/하드 링크**, **마운트**.

### 핵심 단어 5~7개

- [[운영체제]] — OS 통합
- **파일 / 디렉토리 / 메타데이터**
- **HDD vs SSD** — 회전 기계 vs 전기
- **할당 4방식** — 연속·연결·FAT·색인
- **inode** — 리눅스 파일 고유 ID
- **심볼릭 vs 하드 링크** — 경로 vs 같은 inode
- **마운트** — 디스크 ↔ 디렉토리 연결

### 큰 그림

```
파일 시스템 = OS의 디스크 추상화
   ↓
디스크 ─ 블록 단위 ─ 할당 4방식 ─ (연속·연결·FAT·색인)
   ↓
리눅스 inode (색인 방식) → 링크 (심볼릭·하드)
   ↓
마운트 = 디스크를 디렉토리에 연결
```

→ Day 2 8강 중 **4강 (OS 이론 마지막)**. OS 4대 기능 세 번째.

## 📖 본문 (정독용)

### 챕터 1. 파일·디렉토리·파일 시스템 (슬라이드 1~7)

**키워드 (강의 본문)**:
- **파일** = 정보 담는 **논리적 저장 단위**. 비휘발성 기억장치에 저장
- **메타데이터**: 이름·유형·위치·크기·권한·시간
- **디렉토리** = 파일에 대한 정보 가진 **특수한 파일**
- **파일 시스템** = 파일을 쉽게 접근하도록 관리하는 체제. 디스크·네트워크·특수 용도

✦ **강의자 풀이 추정**
**디렉토리도 파일**. Unix·Linux 철학: **"모든 것은 파일이다"**. 디렉토리·디바이스·파이프 모두 파일.

현대 파일 시스템:
- **NTFS** (Windows), **APFS** (macOS), **ext4** (Linux), **exFAT** (USB 호환)
- 네트워크 FS: NFS·SMB·iCloud Drive·Google Drive

→ 사용자 맥북 = APFS. 매일 사용.

**🎯 핵심 메시지**: 파일 = 논리적 단위. 디렉토리도 파일(Unix 철학). NTFS·APFS·ext4가 대표.

---

### 챕터 2. 저장장치 + 디스크 할당 (슬라이드 8~18)

**키워드 (강의 본문)**:
- **순차 접근** (카세트 테이프) vs **직접 접근** (현대 표준)
- **HDD**: 플래터·헤드·트랙·섹터(최소 단위)
- **SSD**: 전기 장치만. 모터 없어 **훨씬 빠름**
- 디스크 = **블록** 단위
- **할당 4방식**:
  1. **연속 할당** — 옆자리. 빠르지만 외부 단편화
  2. **연결 할당** (연결리스트) — 화살표 따라가기. 외부 단편화 X, 직접 접근 X, 손상 시 이후 못 읽음
  3. **FAT** — 화살표 표 모아둠. 직접 접근 OK. **FAT16·FAT32·exFAT**
  4. **색인 할당** — 파일마다 인덱스 블록

✦ **강의자 풀이 추정**
HDD vs SSD:

| | HDD | SSD |
|--|-----|-----|
| 구조 | 회전 플래터 + 헤드 | 전자 셀 |
| 속도 | 느림 | 100~1000배 빠름 |
| 충격 | 약함 | 강함 |

→ 사용자 맥북 = SSD. 부팅 10초.

**FAT 발전사**: FAT16 → FAT32(4GB 제한) → exFAT(64bit, 현재 USB 표준). 사용자 USB 메모리가 exFAT.

→ 다음 챕터 **색인 할당** = 리눅스 inode의 토대.

**🎯 핵심 메시지**: HDD 옛(회전) vs SSD 현대(전기). 할당 4방식 — FAT이 USB 표준, 색인이 리눅스.

⚠️ **흔한 오해**: "현대 컴퓨터는 다 HDD" → 거짓. **SSD 표준**. HDD는 백업·일부 서버.

---

### 챕터 3. 리눅스 inode + 링크 ⭐ (슬라이드 19~24)

**키워드 (강의 본문)**:
- **inode** (index-node) = 파일의 **모든 정보 기록**. 파일·디렉토리마다 **고유 inode**
- 담는 정보: 유형·권한·시간·블록 위치
- **바로가기** — 삭제해도 원본 안 사라짐
- **심볼릭 링크 (Symbolic Link)**: 링크 → **원본 경로** → 원본
  - **원본 삭제·이동 시 링크 깨짐**
  - 별도 inode 가짐
- **하드 링크 (Hard Link)**: 링크가 **같은 inode** 가리킴
  - **원본 삭제해도 사용 가능**
  - inode 공유

명령어:
```bash
ln -s [원본] [심볼릭 링크]   # 심볼릭
ln [원본] [하드 링크]         # 하드
```

✦ **강의자 풀이 추정**
**심볼릭 vs 하드**:

| | 심볼릭 | 하드 |
|--|--------|------|
| 저장 | 원본 경로 | 같은 inode |
| 원본 삭제 시 | 깨짐 | 안전 |
| 다른 FS 가로지름 | 가능 | 불가 |
| 별도 inode | 가짐 | 공유 |

**활용**:
- AI 데이터셋 (수십 GB) → 복사 X, 심볼릭 링크로 여러 프로젝트 공유
- 시스템 라이브러리 → 여러 위치 참조

**🎯 핵심 메시지**: inode = 리눅스 파일 고유 ID. 심볼릭(경로) vs 하드(같은 inode).

⚠️ **흔한 오해**: "파일명이 inode에 있다" → 거짓. **파일명은 디렉토리가 보유**. inode는 메타데이터·블록 위치만.

---

### 챕터 4. 마운트 (슬라이드 25~34)

**키워드 (강의 본문)**:
- **마운트** = 물리적 저장장치를 **특정 디렉토리에 연결**
- 마운트 안 하면 OS가 인식 못 함
- 대상: 하드디스크·USB·CD/DVD·플로피
- 자동 마운트 (Windows·macOS)
- 리눅스: **수동 마운트** 필요 (서버)
- Windows = 드라이브 문자(C:·D:·E:)
- 리눅스 = `/` 아래 디렉토리 트리. `/mnt/disk1` 같은 식
- `lsblk` 명령으로 장치 목록
- `sudo mount [device] [directory]` 명령

✦ **강의자 풀이 추정**
**모든 디스크가 / 아래 통일 트리** = 리눅스 철학. Windows의 분리된 드라이브와 차이.

→ 사용자 맥북도 비슷. 외장 디스크는 `/Volumes/외장디스크`. `diskutil list`로 확인.

→ Day 2 [강의자료] 06강 ([[lectures/1주차-2일차-06-리눅스-명령어|마운트 명령어]])에서 본격 실습.

**🎯 핵심 메시지**: 마운트 = 디스크 ↔ 디렉토리 연결. 리눅스는 수동, macOS·Windows는 자동.

⚠️ **흔한 오해**: "USB 그냥 빼도 됨" → 거짓. 마운트 해제(umount·꺼내기) 필수. 안 하면 데이터 손상 가능.

## 💡 일상 비유

### 비유 1 — LP 레코드 (HDD)

LP 음원 들을 때 바늘(헤드)이 회전 LP(플래터) 위 트랙 따라 읽음. HDD가 정확히 그 구조의 디지털 버전.

### 비유 2 — 한 사람의 별명들 (하드 링크)

"엘리서·은미·yoon" 모두 한 사람 부르는 다른 이름. 어떤 이름으로 부르든 같은 사람. 하드 링크 = 같은 inode 다른 이름.

## 이해 체크 (은미, 2026-05-29)

- [x] **파일 = 알맹이 / 디렉토리(폴더) = 상자 / 메타데이터 = 견출지** (크기·생성일·권한·위치)
- [x] **HDD vs SSD = 구식 냉장고(일일이 찾기) vs 최신형(바로 꺼냄)** ⭐ 너 비유
- [x] **할당 4방식** (디스크에 파일 배치 전략):
  - **연속** = 기차 이어붙이기 (빠름·중간 삭제 시 빈 공간)
  - **연결** = 보물찾기 (다음 조각 위치 가리킴·중간 고장 시 뒤 못 찾음)
  - **FAT** = 네비게이션·전체 안내 지도 (USB·다음 조각 다 적힌 테이블)
  - **색인 (Index)** = 치트키 힌트 (모든 조각 위치 한 장에 다 적힘·직접 접근 빠름)
- [x] **inode = 파일의 주민등록번호** ⭐ — 시스템이 이름이 아닌 번호로 식별
  - inode 안에 메타데이터·디스크 주소 다 들어 있음
  - `ls -i` 로 inode 번호 확인 가능
- [x] **심볼릭 링크 (Symlink) = 바로가기 아이콘** — 경로만 적어둠. 원본 지우면 깨짐
- [x] **하드 링크 = 하나의 영혼 + 두 개 육체** ⭐ — 같은 inode, 다른 이름. 한쪽 지워도 다른쪽 살아있으면 데이터 보존
- [x] **하드 링크 사용 예**: 10GB 영화 → 2개 폴더에 하드 링크 = 용량 그대로 10GB (복사 아님)
- [x] **하드 링크 제약**: 폴더 ❌, 다른 드라이브 ❌ (면접 단골)
- [x] **`ln 원본 새이름`** 명령어로 하드 링크 생성. `ln -s` 면 심볼릭
- [x] **마운트 = 디스크를 폴더에 연결 선언** (복사 ❌). 리눅스는 드라이브 개념 없고 모든 게 폴더 트리
