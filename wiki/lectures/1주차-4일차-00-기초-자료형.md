## 🚀 예습 (5분용)

### 한 줄 요약

**Python 시작 = 출력(print) + 자료형 3종(숫자·문자열·리스트) + 변수 + 연산자 + 인덱싱/슬라이싱**. 손코딩의 ABC.

### 핵심 단어 5~7개

- [[Python]] ⭐⭐ — 강의 전체의 무대
- [[변수]] ⭐ — 자료를 담는 "그릇"
- [[자료형]] — 숫자(int·float)·문자열(str)·리스트(list)
- **`print`** — 출력 함수
- **연산자** — `+ - * / // % **`
- **인덱싱·슬라이싱** — 0부터 시작, `[a:b]` 범위

### 큰 그림

```
컴퓨터 ───print()───→ 사용자       (출력)

자료형 (3종):
  숫자형    3, 3.14, 3+4j
  문자열    "Hello", '안녕'
  리스트    [], ['a', 'b', 2]

변수 = 자료 그릇:
  num  = 10
  name = "Michael"

연산자:
  +  -  *  /        (사칙)
  // %  **           (몫·나머지·제곱)
  + (문자열 이어붙이기)  * (반복)

인덱싱·슬라이싱:
  "Ready"
   01234
  [a:b]  → a 이상 b 미만
```

→ Day 4 3강 중 **1강**. 다음 강에서 input + 조건문 → 자료를 받고 분기.

### 챕터 1. 출력 — print (슬라이드 1~6)

**키워드 (강의 본문)**:
> Python에서는 우리가 원하는 정보나 자료를 컴퓨터가 출력
> `print(출력할 내용)` 형태로 출력
> 콤마(`,`)로 여러 자료 출력
> print 여러 번 = 여러 줄 출력

✦ **강의자 풀이 추정** (LLM 보강)

**`print` = Python의 첫 함수**. 화면에 글자 띄움.

```python
print("나의 꿈은 파이썬 정복!")
# 출력: 나의 꿈은 파이썬 정복!

print(3, "Hello")         # 콤마로 여러 개
# 출력: 3 Hello   (콤마는 공백으로 출력)

print("줄 1")
print("줄 2")             # 줄 바꿈
```

⚠️ **흔한 함정**: `print` 안 쓰면 출력 안 됨. Jupyter Notebook은 마지막 줄 자동 출력하지만, 스크립트는 반드시 `print`.

**🎯 핵심 메시지**: print는 컴퓨터와 사용자를 잇는 첫 다리.

### 챕터 2. 기본 자료형 (슬라이드 7~12)

**키워드 (강의 본문)**:
> 숫자형 (Number): 정수(integer)·실수(float)·복소수
> 문자열 (String): 큰 따옴표·작은 따옴표
> 리스트 (List): 여러 자료 보관, 다른 종류 함께, 순서 존재
> 주석 (Comment): `#`(한 줄)·`""" """`(여러 줄)

✦ **강의자 풀이 추정** (LLM 보강) — [[자료형]] 페이지 참조

**자료형 3종**:

| 자료형 | 예시 | 영문명 | 특징 |
|---|---|---|---|
| **숫자형** | `3`, `3.14`, `3+4j` | `int`·`float`·`complex` | 산술 연산 가능 |
| **문자열** | `"Hello"`, `'안녕'` | `str` | 따옴표로 감쌈 |
| **리스트** | `[]`, `['a', 2]` | `list` | 여러 자료, 순서 있음 |

**숫자형 3종**:
```python
3       # int (정수)
3.14    # float (실수)
3+4j    # complex (복소수, 거의 안 씀)
```

**문자열 — 큰/작은 따옴표 둘 다 OK**:
```python
'Hello!'   # 작은 따옴표
"Hello!"   # 큰 따옴표
'3.14'     # 숫자도 따옴표 안에 들어가면 문자열
```

⚠️ **흔한 오해**: `"3.14"` ≠ `3.14`. 전자는 **문자열**, 후자는 **실수**. 연산 결과 완전히 다름.

**리스트 — 여러 자료를 한 그릇에**:
```python
[]                    # 빈 리스트
['a', 'b', 'c']       # 문자열들
['a', 2]              # 다른 자료형 섞여도 OK
[1, [2, 3], 'hi']     # 리스트 안에 리스트도 OK (중첩)
```

→ 자세한 메서드는 [[리스트]] 페이지 / [[1주차-4일차-02-리스트]] 강의.

**주석 — 컴퓨터가 무시하는 메모**:
```python
# 한 줄 주석

"""
여러 줄 주석
컴퓨터는 무시
"""
```

→ 사용자 본인이나 동료가 코드 이해하기 쉽게 적는 메모. 실행에는 영향 ❌.

**🎯 핵심 메시지**: Python 자료형 3종 = **숫자·문자열·리스트**. 가장 자주 쓰는 트리오.

### 챕터 3. 변수 (슬라이드 13~21)

**키워드 (강의 본문)**:
> 변수(Variable) = 자료를 "그릇"에 담아 보관·사용
> `변수이름 = 자료`
> 이름: 숫자·알파벳·한글·언더바(`_`) 가능
> 표기법: 스네이크 (`prod_name`) / 카멜 (`prodName`)
> 주의점 4가지: 숫자 시작 ❌, 숫자만 ❌, 예약어 ❌, 공백/연산자 ❌

✦ **강의자 풀이 추정** (LLM 보강) — [[변수]] 페이지 ⭐

**변수 = 자료의 그릇**:

```python
num = 10                  # 숫자 그릇
name = "Michael"          # 문자열 그릇
grade = ['A+', 'B+']      # 리스트 그릇

print(num)        # 10
print(name)       # Michael
print(grade)      # ['A+', 'B+']
```

⚠️ **흔한 오해**: `=` 는 "같다" ❌. **"할당"** (오른쪽 값을 왼쪽 그릇에 넣음). "같다"는 다음 강 `==`.

**변수 이름 규칙 4가지** ⚠️:

```python
# 1. 숫자로 시작 ❌
1name = "Coke"            # SyntaxError

# 2. 숫자만 구성 ❌
1234 = 5678               # SyntaxError

# 3. 예약어 사용 ❌
print = "Hello"           # 가능은 하지만 print() 함수 망가짐
                          # for, if, def 등도 마찬가지

# 4. 공백·연산자 ❌
prod name = "milk"        # 공백 안 됨
prod+date = "..."         # +, -, % 등 안 됨
```

**표기법 2종** (강의 권장: 알파벳 + 언더바):

```python
# 스네이크 표기법 (Python 표준)
prod_name = "milk"
prod_date = "2026-10-08"

# 카멜 표기법 (Swift·JavaScript 흔함)
prodName = "milk"
prodDate = "2026-10-08"
```

→ **Python 공식 권장 = 스네이크**. 사용자 SwiftUI 경험상 카멜에 익숙 → 의식적으로 스네이크 연습 필요.

**🎯 핵심 메시지**: 변수는 **자료의 그릇**. `=` 는 할당. 이름 규칙 4가지 위반 ❌.

### 챕터 4. 자료형의 연산 (슬라이드 22~26)

**키워드 (강의 본문)**:
> 사칙연산: `+` `-` `*` `/`
> 특수연산: `//` 몫·`%` 나머지·`**` 제곱
> 문자열 연산: `+` 이어붙이기·`*` 반복

✦ **강의자 풀이 추정** (LLM 보강)

**수학형 자료 연산**:

```python
print(3 + 5)    # 8
print(3 - 5)    # -2
print(3 * 5)    # 15
print(3 / 5)    # 0.6      ← 나눗셈은 항상 float

print(13 // 5)  # 2        ← 몫 (정수 나누기)
print(13 % 5)   # 3        ← 나머지
print(2 ** 4)   # 16       ← 제곱 (2의 4승)
```

⚠️ **흔한 함정**: `/`와 `//` 차이.
- `3 / 5` → `0.6` (실수)
- `3 // 5` → `0` (몫만)

**문자열 자료 연산**:

```python
print("안녕" + "하세요")    # 안녕하세요   (이어붙이기)
print("안녕" * 3)           # 안녕안녕안녕  (반복)
```

⚠️ **흔한 함정**: 문자열 + 숫자는 **에러**.
```python
print("3" + 5)              # TypeError
print(int("3") + 5)         # 8   ← 형 변환 (다음 강)
```

**🎯 핵심 메시지**: 같은 `+` 기호라도 **숫자엔 더하기·문자열엔 이어붙이기**. 자료형 따라 동작 다름.

### 챕터 5. 문자열/리스트 접근하기 — 인덱싱·슬라이싱 ⭐ (슬라이드 27~33)

**키워드 (강의 본문)**:
> 문자열·리스트는 여러 원소, 위치를 0부터 매김
> 인덱싱 — `String/List[index]`로 특정 위치 원소
> 슬라이싱 — `String/List[a:b]`로 일부분 가져오기 (a 이상 b 미만)

✦ **강의자 풀이 추정** (LLM 보강)

**인덱스 = 위치 번호. 0부터 시작** ⭐:

```
문자열:  "Ready"
         01234       ← 인덱스

리스트:  [2, 4, 6, 8]
          0  1  2  3
```

⚠️ **흔한 오해**: "첫 번째 원소 = 1번 인덱스". 틀림. **0번**부터.

**인덱싱 — 한 원소 꺼내기**:

```python
alpha = "Ready"
print(alpha[0])         # R
print(alpha[1])         # e
print(alpha[4])         # y
```

**음수 인덱스 — 뒤에서부터** (02강에서 풀이):
```python
print(alpha[-1])        # y   (마지막)
print(alpha[-2])        # d
```

**슬라이싱 — 범위로 꺼내기**:

```python
beta = [2, 4, 6, 8, 10, 12, 14]
print(beta[2:5])        # [6, 8, 10]   ← 2 이상 5 미만
```

**`[a:b]` 의미**: **a번 인덱스 이상, b번 인덱스 미만**. b는 포함 ❌.

```python
hello = "Hello"
print(hello[1:4])       # ell  (1·2·3번, 4번 미포함)
print(hello[:3])        # Hel  (처음~2번)
print(hello[2:])        # llo  (2번~끝)
print(hello[:])         # Hello (전체 복사)
```

→ `[:]` 자체 복사는 실무에서 유용 (얕은 복사).

**[[파싱]]에 직결** — Day 1 강의의 "원하는 부분만 추출"이 여기 인덱싱·슬라이싱으로 구현.

**🎯 핵심 메시지**: 인덱스는 **0부터**. 슬라이싱 `[a:b]`는 **a 이상 b 미만**.

## 💡 일상 비유

### 비유 1: 변수 = 라벨 붙인 상자

- **상자에 라벨 (변수 이름)** 붙이고 → **물건 (자료) 넣기**
- 같은 상자에 다른 물건 넣으면 **이전 물건은 사라짐** (재할당)
- 상자 이름은 **숫자로 시작 ❌, 공백 ❌**

### 비유 2: 인덱싱 = 아파트 호수

- 동(리스트)의 **0층부터** 시작 (Python 특성)
- `apt[3]` = 4번째 호수의 사람
- 슬라이싱 `apt[2:5]` = 3~5층 사람들 (5층은 안 가져옴)

### 비유 3: 자료형 = 그릇 종류

- **숫자형 그릇**에는 숫자만 (산술 가능)
- **문자열 그릇**에는 글자 (이어붙이기 가능)
- **리스트 그릇**은 큰 그릇 — 다른 그릇들도 담을 수 있음 (중첩)
