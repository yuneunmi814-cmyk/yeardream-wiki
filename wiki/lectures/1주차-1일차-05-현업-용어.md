## 📖 본문 (정독용)

### 챕터 1. Version Control / Git (슬라이드 1~6)

**키워드 (강의 본문)**:
- **Version Control** = 전체 프로젝트 코드를 개발·출시 단계로 관리하는 방법
- **카카오톡 v9.9.0 업데이트 노트** = 출시 노트가 곧 VC의 사용자 측면
- **Git** = 천하통일. 리누스 토발즈 작품
- 대표 서비스:
  - **GitHub** (Microsoft 8조원 인수)
  - **Bitbucket** (Atlassian, 호주 부호 6위)
- Git 효용: **누가 작성했는지·협업·버전별 작업**
- **설탕 1·2·3스푼 비유**: 언제든 시점 이동 가능

✦ **강의자 풀이 추정**
**토발즈 = Linux + Git의 창시자**. 한 사람의 두 작품이 현대 컴퓨터 인프라의 절반.

GitHub MS 인수가(2018, 75억 달러) = **AI 학습 데이터의 가치**. Copilot의 토대.

→ 사용자 yeardream-wiki도 GitHub에 푸시 중. Git 일상 사용자.

**🎯 핵심 메시지**: Git = 코드 타임머신, 사실상 표준. 토발즈 일대 발명.

⚠️ **흔한 오해 1**: "Git = GitHub" → Git은 도구, GitHub는 서비스. Git 없이 GitHub 없고, GitHub 없이도 Git 가능.

⚠️ **흔한 오해 2**: "토발즈가 Linux 운영체제 전체 만듦" → 정확히는 **커널만**. GNU 등 도구와 합쳐져 OS.

### 챕터 2. 기획과 플로우차트 (슬라이드 7~13)

**키워드 (강의 본문)**:
- 정의: 어떤 서비스를 만들지 **핵심 흐름을 데이터와 함께 시각화**
- 쇼핑몰 핵심 로직: 상품 페이지 → 구매 → 배송
- **빈 곳 발견**: 배송 정보는? → **로그인** 추가 = 정보 자연 획득
- 각 단계 필요 데이터:
  - 로그인 → 이름·전화·주소
  - 구매 → 간편결제(카카오페이·네이버페이)
  - 배송 → 배송사 API 레퍼런스 체크
- **미스터리 박스 분기 플로우**: 결제 → 박스 오픈 → (당첨/100% 환불) → 마이페이지

✦ **강의자 풀이 추정**
**플로우차트의 진짜 가치 = 빈틈 발견**. 글로 적으면 놓치는 케이스가 그림으로 드러남.

→ 본격 플로우차트 = 일직선이 아닌 **분기 + 수렴** (미스터리 박스가 그 예시).

**🎯 핵심 메시지**: 플로우차트 = 빈틈 발견의 시각화.

⚠️ **흔한 오해**: "로그인 강제가 사용자 친화적" → 거짓. **간편 로그인**(카카오·구글)이 표준.

---

### 챕터 3. 오픈소스 + 라이선스 ⭐ (슬라이드 14~17)

**키워드 (강의 본문)**:
- **정의**: 프로젝트 전체 코드 공개·활용 가능
- 대표: Firefox·OpenOffice·Blender·PHP·Java·MySQL·Debian·WordPress·Apache·Ubuntu·Linux
- **무료지만 라이선스로 관리** (전부 무료 X)
- **OLIS** — 오픈소스SW 라이선스 종합정보 시스템 (olis.or.kr)
- **안드로이드도 오픈소스 OS**
- 라이선스 기재 의무 — **카카오톡 OSS Notice**: abseil-cpp·android-platform-* 등 다수 **Apache License 2.0**

✦ **강의자 풀이 추정**
인터넷의 거의 모든 인프라가 오픈소스 위에서 작동.

라이선스 4종:
- **MIT**: 거의 자유 (출처만)
- **Apache 2.0**: MIT + 특허 보호. **기업 친화**
- **GPL**: 전염성 강. **파생물도 공개 의무** = 기업 기피
- **BSD**: MIT 비슷

→ 앱·서비스 빌드 시 **카카오톡 OSS Notice 같은 라이선스 표기 의무** 따라옴.

**🎯 핵심 메시지**: 오픈소스 = 무료지만 라이선스. 카카오톡 OSS Notice가 표기 의무의 표준.

⚠️ **흔한 오해 1**: "오픈소스니까 표기 안 해도 됨" → 거짓. MIT·Apache조차 출처 표기 필수.

⚠️ **흔한 오해 2**: "Android = 구글" → AOSP(커널·기본)는 오픈, 구글 앱은 클로즈드. 화웨이 폰에 구글 앱 못 깔리는 이유.

📎 자세히: [[현업-용어]] 오픈소스

### 챕터 4. 클라우드 (슬라이드 18~23)

**키워드 (강의 본문)**:
- **정의**: 실물 서버 없이 인터넷으로 활용
- **기존 서버 단점**: 확장 부족·유지비 고정·예측 실패 위험
- **클라우드 장점**: 유연 셋업·사용량 과금·운영 부담 감소
- 트래픽 변동 시나리오: 증가 → 증설 → 감소 → **폐기**
- **3대 클라우드**: **AWS** (Amazon) / **Microsoft Azure** / **Google Cloud**

✦ **강의자 풀이 추정**
| | 옛 서버 | 클라우드 |
|--|---------|----------|
| 확장 | 일방향 (못 돌림) | 양방향 (Auto Scaling) |
| 비용 | 고정 | 사용량 기반 |
| 운영 | 직접 | 클라우드 회사가 |

→ 사용자 **Cloudflare Pages**가 정확히 클라우드 사용. 서버가 어디 있는지 신경 안 씀.

빅3 점유 70%+:
- AWS — 1위, 가장 폭넓음
- Azure — 기업·MS 연계
- GCP — AI·BigQuery 강함
- (보강) Cloudflare — 엣지·CDN, 사용자 현재 사용

**🎯 핵심 메시지**: 클라우드 = 인터넷으로 빌려 쓰는 서버. 양방향 확장 + 사용량 과금.

⚠️ **흔한 오해**: "AWS가 무조건 최고" → 시장 1위지만 비쌈. 작은 프로젝트엔 Cloudflare·Vercel 더 효율적.

📎 자세히: [[현업-용어]] 클라우드

### 챕터 5. SaaS (슬라이드 24~25)

**키워드 (강의 본문)**:
- **SaaS** = Software as a Service. 클라우드 기반 소프트웨어 서비스
- 특징: 파일 공유 유연·설치 없이 즉시 사용
- **Adobe의 Figma 28조원 인수** = SaaS 위력
- 계속 확장 중

✦ **강의자 풀이 추정**
사용자 일상 도구 = 거의 다 SaaS:
- Notion (메모)
- Canva (디자인)
- Slack (커뮤니케이션)
- Claude (AI)
- Cloudflare (호스팅)
- GitHub

**Figma 28조원 인수 (2022)**:
- 브라우저 디자인 도구. Photoshop·Illustrator 위협
- Adobe가 위협받자 200억 달러 인수 발표
- → **2023년 EU·영국 규제로 인수 무산** ⚠️ (강의 자료는 인수 발표 시점)
- 현재 Figma 독립 회사

**🎯 핵심 메시지**: SaaS = 구독형 클라우드 SW. Figma 28조가 가능성 보여줌. 사용자 일상.

⚠️ **흔한 오해 — 강의 본문 보완**: "Figma 인수는 완료됨" → **2023년 무산**. 강의 자료는 인수 발표 시점이라 무산 전.

## 💡 일상 비유

### 비유 1 — 카카오톡 업데이트 노트 (Version Control)

v9.9.0 업데이트 노트의 각 항목 = 한 버전에 들어간 변경사항. **출시 노트가 VC의 사용자 측면 표현**. 사용자 매일 봄.

### 비유 2 — 우버 vs 자동차 소유 (클라우드)

자동차 소유 = 옛 서버 (매달 보험·기름·세금, 적게 타도 같은 비용).
우버 = 클라우드 (탄 만큼만 비용, 안 타면 0원).
같은 이동, 다른 비용 구조.

## 이해 체크 (은미, 2026-05-29)

- [x] **Git = 내 컴퓨터 일기장 + 게임 슬롯** — Commit하면 그 시점 스냅샷 박제. 망쳐도 과거로 롤백
- [x] **GitHub = Git 일기장을 올리는 블로그/소셜** — 공유·협업 공간
- [x] **플로우차트 = 시각화로 빈틈 발견** — 말로 풀면 뇌가 대충 넘김. 도형·화살표로 그려야 "갈 곳 없는 화살표" 발견
- [x] **오픈소스 라이선스** = "갖다 써도 되지만 이건 지켜라" 규칙. MIT·Apache(자유)·GPL(공개 의무)·BSD(자유)
- [x] **OLIS** = 한국저작권위 오픈소스 정보 시스템 (검색용)
- [x] **OSS Notice** = 앱·서비스 안에 "이 오픈소스 썼어요" 명시. 안 적으면 저작권 소송 위험
- [x] **클라우드 3대장** — AWS(1위)·Azure(MS)·GCP(구글)
- [x] **한국 기업 클라우드 활용**:
  - 당근·배민·토스·쿠팡 = **AWS** (절대다수)
  - 이마트·롯데온 = **Azure** ("아마존은 우리 경쟁자라서")
  - 게임사 (넷마블·엔씨) = **AWS + GCP 멀티 클라우드**
- [x] **SaaS** = 설치 ❌, 웹에서 구독형 사용. **피그마·노션·슬랙·Gmail 다 SaaS**
- [x] **피그마 핵심 4기능**:
  - Canvas (무한 도화지) + Frame (화면 크기 틀)
  - Component (원본) + Instance (복사본) → 원본 하나 바꾸면 복사본 다 자동 변경
  - Prototype = 화살표로 화면 연결 → 진짜 앱처럼 시뮬
  - Dev Mode = 디자인에서 CSS 코드 자동 추출
- [x] **캔바 vs 피그마** — 캔바(대중·템플릿) vs 피그마(전문가·앱 설계). 현재 서로 영역 침범 중
- [x] **Gemini로 정수 추출 → Claude로 위키 누적** ⭐ — 토큰 절약 워크플로우 작동
- [ ] (아직 막힌 거 있으면 여기 추가)
