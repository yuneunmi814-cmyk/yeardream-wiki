## 🚀 예습 (5분용)

### 한 줄 요약

**Linux = 토발즈 1991, 오픈소스 OS, 인터넷 96%**. **Ubuntu**가 사용자 친화 배포판. **Shell(Bash)**가 사용자-커널 통역. 기초 명령어 + **apt(패키지) + sudo(권한)**.

### 핵심 단어 5~7개

- [[현업-용어]] — Git·오픈소스 등 통합
- **GNU/Linux** — 도구(GNU) + 커널(Linux)
- **Ubuntu** — 사용자 친화 배포판
- **Bash / Shell** — 사용자-커널 통역
- **whoami·pwd·ls·cd** — 기초 명령어
- **GUI vs CLI** — 시각 vs 명령어
- **apt / sudo / root** — 패키지·권한

### 큰 그림

```
Linux 구조:
   하드웨어 → Kernel(토발즈) → Shell(Bash) → 응용 프로그램
                                    ↑
                                사용자 명령어
                                    ↑
                              GUI(GNOME) or CLI(터미널)
                                    ↑
                              apt + sudo로 도구 설치
```

→ Day 2 8강 중 **5강**. OS 이론 끝, **리눅스 실습 시리즈 도입**.

## 📖 본문 (정독용)

### 챕터 1. 리눅스 소개 + 구조 (슬라이드 1~10)

**키워드 (강의 본문)**:
- **Linux** = 리누스 토발즈 작품. 자유 소프트웨어·오픈소스 표본
- **GNU** = "Gnu is not Unix" 줄임말. 유닉스 소스 비공개 문화 반발
- 리눅스 구조: 하드웨어 → **Kernel** → **Shell** → 응용 프로그램

✦ **강의자 풀이 추정**
**Linux = 인터넷 서버 96%·안드로이드·임베디드 전부**:
- 카카오톡·인스타·구글·넷플릭스 서버 = Linux
- 안드로이드 = Linux 커널 기반
- Apple은 별도(macOS는 Unix 계열, iOS는 Darwin)

**4계층**:
- 하드웨어 = CPU·메모리·디스크
- **커널(토발즈 작품)** = OS 핵심
- **셸** = 사용자 명령 해석
- **응용 프로그램** = 사용자가 직접 쓰는 앱

식당 비유: 주방장(커널) ↔ 서버(셸) ↔ 손님 메뉴판(앱).

**🎯 핵심 메시지**: Linux 4계층 = 하드웨어·커널·셸·앱. 토발즈는 커널만 만듦.

⚠️ **흔한 오해**: "GNU와 Linux는 같음" → 거짓. **GNU는 도구 모음, Linux는 커널**. 둘이 합쳐서 GNU/Linux.

---

### 챕터 2. 리눅스 특징·종류·우분투 (슬라이드 11~22)

**키워드 (강의 본문)**:
- 특징: 유닉스 기반·멀티 유저/태스킹·CLI/GUI·강력한 네트워킹
- 배포판:
  - **레드햇 계열** (rpm·yum): CentOS·Fedora
  - **데비안 계열** (deb·apt): **Ubuntu**·라즈비안·Linux Mint
  - **안드로이드** (apk)
- 활용: 서버·안드로이드·블랙박스·의료기기·셋톱박스·개인 PC
- **Ubuntu**: 데비안 기반. 개인 PC 최적, 간결, 큰 커뮤니티, 6개월 단위 업데이트
- 우분투 어원: 남아프리카 반투어 "사람들 간의 관계와 헌신"
- **GNOME** 인터페이스 (UI)
- 버전: YY.MM. **LTS = Long Term Support** (가장 안정, 5년 지원)

✦ **강의자 풀이 추정**
**Ubuntu = 입문자 친화** + **클라우드 서버 표준** (AWS·GCP 인스턴스 기본 선택).

→ 사용자가 클라우드 서버 만들면 거의 Ubuntu LTS. 강의의 기준 OS.

**🎯 핵심 메시지**: Linux 배포판 = 라면 회사들 비유. Ubuntu가 대표. LTS가 서버 표준.

⚠️ **흔한 오해**: "최신 버전이 항상 좋음" → 서버는 LTS 안전. 새 버전은 버그 가능.

### 챕터 3. Shell + 기초 명령어 (슬라이드 23~35)

**키워드 (강의 본문)**:
- **Shell** = 커널·사용자 통역. 명령 해석
- 종류: **Bash** (Bourne-Again, GNU 일환, 리눅스 기본)·sh·csh·ksh
- 명령어:
  - `whoami` — 로그인 사용자 ID
  - `passwd` — 비밀번호 변경 (입력 안 보임)
  - `pwd` — 현재 디렉토리 (Print Working Directory)
  - `ls` — 목록
  - `ls -l` — 상세 (권한·소유자·크기·시간)
  - `ls -a` — 숨김 포함 (`.`로 시작)
  - `ls -al` — 둘 다
  - `cd 폴더` — 디렉토리 이동
  - `--help` — 모든 명령어 도움말

✦ **강의자 풀이 추정**
Shell 진화:
- sh (1977 원조) → csh → ksh → **Bash** (1989, Linux 표준) → **Zsh** (보강, macOS 기본, Catalina부터)

→ 사용자 맥북 터미널 = Zsh. Linux 서버는 Bash. 거의 호환.

**cd 자주 쓰는 형태**:
```bash
cd 폴더명   # 그 폴더로
cd ..       # 상위
cd ~        # 홈
cd /        # 루트
cd -        # 직전
```

**ls -al** 출력 예:
```
drwxr-xr-x 1 elicer elicer 21 Sep 10 09:22 .
-rw-r--r-- 1 elicer elicer 4093 Mar 19 17:15 elice_utils.py
```

→ 다음 강 5강에서 권한(rwx) 본격.

**🎯 핵심 메시지**: Shell = 커널·사용자 통역. Bash 표준. ls·cd·pwd·whoami가 매일 쓰는 4종.

⚠️ **흔한 오해**: "passwd 입력 안 됨" → 입력은 되고 있음. **보안상 화면에 안 표시** (별표·점도 안 뜸).

📎 **실습 ✨**: 사용자 맥북 터미널 열고 `whoami`·`pwd`·`ls -al`·`cd ~` 직접 입력해보기. macOS도 작동.

### 챕터 4. GUI vs CLI + apt + sudo/root (슬라이드 36~52)

**키워드 (강의 본문)**:
- **GUI** = 아이콘·이미지. Windows·Mac 표준
- **CLI** = 문자 인터페이스. Windows CMD, Mac Terminal
- Ubuntu = GUI(GNOME) + CLI 둘 다
- **apt** = Advanced Packaging Tool. Ubuntu/Debian 패키지 관리
- **apt 통합**: 옛 `apt-get`·`apt-cache` → 현재 `apt`로 통일
- 주요 명령:
  - `apt install·remove·update·search·show`
- **권한 없음 에러**: "are you root?"
- **sudo** = 임시 관리자 권한. 비밀번호 입력
- **root** = 모든 권한 최고 관리자. 운영체제 전체 제어

✦ **강의자 풀이 추정**
**apt = Ubuntu의 앱스토어**. macOS Homebrew·Windows winget과 같은 역할.

```bash
# 새 서버 만들면 첫 명령
sudo apt update && sudo apt upgrade

# 패키지 설치
sudo apt install git python3 nginx
```

**update vs upgrade 헷갈림**:
- `update` = 목록만 갱신
- `upgrade` = 실제 설치

**root 위험**: 실수하면 시스템 망가짐. 평소엔 일반 계정, **필요할 때만 sudo**.

→ 사용자 맥북도 root 있음 (`sudo -i`로 진입). 단 기본 비활성.

**🎯 핵심 메시지**: apt = Ubuntu 패키지 관리. sudo = 임시 관리자. root는 위험, 최소 사용.

⚠️ **흔한 오해**: "최신 버전이 안전" → 보안 패치는 최신이 좋지만, 메이저 업그레이드는 LTS 권장.

📎 **실습 ✨**: 맥북에서 `brew install tree` 같은 명령 → Homebrew가 apt 역할. 같은 패키지 관리 패턴.

## 💡 일상 비유

### 비유 1 — 식당의 서버 (Shell)

손님(사용자)이 메뉴 주문 → 서버(셸)가 주방장(커널) 언어로 전달 → 음식 완성 후 손님 테이블로. Shell이 정확히 식당 서버.

### 비유 2 — 맥북 Homebrew (apt 대응)

사용자 맥북에서 `brew install ...`로 도구 설치. Ubuntu에선 `sudo apt install ...`. 같은 개념의 OS별 도구.
