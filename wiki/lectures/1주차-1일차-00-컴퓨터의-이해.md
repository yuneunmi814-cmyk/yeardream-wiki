# 00 / 컴퓨터의 이해 — 교재 버전

> **4섹션 구조**: 🚀 예습 / 📖 본문 / 🔄 복습 / 💡 일상 비유 → [[_meta/학습-워크플로우]]

---

## 🚀 예습 (5분용)

### 한 줄 요약

**컴퓨터는 0과 1로만 작동**하는 기계. 프로그래밍 언어가 그 사이의 통역. 운영체제(OS)가 그 위의 매니저.

### 핵심 단어 5~7개

- [[컴퓨터-기초]] — 통합 노트
- **기계어** — 컴퓨터의 모국어 (0과 1)
- **컴파일러 / 인터프리터** — 코드 → 기계어 번역기
- **Low / High level language** — 기계와의 거리
- [[운영체제]] — OS 정의 (2일차 본격)
- **클라이언트 / 서버** — Front-end / Back-end 첫 등장

### 큰 그림

```
사람 ── 프로그래밍 언어 ── 컴파일러/인터프리터 ── 기계어 ── CPU
                              ↓
                       운영체제(OS)가 다 중재
                              ↓
                       클라이언트 ↔ 서버 (다음 강)
```

→ 6강 시리즈의 **토대 강의**. 다음 5강의 모든 개념이 여기서 출발.

## 📖 본문 (정독용)

### 챕터 1. 0과 1, 기계어 (슬라이드 1~4)

**키워드 (강의 본문)**:
- 컴퓨터가 직접 알아듣는 언어 = **0 or 1**
- TV·컴퓨터·에어컨 On/Off → 1/0
- 기계어 예시: `0000 1001 1100 0110 ...` (8비트 단위)
- "복잡한 기계어를 사람이 사용하기 쉽게 만들어 기계를 활용하게 한다"

✦ **강의자 풀이 추정**
컴퓨터는 전기 신호로 동작. 전기 흐름=1, 안 흐름=0. 모든 디지털 = 1/0 묶음. 화면의 모든 이미지·텍스트·동영상도 결국 0/1 천만 개. 1940년대엔 사람이 직접 기계어 입력 → 너무 비효율 → 프로그래밍 언어 등장.

**🎯 핵심 메시지**: 컴퓨터의 모국어는 0/1. 화려한 것은 모두 그 위에 쌓은 추상화.

⚠️ **흔한 오해**: "프로그래머는 0과 1을 직접 친다" → 거짓. 거의 아무도 안 씀.

---

### 챕터 2. 프로그래밍 언어 (슬라이드 5~9)

**키워드 (강의 본문)**:
- 대표 언어: Javascript, Swift, Kotlin, C++, Go, C, Rust, Assembly, Java, C#, Python …
- 모두 **컴퓨터 기계 위에서 실행되기 위한 언어들**
- 번역 흐름: `Code 작성 → 인터프리터/컴파일러 → 기계어 → 컴퓨터 인식`
- **Low level**: 기계 가까운 언어. 빠르고 어려움
- **High level**: 사람 친화적. 단점 보완 위해 등장. **고차원 의미가 아님**

✦ **강의자 풀이 추정**
언어별 주 용도:
- JavaScript = 웹 / Swift = iOS / Kotlin = Android / C·C++·Rust = 시스템·게임 / Python = AI·데이터 / Go = 서버

**컴파일러 vs 인터프리터**:
- 컴파일러 = 전체 미리 번역 → 실행파일 (C·Swift·Rust)
- 인터프리터 = 한 줄씩 즉시 (Python·JavaScript)
- 영화 자막(컴파일) vs 동시통역(인터프리트)

**Low/High level**:
- Low = 자전거 직접 조립. 빠르고 어려움
- High = 완성된 자전거 구매. 편하고 약간 느림
- "High = 고차원" ≠ 사람 친화 추상화

→ 사용자(은미)는 Swift(NyangTalk)·Python·JavaScript 다 써본 상태. 명시적 멘탈 모델 정리 단계.

**🎯 핵심 메시지**: 언어가 많은 건 각자 강점이 다름. Low/High는 기계와의 거리(트레이드오프).

⚠️ **흔한 오해 1**: "High level = 더 고급 언어" → 강의 본문 명시. 단순히 사람 친화 추상화 수준.

⚠️ **흔한 오해 2**: "JavaScript는 인터프리터라 무조건 느리다" → 거짓. V8 엔진이 일부 컴파일해서 빠름.

---

### 챕터 3. 클라이언트와 서버 (슬라이드 10)

**키워드 (강의 본문)**:
```
[배달앱] ⇄ [서버]
(클라이언트)  (서버)
Web: 프론트엔드  Web: 백엔드
→ 주문 / ← 주문완료
```

✦ **강의자 풀이 추정**
앱·웹의 거의 모든 구조 = "클라이언트 ↔ 서버":
- **클라이언트** = 사용자가 만지는 쪽 (배달앱·인스타·카톡)
- **서버** = 요청 처리하고 응답하는 쪽 (회사 데이터센터)

웹에선 다른 이름:
- 클라이언트 ≈ **프론트엔드** (사용자 눈에 보임)
- 서버 ≈ **백엔드** (뒤에서 처리)

→ 다음 강 [[lectures/1주차-1일차-01-앱-웹-서비스|02 앱·웹 서비스]]에서 본격 펼침.

**🎯 핵심 메시지**: 모든 앱·웹 = 클라이언트 ↔ 서버 대화. 2강의 토대.

---

### 챕터 4. 운영체제(OS) (슬라이드 11~22)

**키워드 (강의 본문)**:
- **OS 정의**: 컴퓨팅 기계를 쉽게 다루기 위한 시스템 소프트웨어
- **종류**: macOS / Windows / Linux (데스크탑) + Android / iOS (모바일)
- **흐름**: 공장 제작 → OS 설치 → App 제작 → 사용자 활용
- **카메라 권한 예시**: OS가 하드웨어 접근 코드 제공. **OS별로 코드 다름** → iOS·Android 따로 제작
- **권한 다이얼로그**: "사진 찍기 허용?" — OS가 사용자 보호

✦ **강의자 풀이 추정**
OS = **컴퓨터의 매니저·비서**. 앱과 하드웨어 사이 중간 관리:
- 한글 파일 열기 → OS가 디스크 찾기·메모리 로드·앱 실행·화면 표시 다 알아서

OS의 두 핵심 책임:
1. **하드웨어 추상화** — 앱이 카메라·디스크 직접 만지지 않아도 됨
2. **사용자 보호 (권한 관리)** — 앱의 잘못된 동작 방지

→ Day 2 8강 시리즈에서 본격 펼침 ([[lectures/1주차-2일차-00-운영체제-소개|2일차 00 OS 소개]]).

**🎯 핵심 메시지**: OS = 사용자가 컴퓨터 쉽게 쓰게 해주는 중간 관리자. 2일차에 본격.

⚠️ **흔한 오해**: "운영체제 = Windows·macOS 같은 거" → 맞지만, 폰(iOS·Android)·스마트워치·자동차·IoT 다 OS 있음.

## 💡 일상 비유

### 비유 1 — 카카오톡 읽음 표시 (0/1)

카톡 메시지 옆 "1"이 사라지면 친구가 읽은 거. 그 작은 표시도 컴퓨터 안에선 **읽음=0, 안 읽음=1**. 화려한 UI 뒤엔 결국 1/0의 어마어마한 묶음.

### 비유 2 — 호텔 컨시어지 (OS)

호텔 컨시어지한테 "저녁 식당 예약하고 차도 불러주세요" 하면 알아서 다 해줌. 손님(앱)은 디테일 몰라도 됨. OS가 컴퓨터의 컨시어지.
