## 🚀 예습 (5분용)

### 한 줄 요약

**정규표현식 + grep + 파이프 + 리다이렉션 = Unix의 진짜 위력**. 작은 도구들의 조합으로 복잡한 작업을 한 줄에.

### 핵심 단어 5~7개

- [[파싱]] — 정규표현식과 연결
- **정규표현식 (Regex)** — 패턴 표현 언어
- **grep** — 파일 내 패턴 검색
- **stdin·stdout·stderr** — 3대 스트림
- **`>` `>>` `2>`** — 리다이렉션
- **파이프 `|`** — 명령 체인
- **mount / umount** — 디스크 연결

### 큰 그림

```
Unix 철학 = 작은 도구의 조합

명령 1 | 명령 2 | 명령 3 > 파일
   ↑       ↑       ↑      ↑
출력 →  입력으로  처리   파일 저장

예: cat log.txt | grep "error" | tail -20 > errors.txt
```

## 📖 본문 (정독용)

### 챕터 1. 정규표현식 ⭐ (슬라이드 1~9)

**키워드 (강의 본문)**:
- 정규표현식 (Regex) = 특정 규칙 가진 문자열 집합 표현 형식
- 메타 문자: `^`(시작), `$`(끝), `.`(아무 문자), `\d`(숫자), `\D`(숫자 아님), `\w`(영숫자_)
- 선택 `|`: `alice|elice` = 둘 중 하나
- 문자 집합 `[]`: `[0-9]`, `[A-Z]`, `[^abc]`(abc 아닌)
- 수량자: `?`(0~1), `*`(0이상), `+`(1이상), `{n}`(n개), `{n,m}`(n~m)
- 그룹 `()`: 묶음 + 재사용 (`\1`)
- 전화번호 예: `^010(\D?\d{4}){2}$`

✦ **강의자 풀이 추정**
**메타 문자 5종 기억**:
- `^abc` = abc로 시작
- `abc$` = abc로 끝
- `a.c` = a + 아무 문자 + c
- `\d` = 숫자 (= `[0-9]`)
- `\w` = 영숫자_

**전화번호 정규식 분해** `^010(\D?\d{4}){2}$`:
- `^010` 시작
- `\D?` 숫자 아닌 0~1개 (구분자 -·공백·없음)
- `\d{4}` 숫자 4개
- `(...){2}` 앞 그룹 2번

매칭: `01012345678`·`010-1234-5678`·`010 1234 5678` 모두 ✓

→ 사용자 1인 컨설팅에서 입력 검증·이메일·URL 패턴 매일.

**🎯 핵심 메시지**: Regex = 패턴 언어. ^·$·.·\d·\w·[]·|·{n}·() 가 기초.

⚠️ **흔한 오해**: `[^abc]` = "^abc 시작" → 거짓. **[] 안의 ^는 부정**. abc 아닌 한 문자.

📎 자세히: [[파싱]]

📎 **실습 ✨**: [regex101.com](https://regex101.com)에서 패턴 실시간 테스트.

### 챕터 2. grep ⭐ (슬라이드 10~14)

**키워드 (강의 본문)**:
- **grep** = 파일 내 패턴 매칭 행 출력
- 사용: `grep [option] [pattern] [파일]`
- 옵션:
  - `-i`: 대소문자 무시
  - `-v`: 일치 안 하는 행
  - `-c`: 개수만
  - `-w`: 단어 단위
- 정규식과 결합: `grep [0-9] file`, `grep "^Error" log`
- `grep -v ^# config` = 주석 제외하고 보기

✦ **강의자 풀이 추정**
**자주 쓰는 형태**:
```bash
grep "error" log.txt              # error 들어간 줄
grep -i "ERROR" log.txt           # 대소문자 무시
grep -rn "함수명" .                # 재귀 + 줄번호 (코드 검색의 표준)
grep -l "TODO" *.py               # 매치 파일 이름만
grep -E "(error|warn|fail)" log   # 확장 정규식
```

**🎯 핵심 메시지**: grep = 정규식 + 파일 검색. -i·-v·-r·-n이 핵심 옵션.

📎 **실습 ✨**: `ls /usr/bin | grep python` (시스템의 python 관련 명령 검색).

---

### 챕터 3. File Redirection (슬라이드 15~20)

**키워드 (강의 본문)**:
- 표준 스트림 3종:
  - **stdin (0)**: 키보드 입력
  - **stdout (1)**: 화면 출력
  - **stderr (2)**: 오류 출력
- `>` = 표준 출력 → 파일 (덮어쓰기)
- `>>` = 이어쓰기
- `2>` = 오류 → 파일
- `<` = 파일 → 입력
- 예: `cat < hello.txt > elice.txt` (입력+출력 동시)

✦ **강의자 풀이 추정**
**리다이렉션 일상**:
```bash
ls > files.txt                # 목록 저장
echo "log" >> log.txt         # 로그 누적
command 2> errors.txt         # 오류만
command > out.txt 2>&1        # 출력+오류 같은 파일 (2를 1로)
```

`2>&1` = "stderr(2)를 stdout(1)으로 보내라". 둘 다 한 파일에 합치는 표준 패턴.

**🎯 핵심 메시지**: `>` 덮어쓰기 / `>>` 이어쓰기 / `2>` 오류 / `2>&1` 합치기.

⚠️ **흔한 오해**: `>`로 stderr 캡처 → 거짓. stdout만. **오류는 `2>` 별도**.

📎 **실습 ✨**: `ls > files.txt && cat files.txt`로 결과 확인.

---

### 챕터 4. 파이프 ⭐ (슬라이드 21~24)

**키워드 (강의 본문)**:
- **Pipe `|`** = 한 명령 출력 → 다음 명령 입력
- 예: `head a.txt | grep [0-9]` (앞 10줄 중 숫자 든 행)
- 파이프 + 리다이렉션: `head a.txt | grep [0-9] > result.txt`

✦ **강의자 풀이 추정**
**Unix 철학의 핵심** = "한 가지 일을 잘하는 작은 도구들을 파이프로 조합".

실무 패턴:
```bash
# 큰 로그에서 에러만, 마지막 20줄
cat big.log | grep "ERROR" | tail -20

# 프로세스 메모리 사용 순 상위 10개
ps aux | sort -k4 -nr | head -10

# 디렉토리 파일 수 세기
ls -1 | wc -l

# 특정 단어 든 .py 파일들 검색
find . -name "*.py" | xargs grep "import requests"
```

→ 사용자 콘텐츠 워크플로우도 비슷: Notion 초안 → Claude 다듬기 → Canva 디자인 → 인스타 업로드. 각 단계 출력 = 다음 입력.

**🎯 핵심 메시지**: 파이프 = 명령 체인. Unix 철학의 핵심. 복잡 작업을 단순 도구 조합으로.

📎 **실습 ✨**: `ls -al | grep "^d"` (디렉토리만 출력).

---

### 챕터 5. 마운트 (슬라이드 25~29)

**키워드 (강의 본문)**:
- mount = 보조기억장치 ↔ 디렉토리 연결 (3강 복습)
- Windows = **PnP** (Plug and Play, 자동)
- 리눅스 = **수동 마운트** 필요
- `mount [option] [device] [directory]`
  - `-a`: /etc/fstab 명시된 것 마운트
  - `-t`: 파일 시스템 유형 지정 (ext4 등)
  - `-o`: 추가 옵션
- `umount` = 해제 (n 없음 ⚠️)

✦ **강의자 풀이 추정**
```bash
sudo mount -t ext4 /dev/sdb1 /mnt/data    # ext4 디스크 마운트
sudo mount -a                              # /etc/fstab 기반 자동
sudo umount /mnt/data                      # 해제
```

**`/etc/fstab`** = 부팅 시 자동 마운트 설정. 한 번 등록하면 재부팅에도 유지.

→ 사용자가 AWS EC2에 추가 디스크 붙이면 수동 마운트 필요.

**🎯 핵심 메시지**: 리눅스 마운트 = 수동. mount + umount (n 없음).

⚠️ **흔한 오해**: 명령어가 `unmount`인 줄 → **`umount`** (n 없음). Unix 전통 짧은 이름.

📎 자세히: [[lectures/1주차-2일차-03-파일-시스템|이론 4강 파일 시스템]]

## 이해 체크 (은미, 2026-05-29)

### 파일 명령어 7종
- [x] **mkdir** = Make Directory (폴더 만들기). `-p` 옵션 = 하위 폴더 한 번에
- [x] **touch** = 텅 빈 새 파일 만들기 (`touch note.txt`)
- [x] **rm** = Remove (삭제, 휴지통 없이 영구)
- [x] **cp** = Copy (`cp 원본 복사본`). 폴더는 `cp -r`
- [x] **mv** = Move (이동 또는 이름 변경). 같은 폴더 안이면 이름 변경
- [x] **cat** = 파일 내용 터미널에 출력. ⭐ "고양이가 상자 파헤치기" (은미 비유)
- [x] **find** = 파일 검색. `find 폴더/ -name "*.확장자"`

### rm -rf 위험성 ⚠️
- [x] **-r (Recursive)** = 폴더+하위 다 지움. **마트료시카 인형 다 깐다** 비유
- [x] **-f (Force)** = 묻지도 따지지도 말고 강제 삭제
- [x] **휴지통 없음** = 리눅스 rm은 즉시 영구 삭제. 윈도우와 다름
- [x] **`sudo rm -rf /` 절대 금지** ⚠️⚠️ — OS 전체 영구 삭제
- [x] **-r·-f 각자 사용 가능** (rm -r 폴더 / rm -f 잠긴파일). rf만은 ❌

### 명령어 사용 공식 = 영어 문장
- [x] **동사 + 조건(옵션) + 목적어** 구조 ⭐
- [x] **옵션 글자는 영어 첫글자**: -l(Long)·-a(All)·-r(Recursive)·-f(Force)·-i(Ignore)

- [x] **nohup ~ &** = **No Hang Up**. 터미널 꺼도 24시간 계속 동작. **서버 배포 필수**
- [x] **export 변수=값** = 환경 변수 (메모리에 임시 저장)
- [x] **echo $변수명** = 환경 변수 확인
- [x] **export 보안 약점** ⚠️ — history 기록 + 일회용. 진짜는 `.env` 파일
- [x] **.env 파일 + .gitignore** = 비밀번호 격리 표준

### 정규표현식·grep·스트림·리다이렉션·파이프·마운트 (2026-05-29)

- [x] **정규표현식 (Regex)** = 문자 패턴 표현 미니 언어. `^010-\d{4}-\d{4}$`(전화) / `.+@.+\..+`(이메일)
- [x] **grep 옵션**: `-i`(대소문자 무시)·`-r`(폴더 전체)·`-n`(줄 번호)·`-v`(불일치만)
- [x] **3대 스트림 (영어 풀이)**:
  - **stdin** = **St**an**d**ard **In**put (귀, 입력)
  - **stdout** = **St**an**d**ard **Out**put (입, 정상 응답)
  - **stderr** = **St**an**d**ard **Err**or (비명, 에러 응답)
- [x] **리다이렉션 = 출력 방향 바꾸기**:
  - `>` 덮어쓰기 / `>>` 이어쓰기 / `2>` 에러만 / `&>` 둘 다
  - ⚠️ `>` 한 개는 기존 내용 다 날아감 (로그는 `>>`)
- [x] **파이프 `|` = 컨베이어 벨트** ⭐. 앞 결과 → 뒤 명령어 입력
  - `ps -ef | grep python` (실행 중 python)
  - `cat log | grep ERROR | wc -l` (에러 개수)
  - `ls -lS | head -5` (큰 파일 5개)
  - **Unix 철학**: 작은 명령어들 조합
- [x] **mount = 디스크를 폴더에 연결 선언** (복사 ❌). 리눅스는 드라이브 개념 없음
  - `sudo mount /dev/sdb1 /mnt/usb` → USB가 /mnt/usb로 보임
  - `sudo umount /mnt/usb` → 안전 제거. 안 하고 USB 뽑으면 데이터 손상 위험
  - AWS EC2 추가 디스크 연결 시 실무 자주

### 외우기 트릭 (2026-05-29 추가)

- [x] **ps = Process Status** (실행 중 프로세스 = 윈도우 작업관리자)
- [x] **-ef = Everything + Full** (모든 프로세스 전체 정보)
- [x] **디버깅 = De-Bug** (잘못된 거 찾고 고치기)
- [x] **grep = 파일 안에서 검색** (`grep "무엇을" 어디에서`)
- [x] **파이프 순서**: `ps -ef | grep python` = ps 결과가 grep의 입력으로 → grep 결과만 화면. 회사 직원명단 → "python" 들어간 사람만 골라냄
- [x] **번호로 외우기**: 0=stdin / 1=stdout / 2=stderr → 그래서 `2>`가 에러 리다이렉션
- [x] **리다이렉션 화살표**:
  - `>` = 화살표 1개 = 덮어쓰기 (빈 종이에 적기)
  - `>>` = 화살표 2개 = 이어쓰기 (일기장 추가)
  - `2>` = 에러만 별도 파일로
- [x] **로그는 `>>`** 안전. `>` 잘못 쓰면 기존 다 날아감
- [ ] (아직 막힌 거 있으면 여기 추가)
