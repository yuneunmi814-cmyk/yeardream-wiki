## 🚀 예습 (5분용)

### 한 줄 요약

**리눅스 프로세스 = PID + 부모/자식**. `ps`·`kill`·**job 백그라운드**·**at(1회)·cron(주기적)** 스케줄링·**SSH** 원격 접속.

### 핵심 단어 5~7개

- [[lectures/1주차-2일차-01-프로세스|이론 2강 프로세스]]
- **PID·PPID** — 프로세스 ID·부모 ID
- **ps / kill** — 목록·종료
- **job·foreground·background** — 셸 단위 작업
- **at / cron** — 일회 / 주기 자동화
- **SSH** — 안전한 원격 접속
- **포트** — SSH 기본 22

### 큰 그림

```
프로세스 관리 (리눅스 실무)
  ps (목록) → kill (종료)
       ↓
  job (백그라운드 작업)
       ↓
  at (1회) / cron (주기적) — 자동화
       ↓
  SSH (원격 접속) — Telnet의 안전 버전
```

## 📖 본문 (정독용)

### 챕터 1. 프로세스 (슬라이드 1~10)

**키워드 (강의 본문)**:
- 프로세스 = 메모리 적재되어 실행 중인 모든 프로그램
- 특징: 1개 이상 프로세스·병행·부모/자식·커널 관리·소유자·**고유 PID**
- **PID 1 = init**, **PID 2 = kthreadd**. init이 모든 프로세스의 조상
- 메모리 5영역 (이론 2강은 4영역): 코드·**데이터·BSS·힙·스택**
  - BSS = 초기화 안 된 변수

✦ **강의자 풀이 추정**
**모든 프로세스가 init의 후손**. 가계도처럼 트리. init 죽으면 시스템 다운.

```
PID 1 (init)
├── PID 100 (네트워크 매니저)
├── PID 200 (Bash 셸)
│   ├── PID 300 (python script)
│   └── PID 301 (ls 명령)
└── PID 400 (웹 서버)
```

→ Day 2 [[lectures/1주차-2일차-01-프로세스|이론 2강]] 메모리 4영역에 **BSS 추가 분리**가 정확한 5영역.

**🎯 핵심 메시지**: PID 1 = init = 조상. 메모리 5영역(BSS 분리).

---

### 챕터 2. 프로세스 명령어 ⭐ (슬라이드 11~17)

**키워드 (강의 본문)**:
- `ps [option]` — 프로세스 목록
  - `-e`: 모든 프로세스
  - `-f`: 풀 정보 (PPID·STIME 등)
  - `-a`: 전체 사용자
  - `-x`: 터미널 무관
- `ps -ef` / `ps -aux` 가 자주 쓰는 형태
- `ps -aux`: %CPU·%MEM·VSZ·RSS·STAT 포함
- `kill [option] [PID]`
  - `-1` (SIGHUP): 재실행
  - `-9` (SIGKILL): 강제 종료
  - `-15` (SIGTERM): 정상 종료
- `kill -9 PID` (자주 씀)

✦ **강의자 풀이 추정**
**3대 시그널**:
- **SIGTERM (-15, 기본)**: "정상 끝내" — 프로세스가 정리 후 종료
- **SIGKILL (-9)**: "당장 죽어" — 정리 못 함. 마지막 수단
- **SIGHUP (-1)**: "재시작" — 설정 재로드

**실무 흐름**:
```bash
ps -ef | grep python     # PID 찾기
kill -15 12345           # 정상 종료 시도
ps -ef | grep python     # 아직 살아 있나?
kill -9 12345            # 강제
```

**ps -aux**의 STAT 컬럼:
- S = Sleep
- R = Run
- Z = Zombie

**🎯 핵심 메시지**: ps = 목록, kill = 종료. -15(정상)→-9(강제) 순서.

⚠️ **흔한 오해**: "kill = 무조건 죽임" → 거짓. **신호 전송**일 뿐. SIGKILL(-9)만 무조건.

📎 **실습 ✨**: `ps aux | head -20`·`ps -ef | grep bash` 시도 (macOS도 작동).

### 챕터 3. job (백그라운드) (슬라이드 18~22)

**키워드 (강의 본문)**:
- **job** = 백그라운드 실행 작업. 셸 단위
- 터미널 종료 시 job도 종료
- foreground (전경) ⇄ background (배경)
- 전환: `&` (백그라운드로 실행), `Ctrl+Z` (멈춤), `bg`/`fg`, `jobs` (목록)
- `kill %1` — job 1번 종료 (% 표기로 PID와 구분)

✦ **강의자 풀이 추정**
```bash
sleep 500 &           # 백그라운드 실행
jobs                  # 목록 [1] [2]
kill %1               # 1번 종료
Ctrl+Z                # 현재 작업 멈춤
bg                    # 멈춘 작업 백그라운드로
fg                    # 다시 전경
```

긴 작업(빌드·다운로드)을 백그라운드로 두고 다른 명령 계속.

**🎯 핵심 메시지**: job = 셸 단위 작업. `&`/Ctrl+Z/bg/fg로 전환. `kill %번호`.

---

### 챕터 4. at + cron ⭐ (슬라이드 23~30)

**키워드 (강의 본문)**:
- **at** = 지정 시간 **1회 실행** 예약
  - `at now + 3 hours -f elice.sh`
  - `atq` 목록, `atrm 번호` 삭제
- **crontab** = **주기적 실행**
  - `-l` 보기, `-e` 편집, `-r` 삭제
- crontab 형식: **분 시 일 월 요일 명령어**
  - 요일: 0=일, 1=월, …, 6=토
  - 예: `10 1 1 * * /home/elice/test.sh` (매월 1일 1:10)
  - `0,30 1 * * 0 /backup.sh` (일요일 1시·1:30)

✦ **강의자 풀이 추정**
**cron 표기 5필드**:
```
 ┌─ 분 (0-59)
 │ ┌─ 시 (0-23)
 │ │ ┌─ 일 (1-31)
 │ │ │ ┌─ 월 (1-12)
 │ │ │ │ ┌─ 요일 (0-6, 0=일)
 │ │ │ │ │
 * * * * * command
```

특수:
- `*` = 모든 값
- `,` = 여러 (0,30 = 0분·30분)
- `-` = 범위 (1-5 = 월~금)
- `/` = 간격 (*/5 = 5분마다)

**실무 예시**:
```cron
0 3 * * *          /backup/run.sh      # 매일 오전 3시 백업
*/5 * * * *        /scripts/health.sh  # 매 5분 헬스체크
0 9 * * 1-5        /reports/daily.sh   # 평일 오전 9시 리포트
```

→ 사용자가 클라이언트 서비스 운영하면 cron 작업이 매일 자동.

**🎯 핵심 메시지**: at = 1회, cron = 주기적. 5필드 (분·시·일·월·요일).

📎 **실습 ✨**: [crontab.guru](https://crontab.guru)에서 cron 표기 실시간 해석.

---

### 챕터 5. SSH ⭐ (슬라이드 31~40)

**키워드 (강의 본문)**:
- **SSH** = Secure Shell. 네트워크로 다른 컴퓨터 접근·명령 실행
- vs **Telnet**: 옛 원격 접속. **암호화 없음** → 비밀번호 노출
- SSH는 **암호화** = 안전
- **OpenSSH**: 우분투에서 SSH 패키지
  - openssh-**client** (접속) — 보통 기본
  - openssh-**server** (받기) — 별도 설치
- `sudo service ssh start` — 서비스 시작
- `netstat -antp`로 포트 확인. **SSH 기본 포트 22**
- `ssh [사용자]@[IP/도메인]` 접속

✦ **강의자 풀이 추정**
**SSH 일상 사용**:
```bash
# 기본
ssh user@192.168.1.100

# 도메인
ssh user@server.example.com

# 포트 지정
ssh -p 2222 user@server.com

# SSH 키 인증 (AWS·GitHub 등)
ssh -i ~/.ssh/my_key.pem ec2-user@ec2-instance.com

# 명령 1회 실행
ssh user@server.com "ls -la"
```

→ **사용자가 GitHub SSH 키** 등록 본 적 있으면 SSH 일상. `git clone git@github.com:...`도 SSH.

→ AWS EC2·GCP VM 만들면 키 받고 SSH로 접속.

**🎯 핵심 메시지**: SSH = 암호화 원격 접속. Telnet 대체. 기본 포트 22.

⚠️ **흔한 오해**: "SSH 포트는 무조건 22" → 보안상 변경 권장 (2222·다른 번호). 자동 공격 회피.

📎 **실습 ✨**: macOS 터미널에서 `ssh user@server` 형식 시도. GitHub SSH 키 설정도 같은 패턴.

## 💡 일상 비유

### 비유 1 — 카카오톡 영상 다운로드 (백그라운드)

영상 다운로드 시작 → 백그라운드 진행 → 그동안 다른 채팅 → 끝나면 알림. job·프로세스 관리의 일상.
