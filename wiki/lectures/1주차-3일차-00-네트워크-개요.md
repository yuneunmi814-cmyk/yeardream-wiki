## 🚀 예습 (5분용)

### 한 줄 요약

**네트워크 = 두 대 이상의 컴퓨터를 연결해 데이터를 공유하는 시스템**. 케이블·전파로 신호 보내고, 토폴로지로 묶고, 주소(IP)로 식별하고, 7계층(OSI)으로 통신을 추상화.

### 핵심 단어 5~7개

- [[OSI-7계층]] ⭐ — 통신을 7개 계층으로 나눈 표준 모델 (강의 전체의 골격)
- [[IP-주소]] — 네트워크에서의 내 컴퓨터 주소
- [[MAC-주소]] — 물리적 컴퓨터 주소
- [[네트워크-토폴로지]] — 노드 연결 구조 (스타·버스·링·메시)
- [[네트워크-유형]] — PAN·LAN·MAN·WAN (개인~광역)
- [[네트워크-통신방식]] — 유니/멀티/브로드캐스트
- [[라우팅]] — 패킷을 최적 경로로 보내는 알고리즘

### 큰 그림

```
물리(케이블·전파)
    ↓
연결 구조 (토폴로지: 스타·버스·링·메시)
    ↓
범위 (PAN < LAN < MAN < WAN)
    ↓
식별 (IP 주소 + MAC 주소)
    ↓
전달 방식 (유니/멀티/브로드캐스트)
    ↓
구성 (Edge · Access · Core + 라우팅·포워딩)
    ↓
추상화 (OSI 7계층) ⭐
```

### 챕터 1. 네트워크란? (슬라이드 1~12)

**키워드 (강의 본문)**:
> 노드 (Node), 네트워크 = 두 대 이상의 노드 연결, 장점(자료 공유·미디어 시청), 단점(바이러스·정보 유출),
> 유선 매체(동축·이더넷·광섬유) / 무선 매체(라디오파·마이크로파),
> 해저 광케이블이 국제 인터넷 90% 책임

✦ **강의자 풀이 추정** (LLM 보강)

**네트워크의 가장 기본 정의**: "노드 ≥ 2개 + 연결 매체". 노드는 컴퓨터·휴대폰·라우터·서버 등 데이터를 주고받는 장치. 두 대만 케이블로 연결해도 네트워크.

**유선 매체 3종**:
- **동축 케이블**: 안테나선. 차폐 구조 → 노이즈에 강함, 고주파 장거리 전송
- **이더넷 케이블 (랜선)**: LAN 구축용. 가정·사무실 표준
- **광섬유 케이블**: 레이저 사용. 구리선보다 훨씬 빠름. **해저 케이블 = 국제 인터넷의 척추**

**무선 매체 2종**:
- **라디오파**: 전 방향 전파 + 벽 통과 가능. 와이파이·셀룰러
- **마이크로파**: 설치 쉽고 장애물에 덜 민감. 위성·기지국 통신

**🎯 이 부분의 핵심 메시지**: 네트워크는 **물리 매체로 시작**. 모든 추상화(IP·HTTP·HTTPS)의 밑에는 결국 케이블이나 전파가 있다.

⚠️ **흔한 오해**: "인터넷 = 무선"은 틀림. 국제 인터넷의 90% 이상은 **해저 광케이블**(유선). 와이파이는 마지막 마일(집~공유기)만 무선.

---

### 챕터 2. 네트워크 토폴로지 (슬라이드 13~18)

**키워드 (강의 본문)**:
> 토폴로지 = 노드 연결 구조. 스타·버스·링·메시(완전 연결).
> 스타: 중앙 시스템 문제 시 전체 중단, 제어 복잡, 비용 큼
> 버스: 단순·저렴, 노드 한 곳 에러 시 전체 중단, 동시 전송 불가
> 링: 운용 조직적, 중간 노드 사망 시 데이터 막힘
> 메시(완전 연결): 이중화·대용량, 구축·운영 고가

✦ **강의자 풀이 추정** (LLM 보강)

**토폴로지**는 "노드를 어떻게 물리적으로 묶었는가". 학교 와이파이는 보통 **스타형** (공유기가 중심 노드).

| 토폴로지 | 모양 | 장점 | 단점 | 실사용 예 |
|---|---|---|---|---|
| **스타** | 중앙 허브 + 가지 | 관리 쉬움 | 중앙 망가지면 전체 중단 | 가정 와이파이 |
| **버스** | 일자 선에 줄줄이 | 저렴·간단 | 한 곳 끊기면 전체 중단 | 옛 이더넷 |
| **링** | 원형 | 제어 단순 | 중간 사망 시 막힘 | Token Ring (구식) |
| **메시(완전)** | 모든 노드 상호 연결 | 이중화·고대역폭 | 비용 폭증 | 데이터센터 백본 |

**🎯 이 부분의 핵심 메시지**: 토폴로지는 **신뢰성 vs 비용 트레이드오프**. 메시는 가장 튼튼하지만 가장 비싸다.

---

### 챕터 3. 네트워크 유형 (슬라이드 19~24)

**키워드 (강의 본문)**:
> PAN (Personal Area Network) — 개인 휴대기기 무선 연결
> LAN (Local Area Network) — 지역 범위
> MAN (Metropolitan Area Network) — 도시 범위
> WAN (Wide Area Network) — 광대한 지역

✦ **강의자 풀이 추정** (LLM 보강)

**범위 기준 분류** — 작은 것부터 큰 것 순으로:

```
PAN (~10m)         LAN (~1km)        MAN (~50km)        WAN (지구 전체)
이어폰·블루투스    집·사무실 와이파이  도시 광역망         인터넷
```

- **PAN**: 블루투스 이어폰, AirDrop
- **LAN**: 집 공유기 아래 컴퓨터·핸드폰들
- **MAN**: 한 도시 안의 여러 LAN 묶음 (드물게 일반 사용자가 의식)
- **WAN**: 도시·국가·대륙을 잇는 모든 것. **인터넷 = 가장 큰 WAN**

**🎯 이 부분의 핵심 메시지**: 네트워크는 **범위 크기 4단계**로 분류. PAN ⊂ LAN ⊂ MAN ⊂ WAN.

### 챕터 4. IP 주소 (슬라이드 25~38)

**키워드 (강의 본문)**:
> IP 주소 = 네트워크에서의 컴퓨터 주소
> MAC 주소 = 물리적 컴퓨터 주소
> IP 주소 구성 = 네트워크 ID + 호스트 ID
> 서브넷 마스크 = 로컬 네트워크 구분 수단
> IP 클래스 A~E, IPv4 = 32비트(약 43억개), IPv6 = 128비트
> 사설 네트워크 / 공용 네트워크 / 사설 IP vs 공인 IP
> NAT (Network Address Translation) — IP 주소 부족 해결책

✦ **강의자 풀이 추정** (LLM 보강)

**IP 주소 vs MAC 주소** — 가장 헷갈리는 두 개:

| | IP 주소 | MAC 주소 |
|---|---|---|
| **본질** | 논리적 (네트워크에서의 주소) | 물리적 (하드웨어에 박힌 주소) |
| **변경** | 자주 바뀜 (DHCP, 와이파이 바꿀 때) | 거의 안 바뀜 (제조 시 고정) |
| **형식** | 17.172.213.53 (IPv4) | AA:BB:CC:DD:EE:FF |
| **비유** | 우리 집 주소 (이사 가능) | 주민등록번호 |

**IPv4 vs IPv6** ([[IP-주소]] 참조):
- **IPv4**: 32비트 = 약 43억개. 1980년대 표준. **이미 고갈됨** → NAT가 임시 해결책
- **IPv6**: 128비트 = 43억⁴개 (사실상 무한). 현재 전환 중

**사설 IP vs 공인 IP**:
- **사설 IP**: 집/회사 안에서만 유효 (`192.168.x.x` 같은 거). 외부에서 못 봄
- **공인 IP**: 인터넷 전체에서 유일. ISP(통신사)가 할당

**NAT**: 사설 IP를 공인 IP로 번역해서 인터넷 보냄. 집의 와이파이 공유기가 매일 하는 일.

```
[내 노트북: 192.168.0.5 (사설)]
        ↓
[공유기 (NAT): 공인 IP 222.111.x.x]
        ↓
[인터넷]
```

**🎯 이 부분의 핵심 메시지**: IP는 **논리적 주소** (소프트웨어), MAC은 **물리적 주소** (하드웨어). NAT 덕분에 43억개 IPv4가 아직 버틴다.

⚠️ **흔한 오해**: "IP 주소 = 위치". IP는 **네트워크에서의 식별자**이지 GPS 좌표 아님. VPN 쓰면 IP만 바뀌고 물리적 위치는 그대로.

### 챕터 5. 네트워크 통신방식 (슬라이드 39~43)

**키워드 (강의 본문)**:
> 유니캐스트 = 1:1, 가장 많이 사용, CPU 부담 적음
> 멀티캐스트 = 1:N (특정 그룹), 일부에게만
> 브로드캐스트 = 1:전체, 로컬 네트워크 모든 시스템에 프레임 전송
> 프레임 = 데이터 컨테이너, 라우터 = 최적 경로 찾는 장비

✦ **강의자 풀이 추정** (LLM 보강)

**3가지 통신 방식**:

| 방식 | 대상 | 비유 |
|---|---|---|
| **유니캐스트** | 1:1 (특정 한 명) | 카톡 DM |
| **멀티캐스트** | 1:N (특정 그룹) | 카톡 단톡방 |
| **브로드캐스트** | 1:전체 (모두) | 학교 전체 방송 |

- **유니캐스트**: 가장 흔함. 웹사이트 보면 서버 → 내 노트북 1:1. CPU 부담 적음.
- **멀티캐스트**: IPTV 방송 등. 같은 채널 보는 사람만 받음.
- **브로드캐스트**: 같은 LAN 안의 모든 기기에 동시 송신. 라우터끼리 서로 인사할 때나 새 기기 찾을 때.

**🎯 이 부분의 핵심 메시지**: 통신 방식은 **수신자 수**로 분류. 1·N·전체.

### 챕터 6. 네트워크의 구성 (슬라이드 44~50)

**키워드 (강의 본문)**:
> 네트워크 엣지 (Edge) — 가장자리. 종단 시스템
> 접속 네트워크 (Access) — 엣지를 첫 번째 라우터에 연결
> 네트워크 코어 (Core) — 중앙. 데이터 전송 핵심
> 포워딩 = 라우터로 들어온 패킷을 올바른 출구로 (테이블 참조)
> 라우팅 = 패킷이 어디로 갈지 판단하는 과정

✦ **강의자 풀이 추정** (LLM 보강)

**네트워크 3층 구조**:

```
┌──────── Edge ────────┐    ← 내 노트북·휴대폰 (종단)
│                       │
│   Access Network      │    ← 통신사 첫 라우터 (집 공유기 → ISP 라우터)
│                       │
│   ╔═══ CORE ═══╗     │    ← 인터넷 백본 (대형 라우터들)
│   ║            ║     │
│   ╚════════════╝     │
└───────────────────────┘
```

**포워딩 vs 라우팅** — 자주 헷갈림:
- **라우팅**: 패킷의 **경로를 결정**하는 과정 (알고리즘, "어디로 갈지")
- **포워딩**: 라우터가 들어온 패킷을 **올바른 출구 포트로 보내는** 동작 (테이블 룩업, "어느 문으로 내보낼지")

→ **라우팅 = 지도 만들기**, **포워딩 = 지도 보고 갈림길에서 방향 결정**.

라우팅 알고리즘으로 만들어진 결과 = **포워딩 테이블**. 라우터는 패킷 헤더의 목적지 IP를 보고 테이블 참조 → 출력 포트 결정.

**🎯 이 부분의 핵심 메시지**: 인터넷은 **Edge–Access–Core 3층 구조**. 라우팅(경로 계획)과 포워딩(경로 실행)이 분리.

### 챕터 7. OSI 7계층 ⭐ (슬라이드 51~60)

**키워드 (강의 본문)**:
> OSI = Open System Interconnection
> 7계층 모델: 응용·표현·세션·전송·네트워크·데이터링크·물리
> 각 계층마다 다른 프로토콜·데이터 단위·역할
> 7 응용 (HTTP·SMTP·SSH) — 사용자가 만나는 서비스
> 6 표현 (JPEG·MPEG) — 인코딩/디코딩, 암호화
> 5 세션 (SSL·TLS) — 세션 만들고 없앰
> 4 전송 (TCP·UDP) — 종단 간 신뢰성·오류 검출
> 3 네트워크 (IP·IPX·ARP) — 경로 설정, IP 주소
> 2 데이터링크 (Ethernet) — MAC 주소, 프레임 단위
> 1 물리 (전압·허브) — 비트 단위

✦ **강의자 풀이 추정** (LLM 보강) — **강의 전체의 골격** ⭐

**OSI 7계층** = 통신을 **7개 추상화 단계로 쪼갠 표준 모델**. 1980년대 ISO가 만듦. 실무에선 TCP/IP 4계층(다음 강)이 더 자주 쓰이지만, 개념 이해는 7계층이 표준.

```
계층    이름            데이터 단위    예시 프로토콜            역할
─────────────────────────────────────────────────────────────────
7    응용 (Application)   Data       HTTP, SMTP, SSH, DNS    사용자 서비스
6    표현 (Presentation)  Data       JPEG, MPEG, ASCII       인코딩·암호화
5    세션 (Session)       Data       SSL, TLS, NetBIOS       세션 관리
4    전송 (Transport)     Segment    TCP, UDP                종단 간 신뢰성
3    네트워크 (Network)   Packet     IP, ICMP, ARP           경로 설정 (IP)
2    데이터링크 (DataLink) Frame      Ethernet, Wi-Fi         MAC 주소
1    물리 (Physical)      Bit        전압·케이블             전기 신호
─────────────────────────────────────────────────────────────────
```

**왜 7계층으로 나누는가?**

각 계층은 **위·아래만 신경 씀**. 예: HTTP(7층)는 IP(3층)가 어떻게 패킷을 라우팅하는지 몰라도 됨. TCP(4층)는 케이블이 광섬유인지 구리인지 몰라도 됨. **추상화 = 복잡성 관리**.

**암기 트릭** (영어 첫글자):
- **A**ll **P**eople **S**eem **T**o **N**eed **D**ata **P**rocessing
- (Application·Presentation·Session·Transport·Network·DataLink·Physical)

**데이터 단위 변화** — 위에서 아래로 가면서 헤더가 붙음:
```
[Data]                      ← 7~5 계층
[TCP|Data]                  ← 4 계층 (Segment)
[IP|TCP|Data]               ← 3 계층 (Packet)
[Frame|IP|TCP|Data|CRC]     ← 2 계층 (Frame)
01010101...                 ← 1 계층 (Bit)
```

**🎯 이 부분의 핵심 메시지**: OSI 7계층은 **통신을 추상화한 표준 그릇**. 4·3·2계층(TCP·IP·Ethernet)이 인터넷 실무의 핵심.

⚠️ **흔한 오해**: "OSI 7계층 = 실제 인터넷 구조". 실제 인터넷은 **TCP/IP 4계층**(다음 강)으로 동작. OSI는 **개념 모델**이고 실무에선 TCP/IP가 표준.

## 💡 일상 비유

### 비유 1: OSI 7계층 = 택배 시스템

택배 보낼 때 단계별 책임:
- **7층 (응용)** = 보내는 사람 ("선물 보낼래")
- **4층 (전송)** = 택배회사가 박스 묶고 추적번호 부여 (TCP의 일 sequence·재전송)
- **3층 (네트워크)** = 주소지 라벨링 + 어느 허브 거칠지 결정 (IP)
- **2층 (데이터링크)** = 트럭에 싣기, 트럭 번호판 (MAC)
- **1층 (물리)** = 실제 도로·트럭 바퀴

상대방은 역순으로 받음. 각 단계는 **자기 일만 신경 씀** — 보내는 사람은 트럭 번호판 모르고, 트럭 기사는 박스 안 내용물 모름.

### 비유 2: 네트워크 = 아파트 단지

- **물리 매체** = 도로
- **토폴로지** = 단지 구조 (단일동·여러 동·격자형)
- **PAN/LAN/MAN/WAN** = 집 안 → 단지 안 → 동네 → 도시·국가
- **IP 주소** = 호수 (102동 503호)
- **MAC 주소** = 주민등록번호 (이사 가도 안 바뀜)
- **유니캐스트** = 옆집에 인사
- **브로드캐스트** = 단지 방송 ("관리실에서 알립니다")
- **라우터** = 단지 정문 경비 (어디 보낼지 결정)
