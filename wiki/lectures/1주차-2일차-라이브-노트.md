## 강의자 인사이트 (메모)

### 1. LLM 기본 문서 형식 = PDF + 마크다운

LLM에 자료 넣을 때 표준 형식. 그래서 위키도 마크다운으로 작성하는 게 자연스러움 ([[마크다운]] 참조).

✦ **연결**: 사용자가 매일 raw OCR 마크다운 → wiki 마크다운 → Claude API에 입력. 전체 워크플로우가 마크다운 위에서 돈다.

### 2. 폴더 만들 때도 리눅스 활용해서 연습하기

GUI(Finder) 대신 터미널에서 `mkdir`로 폴더 생성 연습. 손에 익히기.

✦ **실습 권장**:
```bash
# 일상에서 한 번씩 시도
mkdir -p ~/projects/yoon/2026-05/test
cd ~/projects/yoon/2026-05/test
touch hello.md
ls -al
```

→ [[VI-편집기]] 별도 페이지로 정리. 6개월 동안 점진적 익히기.

### 4. echo `>` vs `>>` 차이 확실히

[[1주차-2일차-06-리눅스-명령어]] 챕터 3에 있는 내용. 다시 강조:

```bash
echo "hello" > file.txt    # 덮어쓰기 (있던 내용 다 지움)
echo "world" >> file.txt   # 이어쓰기 (기존 내용 뒤에 추가)
```

⚠️ **흔한 실수**: `>` 한 개로 중요한 파일에 쓰면 원래 내용 다 날아감. **로그·설정 파일엔 `>>` 가 안전**.
