### 2. 폴더 만들 때도 리눅스 활용해서 연습하기

→ [[VI-편집기]] 별도 페이지로 정리. 6개월 동안 점진적 익히기.

### 4. echo `>` vs `>>` 차이 확실히

[[1주차-2일차-06-리눅스-명령어]] 챕터 3에 있는 내용. 다시 강조:

```bash
echo "hello" > file.txt    # 덮어쓰기 (있던 내용 다 지움)
echo "world" >> file.txt   # 이어쓰기 (기존 내용 뒤에 추가)
```

⚠️ **흔한 실수**: `>` 한 개로 중요한 파일에 쓰면 원래 내용 다 날아감. **로그·설정 파일엔 `>>` 가 안전**.

```
백엔드:
-- Python 프로그래밍 로직 구현 (속도)
```

### 서비스 기획 방법론 ⭐ — 초보 vs 고수

**초보**: 일반 사용자의 서비스 로직만 생각
**고수**:
- 일반 사용자: 테스트 / 베타테스트 모집 / 링크 전송 / 피드백

- **관리자 페이지 + 보안** 고려
- **개인정보 처리 방법** → **법령** 검토

→ "고수" = 너가 가야 할 방향 (1인 컨설팅·yoon.ai.lab 운영자).

### 고연차 vs 저연차 ⭐

| | 정의 |
|---|---|
| **고연차** | 에러 보고 **즉각** 뭘 해야 할지 **아는** 사람 |
| **저연차** | 에러 보고 즉각 뭘 해야 할지 **모르는** 사람 |

→ **결국 프롬프트 질의 차이로 직결**. 고연차 = AI한테 정확한 질문 가능.

→ 너 학습 목표 = **에러 패턴 인식 능력** 기르기. 매일 정수 추출 패턴이 이걸 키움.

### 도구 진화 흐름

```
프로세스 관리 어려움 → Docker (컨테이너화)
       ↓
Docker도 관리 어려움 → Kubernetes (오케스트레이션)
       ↓
밑바닥 = 리눅스 = 가장 중요 (강사 강조)
```

### 컨테이너화 = 이삿짐센터 ⭐ (은미 비유 정답)

> "컨테이너 = 박스. OS·개발환경·웹 환경 통째로 담아 배송. 이삿짐센터 배송하듯이 내가 쓰던 환경이랑 패키지를 그대로 다운받을 수 있음"

### 쿠팡 패턴 — 분산 컨테이너 (보안 > 편의성)

> "쿠팡이 사용하는 포트 몇 개? 프로그램·MySQL·Oracle·Postgres·Elasticsearch 하나 컨테이너에 담아도 괜찮을까? 아니. 분산해야."

→ **편의성 < 보안**. 한 컨테이너 뚫리면 다 노출.

→ 각자 다른 컨테이너 = 각자 다른 포트 = **쿠버네티스가 관리**.
