## 프로세스 (강의자 명시)

```
1. 웹사이트 접속 → 책 정보 수집 (크롤링, 5페이지 분량)
       ↓
2. 수집 데이터 → GitHub 블로그 양식 .md로 변환
       ↓
3. git push (bash 스크립트)
       ↓
4. 위 모든 작업 순차 스케줄링 ([[1주차-2일차-07-프로세스와-네트워크|cron]] 또는 GitHub Actions)
```

### Phase 2 — Markdown 변환 (1일)

```python
def to_markdown(book):
    return f"""---
title: "{book['title']}"
author: {book['author']}
date: {book['date']}
---

# {book['title']}

{book['description']}
"""
```

### Phase 3 — Git Push 자동화 (1일)

```bash
#!/bin/bash
cd ~/my-blog
git add posts/*.md
git commit -m "📚 자동 업로드: $(date)"
git push origin main
```

### Phase 4 — 스케줄링 (1일)

**옵션 A — GitHub Actions Cron** ⭐ 권장:
```yaml
# .github/workflows/auto-post.yml
on:
  schedule:
    - cron: '*/5 * * * *'   # 5분마다
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: python crawl_and_publish.py
```

**옵션 B — 로컬 cron** (Mac):
```bash
crontab -e
*/5 * * * * /Users/yoon/auto-post.sh
# ⚠️ 노트북 켜져 있을 때만 동작
```

→ **GitHub Actions가 우월**. 노트북 꺼도 동작 + 무료.
